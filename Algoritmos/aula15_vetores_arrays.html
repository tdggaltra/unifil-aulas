🤔 Por que precisamos de vetores?

O problema de guardar muitos valores com variáveis simples

Guardar 100 notas em 100 variáveis é inviável. Um vetor armazena todas em uma única estrutura!

❌ Sem vetor — inviável

// 10 alunos = 10 variáveis var nota1, nota2, nota3 : real nota4, nota5, nota6 : real nota7, nota8, nota9 : real nota10 : real // 100 alunos = 100 linhas! // Impossível percorrer com for. // Impossível generalizar para N.

✅ Com vetor — elegante

// 100 notas em 1 variável! var notas: vetor[1..100] de real // Percorrer com for — simples: para i <- 1 ate 100 faca leia(notas[i]) fimpara // Fácil de mudar para 1000!

🗃️ Analogia: Gavetas Numeradas

Um vetor é como um armário com gavetas numeradas. Cada gaveta guarda um valor. O número da gaveta é o índice. Você acessa qualquer gaveta instantaneamente pelo seu número — sem precisar abrir todas as anteriores.

🌍 Vetores no cotidiano

  • Placar de um jogo com N jogadores
  • Lista de contatos do celular
  • Histórico de transações bancárias
  • Pixels de uma linha de imagem

📐 Características

  • Mesmo tipo para todos os elementos
  • Tamanho fixo definido na declaração
  • Acesso por índice em tempo constante
  • Percorrido naturalmente com for

// Declaração int idades[5]; // Atribuição idades[0] = 25; idades[1] = 32; // Leitura com for for(int i=0; i<5; i++) scanf("%d", &idades[i]); // Índices: 0 até 4 // CUIDADO: idades[5] = erro!

🟠 Java

// Declaração int[] idades = new int[5]; // Atribuição idades[0] = 25; idades[1] = 32; // Leitura com for for(int i=0; i<5; i++) idades[i] = sc.nextInt(); // Índices: 0 até 4 // idades.length == 5

⚠️ Atenção: índice 0 vs índice 1

Portugol: índices começam em 1 (você define o intervalo: [1..N]).
C e Java: índices começam em 0 e vão até N-1. Acessar vetor[N] em C/Java causa erro de memória (segfault ou ArrayIndexOutOfBoundsException)!

⚙️ Operações Básicas com Vetores

Leitura, escrita, soma, média, maior e menor

🟢 Leitura + Impressão + Estatísticas (Portugol)

algoritmo "VetorBasico" var v : vetor[1..5] de real i : inteiro soma, maior, menor: real inicio // === LEITURA === para i <- 1 ate 5 faca escreva("v[", i, "]: ") leia(v[i]) fimpara // === ESTATÍSTICAS === soma <- 0 maior <- v[1] menor <- v[1] para i <- 1 ate 5 faca soma <- soma + v[i] se v[i] > maior entao maior <- v[i] fimse se v[i] < menor entao menor <- v[i] fimse fimpara // === EXIBIÇÃO === escreva("Vetor: [") para i <- 1 ate 5 faca escreva(v[i]) se i < 5 entao escreva(", ") fimse fimpara escreva("]") escreva("Soma : ", soma) escreva("Média : ", soma/5:5:2) escreva("Maior : ", maior) escreva("Menor : ", menor) fimalgoritmo

⚡ Vantagem do vetor sobre variáveis simples

  • Mesma fórmula funciona para N=5 ou N=1000

🔍 Busca Linear em Vetores

Procurar um elemento percorrendo o vetor posição a posição

📌 Problema

Dado um vetor de N notas, verificar se uma nota específica existe. Se sim, exibir sua posição. Se não, informar que não foi encontrada.

🟢 Portugol — Busca Linear

algoritmo "BuscaLinear" var v : vetor[1..8] de real alvo : real i, pos : inteiro achado : logico inicio // Carga do vetor para i <- 1 ate 8 faca escreva("v[",i,"]: ") leia(v[i]) fimpara escreva("Buscar: ") leia(alvo) // Busca linear achado <- falso pos <- -1 para i <- 1 ate 8 faca se (v[i] == alvo) entao achado <- verdadeiro pos <- i interrompa ← break! fimse fimpara se (achado) entao escreva("✅ Encontrado na pos. ", pos) senao escreva("❌ Não encontrado") fimse fimalgoritmo

📊 Complexidade da Busca Linear

  • Melhor caso: elemento na posição 1 → 1 comparação
  • Pior caso: elemento no fim ou ausente → N comparações
  • Caso médio: N/2 comparações
  • Notação: O(N) — linear no tamanho do vetor

✅ Busca linear em C

int pos = -1; for(int i=0; i<n; i++) if(v[i]==alvo){ pos=i; break; } if(pos==-1) printf("Nao achou\n"); else printf("Pos: %d\n", pos);

🔀 Ordenação — Bubble Sort

Ordenar um vetor comparando pares adjacentes e trocando

🫧 Por que "Bubble" (bolha)?

Os maiores valores "sobem" para o final a cada passada — como bolhas subindo em água. É o algoritmo mais simples de entender, ideal para aprendizado.

🟢 Portugol — Bubble Sort

algoritmo "BubbleSort" var v : vetor[1..5] de inteiro i, j, temp: inteiro inicio // Carga para i <- 1 ate 5 faca leia(v[i]) fimpara // Ordenação — 2 fors aninhados para i <- 1 ate 4 faca ← passadas para j <- 1 ate 5-i faca ← comparações se (v[j] > v[j+1]) entao temp <- v[j] v[j] <- v[j+1] v[j+1] <- temp fimse fimpara fimpara // Exibir ordenado escreva("Ordenado: [") para i <- 1 ate 5 faca escreva(v[i]) se i<5 entao escreva(", ") fimse fimpara escreva("]") fimalgoritmo

🔑 Chave da troca

temp = v[j]; v[j] = v[j+1]; v[j+1] = temp;
Sempre use uma variável temporária temp — sem ela você perde um dos valores!

💻 Exemplo Completo — Sistema de Notas

Leitura, estatísticas, busca e ordenação em um único programa

📌 Problema Intermediário

Leia as notas de 6 alunos. Calcule média, maior e menor. Exiba aprovados (≥ 5). Ordene as notas em ordem crescente e exiba.

🔵 C — Sistema de Notas Completo

#include <stdio.h> #define N 6 int main() { float v[N], temp, soma=0; float maior, menor; int aprov=0; // === LEITURA === for(int i=0; i<N; i++) { printf("Nota %d: ", i+1); scanf("%f", &v[i]); } // === ESTATÍSTICAS === maior = menor = v[0]; for(int i=0; i<N; i++) { soma += v[i]; if(v[i]>maior) maior=v[i]; if(v[i]<menor) menor=v[i]; if(v[i]>=5.0f) aprov++; } printf("Media: %.2f\n", soma/N); printf("Maior: %.1f | Menor: %.1f\n", maior, menor); printf("Aprovados: %d/%d\n", aprov, N); // === ORDENAÇÃO (Bubble Sort) === for(int i=0; i<N-1; i++) for(int j=0; j<N-1-i; j++) if(v[j]>v[j+1]){ temp=v[j]; v[j]=v[j+1]; v[j+1]=temp; } // === EXIBIÇÃO ORDENADA === printf("Ordenado: ["); for(int i=0; i<N; i++) printf("%.1f%s", v[i], i1?", ":"]"); return 0; }

Guardamos os dados na memória, processamos estatísticas, depois ordenamos — tudo usando o mesmo vetor. Com variáveis simples, isso seria impossível.

💻 Vetores de Texto (Strings)

Armazenar nomes, palavras e frases em vetores

📋 Vetor de nomes — Portugol

algoritmo "VetorNomes" var nomes : vetor[1..5] de caractere notas : vetor[1..5] de real i : inteiro inicio para i <- 1 ate 5 faca escreva("Nome ", i, ": ") leia(nomes[i]) escreva("Nota: ") leia(notas[i]) fimpara escreval("=== BOLETIM ===") para i <- 1 ate 5 faca escreva(nomes[i], ": ", notas[i]) se (notas[i] >= 5.0) entao escreva(" ✅ APROVADO") senao escreva(" ❌ REPROVADO") fimse fimpara fimalgoritmo

🔵 C — Array de strings

#include <stdio.h> #define N 5 int main() { // Array de strings: char[N][tamanho] char nomes[N][50]; float notas[N]; for(int i=0; i<N; i++) { printf("Nome %d: ", i+1); scanf("%49s", nomes[i]); printf("Nota : "); scanf("%f", &notas[i]); } printf("\n=== BOLETIM ===\n"); for(int i=0; i<N; i++) printf("%-20s %.1f %s\n", nomes[i], notas[i], notas[i]>=5.0f? "APROVADO": "REPROVADO"); return 0; }

💡 Dois vetores paralelos

nomes[i] e notas[i] representam o mesmo aluno. Isso se chama vetores paralelos — uma técnica antes de existir orientação a objetos. O índice i é o elo que liga os dois vetores.

🚨 Erros Comuns com Vetores

Armadilhas que causam bugs difíceis de rastrear

❌ Erro 1: Acesso fora dos limites (out of bounds)

// Vetor de tamanho 5 em C int v[5]; ← índices 0 a 4 v[5] = 10; ← ERRO! Índice 5 não existe! v[-1] = 20; ← ERRO! Índice negativo! // Em Java: lança ArrayIndexOutOfBoundsException // Em C: corrompe memória silenciosamente! // CORRETO for(int i=0; i<5; i++) ← usa < não <= v[i] = i*2;

❌ Erro 2: Não inicializar antes de somar

// ERRADO: soma tem lixo na memória int soma; for(int i=0; i<n; i++) soma += v[i]; ← resultado imprevisível! // CORRETO int soma = 0; for(int i=0; i<n; i++) soma += v[i];

❌ Erro 3: Confundir tamanho e último índice

// Vetor com 5 elementos em C int v[5] = {10,20,30,40,50}; // tamanho = 5 // último índice = 4 (não 5!) // ERRADO: <= com tamanho for(int i=0; i<=5; i++) ← acessa v[5]! // CORRETO: < com tamanho for(int i=0; i<5; i++) ← 0 a 4 ✓

❌ Erro 4: Troca sem temporário

// ERRADO: perde o valor de v[j] v[j] = v[j+1]; ← v[j] perdido! v[j+1] = v[j]; ← copia v[j+1] de novo! // CORRETO: usa temp temp = v[j]; v[j] = v[j+1]; v[j+1] = temp; ← ✓

✅ Checklist de vetores

☑ C/Java: índice vai de 0 a N-1  |  ☑ Acumuladores inicializados com 0  |  ☑ Condição do for: i < N (não i <= N)  |  ☑ Troca de elementos: sempre usar variável temp
