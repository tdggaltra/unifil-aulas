⚖️ Operadores Relacionais

Comparando valores

Operadores relacionais comparam dois valores e retornam verdadeiro ou falso
Operador Nome Exemplo Resultado Significado
> Maior que 5 > 3 verdadeiro 5 é maior que 3?
< Menor que 5 < 3 falso 5 é menor que 3?
>= Maior ou igual 5 >= 5 verdadeiro 5 é maior ou igual a 5?
<= Menor ou igual 3 <= 5 verdadeiro 3 é menor ou igual a 5?
== Igual a 5 == 5 verdadeiro 5 é igual a 5?
!= Diferente de 5 != 3 verdadeiro 5 é diferente de 3?

⚠️ ERRO MUITO COMUM!

Atribuição (=) é diferente de Comparação (==):

int x = 5; // ✅ ATRIBUIÇÃO: x recebe 5 if (x == 5) { // ✅ COMPARAÇÃO: x é igual a 5? printf("Igual!"); } if (x = 5) { // ❌ ERRO! Atribuindo 5 a x, não comparando! printf("Bug!"); }

Use = para atribuir, == para comparar!

💡 Observação Importante

O resultado de uma comparação é sempre um valor lógico (booleano):

  • ✅ verdadeiro (Portugol), true (Java), 1 (C)
  • ❌ falso (Portugol), false (Java), 0 (C)

📝 Exemplos Práticos de Comparações

Testando operadores relacionais

🟢 Portugol

programa { funcao inicio() { inteiro idade = 20 logico resultado resultado = idade >= 18 escreva(resultado) // verdadeiro resultado = idade == 18 escreva(resultado) // falso resultado = idade != 20 escreva(resultado) // falso } }

🔵 C

#include <stdio.h> int main() { int idade = 20; int resultado; resultado = idade >= 18; printf("%d\n", resultado); // 1 (true) resultado = idade == 18; printf("%d\n", resultado); // 0 (false) resultado = idade != 20; printf("%d\n", resultado); // 0 (false) return 0; }

🟠 Java

public class Exemplo { public static void main(String[] args) { int idade = 20; boolean resultado; resultado = idade >= 18; System.out.println(resultado); // true resultado = idade == 18; System.out.println(resultado); // false resultado = idade != 20; System.out.println(resultado); // false } }

🔗 Operadores Lógicos

Combinando condições

Operadores lógicos combinam múltiplas condições em uma única expressão
&&

E / AND

Retorna verdadeiro se AMBAS as condições forem verdadeiras

(idade >= 18) && (cpf != "") // Maior de idade E tem CPF

Portugol: e
C/Java: &&

||

OU / OR

Retorna verdadeiro se PELO MENOS UMA condição for verdadeira

(dia == "sábado") || (dia == "domingo") // Sábado OU domingo

Portugol: ou
C/Java: ||

!

NÃO / NOT

Inverte o valor lógico: verdadeiro vira falso, falso vira verdadeiro

!(idade < 18) // NÃO é menor de idade // Equivale a: idade >= 18

Portugol: nao
C/Java: !

🚪 Analogia: Portas de Entrada

E (AND): Porta com 2 chaves. Você precisa das DUAS chaves para abrir.
→ Exemplo: Entrar na boate (ser maior de 18 E estar na lista)

OU (OR): Porta com 2 fechaduras. Qualquer uma das chaves abre.
→ Exemplo: Pagar conta (dinheiro OU cartão)

NÃO (NOT): Porta com trava invertida. Se está aberta, fecha. Se está fechada, abre.
→ Exemplo: Entrar se NÃO estiver proibido

📊 Tabelas Verdade

Todas as combinações possíveis

Tabelas verdade mostram o resultado de operações lógicas para todas as combinações de entrada

E / AND (&&)

A B A && B
F F FALSO
F V FALSO
V F FALSO
V V VERDADEIRO

Só é verdadeiro quando AMBOS são verdadeiros

OU / OR (||)

A B A || B
F F FALSO
F V VERDADEIRO
V F VERDADEIRO
V V VERDADEIRO

É verdadeiro se PELO MENOS UM for verdadeiro

NÃO / NOT (!)

A !A
FALSO VERDADEIRO
VERDADEIRO FALSO

Sempre INVERTE o valor

💡 Resumo Rápido

  • E (&&): Só é verdadeiro se TUDO for verdadeiro
  • OU (||): Basta UM ser verdadeiro
  • NÃO (!): Inverte o valor

🧮 Exemplos de Expressões Lógicas

Combinando operadores

📌 Exemplo 1: Verificar Maioridade

int idade = 20; // Maior ou igual a 18? boolean maiorIdade = idade >= 18; // true if (maiorIdade) { System.out.println("Pode entrar!"); }

📌 Exemplo 2: Faixa de Valores (E)

int nota = 75; // Nota entre 70 e 100? boolean notaBoa = (nota >= 70) && (nota <= 100); // true if (notaBoa) { System.out.println("Aprovado com distinção!"); }

💡 O E (&&) garante que está dentro da faixa

📌 Exemplo 3: Final de Semana (OU)

String dia = "sábado"; // É sábado OU domingo? boolean fimDeSemana = (dia == "sábado") || (dia == "domingo"); // true if (fimDeSemana) { System.out.println("Pode descansar!"); }

💡 O OU (||) aceita qualquer uma das opções

📌 Exemplo 4: Negação (NÃO)

boolean chovendo = false; // NÃO está chovendo? boolean podeSair = !chovendo; // true if (podeSair) { System.out.println("Pode sair sem guarda-chuva!"); }

💡 O NÃO (!) inverte: false vira true

📌 Exemplo 5: Combinação Complexa

int idade = 25; boolean temCNH = true; boolean temCarro = false; // Pode dirigir: (maior de 18 E tem CNH) E (tem carro OU pode alugar) boolean podeDirigir = (idade >= 18 && temCNH) && (temCarro || true); System.out.println(podeDirigir); // true

🎯 Precedência: Aritméticos + Relacionais + Lógicos

Ordem completa de execução

Quando misturamos operadores, há uma ordem de precedência que determina qual é executado primeiro
Prioridade Operadores Tipo Exemplo
1ª (Maior) ( ) Parênteses (x + 5) executa primeiro
2ª ! (NÃO) Lógico (unário) !verdadeiro = falso
3ª * / % Aritméticos 5 * 2 antes de +
4ª + - Aritméticos Soma e subtração
5ª > < >= <= Relacionais Comparações de magnitude
6ª == != Relacionais Igualdade e diferença
7ª && (E) Lógico Conjunção
8ª (Menor) || (OU) Lógico Disjunção

📝 Exemplo Passo a Passo

Expressão: 5 + 3 > 10 / 2 && !falso

→ Passo 1: !falso = verdadeiro (NÃO)

→ Passo 2: 10 / 2 = 5 (Divisão)

→ Passo 3: 5 + 3 = 8 (Adição)

→ Passo 4: 8 > 5 = verdadeiro (Relacional)

→ Passo 5: verdadeiro && verdadeiro = VERDADEIRO (E)

💡 Dica de Ouro

Na dúvida sobre precedência, USE PARÊNTESES! Eles tornam a expressão mais clara:

// Menos claro if (x > 5 && y < 10 || z == 0) { ... } // Mais claro if ((x > 5 && y < 10) || (z == 0)) { ... }

⚡ Avaliação em Curto-Circuito

Otimização inteligente

Operadores && e || param de avaliar assim que o resultado é conhecido

⚡ E (&&) - Curto-Circuito

Se a primeira condição for FALSA, a segunda nem é testada!

boolean resultado; // false && ??? = sempre false resultado = (false && qualquerCoisa); // qualquerCoisa NEM é avaliado! // Exemplo prático: int x = 5; if (x > 10 && x < 20) { // x < 20 NÃO é testado! // Porque x > 10 já é false }

⚡ OU (||) - Curto-Circuito

Se a primeira condição for VERDADEIRA, a segunda nem é testada!

boolean resultado; // true || ??? = sempre true resultado = (true || qualquerCoisa); // qualquerCoisa NEM é avaliado! // Exemplo prático: int x = 15; if (x > 10 || x < 5) { // x < 5 NÃO é testado! // Porque x > 10 já é true }

💡 Por que isso é útil?

1. Performance: Economiza processamento ao não avaliar condições desnecessárias

2. Segurança: Evita erros ao verificar condições que poderiam causar problemas

// Exemplo de segurança: int divisor = 0; // SEGURO: Verifica divisor != 0 ANTES de dividir if (divisor != 0 && 10 / divisor > 2) { // Se divisor == 0, a divisão NEM acontece! // Evita erro de divisão por zero }

🏃 Analogia: Corrida de Revezamento

E (&&): Equipe com 4 corredores. Se o primeiro cai, a equipe está eliminada. Não precisa esperar os outros correrem!

OU (||): Competição onde basta UM ganhar. Se o primeiro vence, não precisa esperar o resultado dos outros!
