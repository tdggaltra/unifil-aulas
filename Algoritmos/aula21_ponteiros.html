O Endereço É o Poder!

Um ponteiro é uma variável que armazena o endereço de memória de outra variável. Em vez de guardar um valor diretamente, ele aponta para onde esse valor está — permitindo modificar qualquer dado, navegar por estruturas e gerenciar memória dinamicamente.

42
← int x = 42

0x1000
← int *p = &x ↗ aponta para x!
x = 42  |  p = 0x1000  |  *p = 42 (desreferenciar)  |  &x = 0x1000 (endereço)

🏗️ O Que É um Ponteiro?

Variáveis têm endereço na memória — ponteiros armazenam esse endereço

Todo dado na memória tem um endereço. Ponteiro é uma variável que armazena esse endereço! 🗺️

🔵 C — declarar, inicializar e usar

// Declaração: tipo *nome int x = 42; int *p; ← ponteiro para int // & = operador endereço p = &x; ← p guarda o endereço de x // * = operador desreferência printf("%d\n", *p); ← lê o valor em p → 42 *p = 99; ← escreve em x via p! printf("%d\n", x); ← x agora vale 99! // Imprimir o próprio endereço printf("%p\n", p); ← ex: 0x7ffd1234 printf("%p\n", &x); ← mesmo valor! // Tamanho do ponteiro (sempre 8 bytes em 64-bit) printf("%zu\n", sizeof(p)); ← 8

🏠 Analogia: Endereço de Casa

A variável é a casa. O endereço &x é o número da rua. O ponteiro p é um papel onde você anota o endereço. Desreferenciar *p é ir até a casa e ver o que tem lá dentro.

➕ Aritmética de Ponteiros

Incrementar um ponteiro avança pelo tamanho do tipo — a base dos vetores!

p + 1 não soma 1 ao endereço — soma sizeof(tipo)! Avança para o próximo elemento. 🎯

🔵 C — aritmética com int*

int v[] = {10, 20, 30, 40, 50}; int *p = v; ← p aponta para v[0] printf("%d\n", *p); ← 10 (v[0]) printf("%d\n", *(p+1)); ← 20 (v[1]) printf("%d\n", *(p+4)); ← 50 (v[4]) p++; ← avança 4 bytes (sizeof int) printf("%d\n", *p); ← 20 (v[1]) // Percorrer com ponteiro: int *q = v; while(q < v + 5) { ← q < endereço após o array printf("%d ", *q); q++; } // 10 20 30 40 50 // v[i] é IGUAL a *(v+i) // v[2] == *(v+2) == 30 ✅

✅ Nome do array = ponteiro para [0]

Em C, o nome v é um ponteiro constante para o primeiro elemento. v[i] e *(v+i) são exatamente equivalentes. Por isso vetores são sempre passados por referência para funções!

⚙️ Ponteiros em Funções

Passar endereços permite modificar variáveis fora da função — passagem por referência real

🔵 C — funções clássicas com ponteiros

// Trocar dois valores (swap) void trocar(int *a, int *b) { int tmp = *a; *a = *b; *b = tmp; } int x=5, y=9; trocar(&x, &y); printf("%d %d\n", x, y); ← 9 5 ✅ // Retornar dois valores via ponteiros void minMax(int v[], int n, int *min, int *max) { *min = *max = v[0]; for(int i=1; i<n; i++) { if(v[i] < *min) *min = v[i]; if(v[i] > *max) *max = v[i]; } } int mn, mx; minMax(v, 5, &mn, &mx); printf("Min=%d Max=%d\n", mn, mx);

🔵 C — scanf usa ponteiros!

// scanf sempre recebe ponteiros int n; scanf("%d", &n); ← &n = ponteiro para n // Por dentro do scanf: void lerInteiro(int *destino) { // lê do teclado e armazena // em *destino (na variável original) } // Função que "retorna" dois resultados void dividir(int a, int b, int *quoc, int *resto) { *quoc = a / b; *resto = a % b; } int q, r; dividir(17, 5, &q, &r); printf("17÷5 = %d resto %d\n",q,r); // 17÷5 = 3 resto 2

✅ Por que usar ponteiros em funções?

  • Modificar a variável original (swap, scanf)
  • "Retornar" mais de um valor
  • Evitar cópia de estruturas grandes (eficiência)
  • Trabalhar com arrays (sempre por referência)

🛡️ Ponteiro NULL e Boas Práticas

NULL significa "não aponta para nada" — sempre verifique antes de desreferenciar!

🔵 C — NULL na prática

// NULL = ponteiro inválido / "vazio" int *p = NULL; ← inicialize com NULL! // Sempre verifique antes de usar! if(p != NULL) { printf("%d\n", *p); ← seguro! } else { printf("Ponteiro inválido!\n"); } // Padrão de busca com NULL int* buscar(int v[], int n, int alvo) { for(int i=0; i<n; i++) if(v[i] == alvo) return &v[i]; return NULL; ← não encontrou } int *res = buscar(v, 5, 30); if(res != NULL) printf("Achei: %d\n", *res); else printf("Não encontrado.\n");

⚠️ Ponteiro selvagem (dangling/wild pointer)

// ERRADO 1: sem inicializar (lixo!) int *p; ← contém endereço aleatório! *p = 42; ← CRASH ou corrupção! ❌ // ERRADO 2: desreferenciar NULL int *p = NULL; *p = 42; ← Segmentation Fault! ❌ // CORRETO: sempre inicializar int x = 0; int *p = &x; ← aponta para variável ✅ // ou int *p = NULL; ← inicializa como inválido ✅ // e verifique antes de usar!

🧠 Alocação Dinâmica de Memória

malloc e free — alocar memória em tempo de execução conforme necessário

malloc aloca na heap. free libera. Sem free = vazamento de memória. Sempre em par! 🔒

🔵 C — malloc, calloc, realloc, free

#include <stdlib.h> // MALLOC — aloca sem inicializar int *v = (int*)malloc(10 * sizeof(int)); if(v == NULL) { printf("Sem memória!\n"); return 1; } v[0] = 42; v[9] = 99; ← usa como array! free(v); ← obrigatório! libera memória v = NULL; ← boa prática após free // CALLOC — aloca e inicializa com zero int *u = (int*)calloc(10, sizeof(int)); // u[0]..u[9] = 0 garantido // REALLOC — redimensiona v = (int*)realloc(v, 20 * sizeof(int)); // agora v tem espaço para 20 ints // dados anteriores preservados!

int n; printf("Quantos elementos? "); scanf("%d", &n); // Tamanho decidido em tempo de execução! int *v = (int*)malloc(n * sizeof(int)); if(!v) { printf("Erro!\n"); return 1; } for(int i=0;i<n;i++) scanf("%d",&v[i]); // processar v[0]..v[n-1] free(v); v=NULL;

💻 Exemplo 1 — Vetor Dinâmico com Realloc

Crescer o vetor conforme necessário — padrão usado em listas profissionais

📌 Problema Intermediário

Criar um vetor que cresce automaticamente: começa com capacidade 4, dobra quando cheio. O usuário digita números (0 para parar) e o vetor acomoda todos sem limite fixo.

🔵 C — vetor com crescimento dinâmico

int *v = (int*)malloc(4 * sizeof(int)); int cap = 4; ← capacidade atual int n = 0; ← quantidade usada int num; printf("Digite (0 para parar):\n"); while(1) { scanf("%d", &num); if(num == 0) break; // Vetor cheio? Dobrar capacidade! if(n == cap) { cap *= 2; v = (int*)realloc(v, cap*sizeof(int)); if(!v) { printf("Erro!\n"); return 1; } printf("[expandiu para %d]\n", cap); } v[n++] = num; } printf("Guardei %d números:\n", n); for(int i=0; i<n; i++) printf("%d ", v[i]); free(v); v=NULL;

📈 Trace de capacidade

Elemento n cap Ação
1º, 2º, 3º, 4º1..44insere normalmente
5º elemento44→8realloc dobra!
9º elemento88→16realloc dobra!

✅ Por que dobrar a capacidade?

Dobrar a cada expansão garante complexidade amortizada O(1) por inserção. Se expandisse +1 a cada vez, seriam N reallocações para N elementos — O(N²). Dobrar reduz para O(log N) reallocações. É o mesmo princípio do ArrayList do Java!

🎭 Ponteiro para Função

Funções também têm endereço — ponteiros para função permitem comportamento dinâmico

Um ponteiro para função armazena o endereço de uma função — chame-a de forma dinâmica! 🎯

🔵 C — declarar e usar ponteiro para função

// Tipo: int (*nome)(int, int) int somar (int a, int b){ return a+b; } int subtrair(int a, int b){ return a-b; } int multiplicar(int a, int b){ return a*b; } // Ponteiro para função: int (*fp)(int,int) int (*fp)(int, int); fp = somar; printf("%d\n", fp(3,4)); ← 7 fp = subtrair; printf("%d\n", fp(10,3)); ← 7 // Vetor de ponteiros para função int (*ops[3])(int,int) = { somar, subtrair, multiplicar }; for(int i=0; i<3; i++) printf("%d\n", ops[i](6, 2)); // 8, 4, 12

🔵 C — qsort usa ponteiro para função!

// qsort recebe função de comparação int cmpCrescente(const void *a, const void *b) { return *((int*)a) - *((int*)b); } int cmpDecrescente(const void *a, const void *b) { return *((int*)b) - *((int*)a); } int v[] = {5,2,8,1,9}; // Ordena crescente: qsort(v, 5, sizeof(int), cmpCrescente); // {1, 2, 5, 8, 9} // Ordena decrescente (mesma função, // comparador diferente!): qsort(v, 5, sizeof(int), cmpDecrescente); // {9, 8, 5, 2, 1}

✅ Flexibilidade com ponteiros de função

Ponteiros para função permitem comportamento parametrizável: a mesma rotina de ordenação serve para crescente, decrescente, por nome, por data — só muda a função de comparação passada. É a base das callbacks e do padrão Strategy.

💻 Exemplo 2 — Lista Encadeada Simples

Cada nó aponta para o próximo — estrutura dinâmica que cresce sem limite

📌 Problema Avançado

Implementar uma lista encadeada de inteiros com funções: inserir no início, inserir no fim, remover e exibir.

🔵 C — definição e inserção

// Cada nó: dado + ponteiro para próximo typedef struct No { int valor; struct No *prox; ← aponta para o próximo! } No; // Inserir no início — O(1) No* inserirInicio(No *lista, int val) { No *novo = (No*)malloc(sizeof(No)); novo->valor = val; novo->prox = lista; ← novo aponta p/ antiga cabeça return novo; ← nova cabeça } // Inserir no fim — O(n) No* inserirFim(No *lista, int val) { No *novo = (No*)malloc(sizeof(No)); novo->valor = val; novo->prox = NULL; if(!lista) return novo; ← lista vazia No *cur = lista; while(cur->prox) cur=cur->prox; cur->prox = novo; return lista; }

🔵 C — exibir, remover e liberar

// Exibir a lista void exibir(No *lista) { No *cur = lista; while(cur) { printf("%d → ", cur->valor); cur = cur->prox; } printf("NULL\n"); } // Liberar toda a lista void liberar(No **lista) { No *cur = *lista; while(cur) { No *prox = cur->prox; free(cur); cur = prox; } *lista = NULL; } // Uso completo: No *L = NULL; L = inserirFim(L, 10); L = inserirFim(L, 20); L = inserirFim(L, 30); exibir(L); ← 10 → 20 → 30 → NULL liberar(&L);

🚨 Erros Comuns com Ponteiros

Os bugs mais perigosos em C — aprenda a evitá-los!

❌ Erro 1: Ponteiro não inicializado

// ERRADO: lixo na memória int *p; ← endereço aleatório! *p = 42; ← CRASH ou pior! ❌ // CORRETO: sempre inicializar int x = 0; int *p = &x; ← aponta para x ✅ // ou int *p = NULL; ← explicitamente vazio ✅

❌ Erro 2: Uso após free (use-after-free)

// ERRADO: usar após liberar int *p = (int*)malloc(sizeof(int)); *p = 42; free(p); printf("%d\n", *p); ← comportamento indefinido! ❌ // CORRETO: anular após free free(p); p = NULL; ← seguro ✅ if(p) *p = 1; ← não executa

❌ Erro 3: Memory leak (esquecer o free)

// ERRADO: malloc sem free void processarDados() { int *v = (int*)malloc(1000*sizeof(int)); // usa v... return; ← v NÃO é liberado! Leak! ❌ } // Chamada 1000x → 4MB perdidos! // CORRETO: free antes de retornar void processarDados() { int *v = (int*)malloc(1000*sizeof(int)); // usa v... free(v); v=NULL; return; ← ✅ }

❌ Erro 4: Retornar ponteiro para variável local

// ERRADO: local morre quando função retorna! int* criar() { int x = 42; return &x; ← x some da stack! ❌ } // CORRETO: alocar na heap int* criar() { int *p = (int*)malloc(sizeof(int)); *p = 42; return p; ← heap persiste! ✅ }

✅ Regras de ouro dos ponteiros

☑ Sempre inicializar (&var ou NULL)  |  ☑ Verificar != NULL antes de desreferenciar  |  ☑ Todo malloc tem um free correspondente  |  ☑ Após free, atribuir NULL  |  ☑ Nunca retornar endereço de variável local

Ponteiros — o coração do C e a porta para estruturas de dados avançadas!

Ponteiro = endereço. & pega o endereço. * acessa o conteúdo. NULL = vazio. free = obrigatório! 📍

✅ Conceitos aprendidos

  • &x — endereço de x na memória
  • *p — valor no endereço guardado em p
  • Aritmética: p+1 avança sizeof(tipo) bytes
  • malloc/free — alocar e liberar na heap
  • Ponteiro para função — comportamento parametrizável

⚠️ Erros a evitar

  • Ponteiro sem inicialização → crash
  • Desreferenciar NULL → Segmentation Fault
  • malloc sem free → memory leak
  • Retornar endereço de variável local
