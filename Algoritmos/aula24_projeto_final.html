🎯 Objetivos do Projeto Final

O que você vai demonstrar e por que este projeto importa para sua carreira

Este projeto é seu portfólio de entrada no mercado — um sistema real, seu, do zero! 💼

🎓 Objetivos de aprendizagem

  • Modelar dados reais com structs e enums bem projetados
  • Organizar o código em funções coesas (princípio DRY)
  • Implementar persistência de dados com arquivos binários
  • Realizar busca e ordenação em vetores de structs
  • Gerar relatórios em arquivo texto e CSV
  • Tratar entradas inválidas e erros de arquivo graciosamente
  • Escrever código legível: nomes descritivos, comentários úteis

🗺️ Requisitos mínimos obrigatórios

  • Ao menos 1 struct com mínimo de 5 campos de tipos variados
  • Vetor de structs com capacidade ≥ 50 registros
  • Menu do-while com mínimo 6 opções funcionais
  • CRUD completo: cadastrar, listar, buscar, editar, excluir
  • Persistência: carregar ao iniciar + salvar ao sair
  • Ordenação por ao menos um campo
  • Relatório exportado em arquivo texto ou CSV
  • Código compila sem erros com gcc -Wall

✅ Liberdade de tema

Você escolhe o domínio do sistema — biblioteca, farmácia, academia, estoque, clínica, escola, hotel, concessionária. O importante é que o tema seja relevante e motivador para você!

// Carregar ao iniciar (se arquivo existe) void carregar() { FILE *fp = fopen("dados.dat", "rb"); if(!fp) { printf("Novo cadastro.\n"); return; } fread(&total, sizeof(int), 1, fp); fread(dados, sizeof(NomeTipo), total, fp); fclose(fp); } // Salvar ao sair void salvar() { FILE *fp = fopen("dados.dat", "wb"); if(!fp) { perror("Erro"); return; } fwrite(&total, sizeof(int), 1, fp); fwrite(dados, sizeof(NomeTipo), total, fp); fclose(fp); }

🏗️ Arquitetura Sugerida do Projeto

Organização em camadas: dados → lógica → interface → persistência

Separe responsabilidades: dados, lógica de negócio, interface e arquivo — cada função faz uma coisa! 🧩

💡 Dica de ouro: escreva uma função por vez

Implemente carregar → salvar → cadastrar → listar primeiro. Teste cada uma. Só depois adicione buscar, editar, excluir. Código que funciona parcialmente é melhor que código completo que não compila!

// Excluir por índice — compactar vetor void excluir() { int id; printf("ID do livro: "); scanf("%d", &id); int pos = buscarPorId(id); if(pos == -1){puts("Não encontrado.");return;} printf("Excluir \"%s\"? (s/n) ", dados[pos].titulo); char conf; scanf(" %c", &conf); if(conf != 's' && conf != 'S') return; // Compactar: mover todos uma posição for(int i=pos; i<total-1; i++) dados[i] = dados[i+1]; total--; printf("✅ Livro removido!\n"); } // Buscar por ID — retorna índice int buscarPorId(int id) { for(int i=0; i<total; i++) if(dados[i].id == id) return i; return -1; }

void editar() { int id; printf("ID do livro a editar: "); scanf("%d", &id); int pos = buscarPorId(id); if(pos==-1){puts("Não encontrado.");return;} Livro *l = &dados[pos]; ← ponteiro direto! int op; do { printf("\n=== Editando: %s ===\n", l->titulo); printf("1. Título [%s]\n", l->titulo); printf("2. Autor [%s]\n", l->autor); printf("3. Ano [%d]\n", l->ano); printf("4. Status [%s]\n", statusStr(l->status)); printf("0. 🔵 Exportar relatório CSV completo

void exportarRelatorio() { FILE *fp = fopen("relatorio.csv", "w"); if(!fp){ perror("Erro"); return; } // Cabeçalho fprintf(fp, "ID,Titulo,Autor,Ano,Status\n"); int disp=0, emp=0; for(int i=0;i<total;i++){ fprintf(fp, "%d,\"%s\",\"%s\",%d,%s\n", dados[i].id, dados[i].titulo, dados[i].autor, dados[i].ano, statusStr(dados[i].status)); if(dados[i].status==DISPONIVEL) disp++; else emp++; } fprintf(fp,"\nTotal,%d,,," "\n",total); fprintf(fp,"Disponíveis,%d,,,\n",disp); fprintf(fp,"Emprestados,%d,,,\n",emp); fclose(fp); printf("📊 relatorio.csv gerado!\n"); }

✅ Editar via ponteiro

Livro *l = &dados[pos] cria um ponteiro para o registro no vetor. Modificar l->campo altera diretamente o dado original — sem precisar copiar e reescrever!

💡 Dicas Finais de Desenvolvimento

Boas práticas que separam um código bom de um código excelente

✅ Compile frequentemente

  • Compile após cada função nova
  • Use gcc -Wall -o prog main.c
  • Corrija warnings — são bugs esperando
  • Compile antes de dormir, nunca só na entrega

✅ Nomes que comunicam

  • buscarPorMatricula() não f2()
  • totalCadastrados não n
  • CAPACIDADE_MAX não 100 hardcoded
  • Comentário explica o porquê, não o quê

✅ Testar casos limite

  • O que acontece com 0 registros?
  • Buscar algo que não existe?
  • Cadastrar quando o vetor está cheio?
  • Fechar e reabrir 3 vezes seguidas?

🔵 Padrão de main() limpo

int main() { carregar(); ← sempre primeiro printf("Sistema iniciado. %d registro(s).\n", total); int opcao; do { exibirMenu(); scanf("%d", &opcao); switch(opcao) { case 1: cadastrar(); break; case 2: listar(); break; case 3: buscarInterativo(); break; case 4: editar(); break; case 5: excluir(); break; case 6: ordenarExibir(); break; case 7: exportarRelatorio();break; case 0: puts("Até logo!"); break; default:puts("Opção inválida."); } } while(opcao != 0); salvar(); ← sempre último return 0; }

⚠️ Erros que travam na última hora

  • Usar gets() em vez de fgets() — inseguro, não compila em alguns compiladores
  • Esquecer fclose() — dados somem silenciosamente
  • Comparar strings com == — bug difícil de detectar
  • Esquecer break no switch — fall-through inesperado
  • Abrir arquivo com "w" no lugar de "a" — apaga o histórico
