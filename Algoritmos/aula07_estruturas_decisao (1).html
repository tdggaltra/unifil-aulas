🤔 Por que precisamos de Decisões?

Programas reais precisam reagir a diferentes situações

Todo programa útil precisa tomar decisões: exibir mensagens diferentes, liberar ou bloquear ações, calcular valores distintos conforme a situação

🌍 Exemplos do Cotidiano

  • Se chover → levar guarda-chuva
  • Se nota ≥ 6 → aluno aprovado
  • Se saldo > 0 → liberar saque
  • Se usuário = admin → exibir painel
  • Se produto disponível → permitir compra

💻 Em Sistemas Reais

  • Validar login e senha do usuário
  • Calcular desconto por faixa de preço
  • Exibir mensagem de erro ou sucesso
  • Verificar CPF antes de cadastrar
  • Bloquear acesso a menores de idade

🚦 Analogia: Semáforo de Trânsito

Um semáforo toma decisões baseadas em condições: SE a luz estiver verde → carros passam. SE NÃO → carros param. Seu programa funciona da mesma forma — testando condições e escolhendo o caminho a seguir.

📌 Definição

Estruturas de Decisão permitem que o programa execute ou ignore blocos de instruções dependendo do resultado de uma expressão lógica (condição) — sempre verdadeira ou falsa.

🔹 Decisão Simples — SE (if)

Executa um bloco apenas quando a condição é verdadeira

Condição VERDADEIRA → bloco executado  |  Condição FALSA → bloco ignorado e o programa continua

📊 Fluxograma

INÍCIO
leia dados
condição?
✔ VERDADEIRO
Bloco SE
✖ FALSO
FIM

💡 Regra de ouro

Se a condição for falsa, o bloco é simplesmente ignorado. O programa continua normalmente após o fimse / }.

🟢 Portugol

se (condição) entao instrução_1 instrução_2 fimse

🔵 C

if (condição) { instrução_1; instrução_2; }

🟠 Java

if (condição) { instrução_1; instrução_2; }

💻 Exemplo Prático — Decisão Simples

Verificar se o aluno foi aprovado (nota ≥ 6)

🟢 Portugol

algoritmo "Aprovacao" var nota: real inicio escreva("Digite a nota: ") leia(nota) se (nota >= 6.0) entao escreva("Aprovado!") fimse escreva("Fim do programa.") fimalgoritmo

🔵 C

#include <stdio.h> int main() { float nota; printf("Digite a nota: "); scanf("%f", &nota); if (nota >= 6.0) { printf("Aprovado!\n"); } printf("Fim do programa.\n"); return 0; }

🟠 Java

import java.util.Scanner; public class Aprovacao { public static void main (String[] args) { Scanner sc = new Scanner(System.in); System.out.print( "Digite a nota: "); double nota = sc.nextDouble(); if (nota >= 6.0) { System.out.println( "Aprovado!"); } System.out.println( "Fim do programa."); } }

✅ Teste com nota = 8.0

8.0 >= 6.0 → VERDADEIRO
Saída: "Aprovado!" e "Fim do programa."

⚠️ Teste com nota = 4.5

4.5 >= 6.0 → FALSO
Saída: apenas "Fim do programa."

🔸 Decisão Composta — SE / SENÃO (if / else)

Sempre executa um dos dois blocos — nunca os dois, nunca nenhum

📊 Fluxograma

INÍCIO
condição?
✔ VERDADEIRO
Bloco SE
✖ FALSO
Bloco SENÃO
FIM

⚡ Diferença-chave

Na decisão composta o programa sempre executa um dos dois caminhos. Não existe "não executar nada".

🟢 Portugol

se (condição) entao bloco_verdadeiro senao bloco_falso fimse

🔵 C

if (condição) { bloco_verdadeiro; } else { bloco_falso; }

🟠 Java

if (condição) { bloco_verdadeiro; } else { bloco_falso; }

💡 Quando usar SE/SENÃO?

  • ✅ Aprovado ou Reprovado
  • ✅ Par ou Ímpar
  • ✅ Positivo ou Negativo
  • ✅ Sempre que há exatamente 2 caminhos

💻 Exemplo Prático — Decisão Composta

Aprovado ou Reprovado com mensagem personalizada

🟢 Portugol

algoritmo "AprovaRep" var nota: real inicio escreva("Nota: ") leia(nota) se (nota >= 6.0) entao escreva("APROVADO ✔") escreva("Parabéns!") senao escreva("REPROVADO ✖") escreva("Estude mais!") fimse fimalgoritmo

🔵 C

#include <stdio.h> int main() { float nota; printf("Nota: "); scanf("%f", &nota); if (nota >= 6.0f) { printf("APROVADO\n"); printf("Parabens!\n"); } else { printf("REPROVADO\n"); printf("Estude mais!\n"); } return 0; }

🟠 Java

import java.util.Scanner; public class AprovaRep { public static void main (String[] args) { Scanner sc = new Scanner(System.in); System.out.print( "Nota: "); double nota = sc.nextDouble(); if (nota >= 6.0) { System.out.println( "APROVADO ✔"); System.out.println( "Parabéns!"); } else { System.out.println( "REPROVADO ✖"); System.out.println( "Estude mais!"); } } }

🔍 Trace Mental

Nota 7.5: 7.5 >= 6.0 → VERDADEIRO → "APROVADO / Parabéns!"   |   Nota 3.0: 3.0 >= 6.0 → FALSO → "REPROVADO / Estude mais!"

🔗 SE Encadeado — if / else if / else

Múltiplas condições testadas em sequência

Quando há mais de dois caminhos, encadeamos vários senao se / else if. Apenas o primeiro bloco verdadeiro é executado.

🟢 Portugol

se (nota >= 9.0) entao escreva("Conceito A") senao se (nota >= 7.0) entao escreva("Conceito B") senao se (nota >= 5.0) entao escreva("Conceito C") senao escreva("Conceito D") fimse fimse fimse

🔵 C / 🟠 Java

if (nota >= 9.0) { printf("Conceito A\n"); } else if (nota >= 7.0) { printf("Conceito B\n"); } else if (nota >= 5.0) { printf("Conceito C\n"); } else { printf("Conceito D\n"); }
Nota Condição Verdadeira Saída
9.5nota >= 9.0 ✔Conceito A
8.0nota >= 7.0 ✔Conceito B
6.0nota >= 5.0 ✔Conceito C
3.0nenhuma ✖Conceito D

⚠️ A Ordem Importa!

  • → Condições testadas de cima para baixo
  • → 1ª condição verdadeira encerra o teste
  • → Sempre comece pelas mais restritivas
  • → Ordem errada gera resultados incorretos!

🎛️ Escolha-Caso — switch / case

Ideal para múltiplas opções baseadas em um valor exato

O switch compara uma variável com vários valores exatos e executa o bloco do caso que coincide

🟢 Portugol

escolha (variável) caso 1 escreva("Segunda-feira") caso 2 escreva("Terça-feira") caso 3 escreva("Quarta-feira") caso 4 escreva("Quinta-feira") caso 5 escreva("Sexta-feira") outrocaso escreva("Fim de semana") fimescolha

🔵 C / 🟠 Java

switch (variavel) { case 1: printf("Segunda-feira\n"); break; case 2: printf("Terca-feira\n"); break; case 3: printf("Quarta-feira\n"); break; case 4: printf("Quinta-feira\n"); break; case 5: printf("Sexta-feira\n"); break; default: printf("Fim de semana\n"); }

⚠️ O break é obrigatório (C/Java)!

Sem break, o switch executa os casos seguintes também — comportamento chamado de fall-through. No Portugol isso não ocorre.

🚫 Switch NÃO suporta

  • Comparações >, <, ≥, ≤
  • Tipos float / double
  • Expressões lógicas complexas

🧮 Exemplo Completo — Calculadora com Menu

Combinando switch e if-else: decisões aninhadas

🟢 Portugol

algoritmo "Calculadora" var a, b : real op : inteiro inicio escreva("1-Soma 2-Sub") escreva("3-Mult 4-Div: ") leia(op) escreva("Valor A: ") leia(a) escreva("Valor B: ") leia(b) escolha (op) caso 1: escreva("= ", a+b) caso 2: escreva("= ", a-b) caso 3: escreva("= ", a*b) caso 4: se (b <> 0) entao escreva("= ", a/b) senao escreva("Erro: div/0!") fimse outrocaso escreva("Opção inválida!") fimescolha fimalgoritmo

🔵 C

#include <stdio.h> int main() { float a, b; int op; printf("1-+ 2-- 3-* 4-/: "); scanf("%d", &op); printf("A: "); scanf("%f", &a); printf("B: "); scanf("%f", &b); switch(op) { case 1: printf("=%.2f\n",a+b); break; case 2: printf("=%.2f\n",a-b); break; case 3: printf("=%.2f\n",a*b); break; case 4: if(b != 0) printf("=%.2f\n",a/b); else printf("Div por zero!\n"); break; default: printf("Invalido!\n"); } return 0; }

🔍 Conceitos Combinados

  • ✅ switch gerencia o menu de operações
  • ✅ if/else protege a divisão por zero
  • ✅ Demonstra decisões aninhadas
  • ✅ Cada caminho tem sua resposta correta

⚖️ if-else vs switch — Quando usar cada um?

Comparação prática para escolher a estrutura certa

Critério if / else if / else switch / case
Tipo de condição ✅ Qualquer expressão lógica ⚠️ Apenas igualdade exata (==)
Tipos de dados ✅ Todos (int, float, bool…) ⚠️ int e char (sem float/double)
Comparações >, <, ≥, ≤ ✅ Sim ❌ Não
Faixas de valores ✅ Perfeito (nota ≥ 7.0) ❌ Não suportado
Muitas opções fixas ⚠️ Funciona, mas fica verboso ✅ Ideal — limpo e legível
Legibilidade com +5 casos ⚠️ Pode ficar confuso ✅ Muito mais claro

✅ Use if-else quando...

  • Condição envolve intervalos (nota >= 7.0)
  • Comparar tipos reais (float/double)
  • Lógica com && e ||
  • Poucas alternativas (2–3 caminhos)

✅ Use switch quando...

  • Menu de opções numeradas
  • Código de dia da semana ou mês
  • Muitos casos com valores exatos
  • Legibilidade é prioridade

🚨 Erros Comuns — Aprenda com os Equívocos!

Os principais erros ao usar estruturas de decisão

❌ Erro 1: = em vez de ==

// ERRADO (atribuição, não comparação!) if (nota = 7.0) ... ← bug silencioso // CORRETO if (nota == 7.0) ...

❌ Erro 2: Ponto e vírgula após if

// ERRADO: if sem efeito! if (nota >= 6.0); ← ; encerra o if! printf("Aprovado");← sempre executa // CORRETO if (nota >= 6.0) { printf("Aprovado"); }

❌ Erro 3: break ausente no switch

switch(op) { case 1: printf("Um\n"); ← sem break! case 2: printf("Dois\n"); ← executa! default:printf("Def\n"); ← e este! }

❌ Erro 4: Condições na ordem errada

// ERRADO: nota 9.5 cai em "Conceito C"! if (nota >= 5.0) printf("C"); else if (nota >= 7.0) printf("B"); else if (nota >= 9.0) printf("A"); // CORRETO: mais restritivo primeiro! if (nota >= 9.0) printf("A"); else if (nota >= 7.0) printf("B"); else if (nota >= 5.0) printf("C");

❌ Erro 5: Comparar float com ==

// PERIGOSO — imprecisão de ponto flutuante if (val == 0.1 + 0.2) ← pode falhar! // CORRETO — margem de tolerância if (fabs(val - 0.3) < 0.0001) ...

✅ Dica de Ouro

Sempre trace manualmente o algoritmo com pelo menos 3 valores de teste antes de programar. Isso evita a maioria dos erros lógicos!

✅ SE Simples (if)

  • Bloco executado só se verdadeiro
  • Bloco ignorado se falso
  • Portugol: se...entao...fimse
  • C/Java: if (...) { }

✅ SE / SENÃO (if-else)

  • Sempre executa um dos dois blocos
  • Verdadeiro → bloco SE
  • Falso → bloco SENÃO
  • Portugol: se...senao...fimse

✅ SE Encadeado (else if)

  • Múltiplas condições em sequência
  • Apenas o 1º bloco verdadeiro executa
  • Condições mais restritivas primeiro
  • Finalizar sempre com else

✅ ESCOLHA-CASO (switch)

  • Comparação por valores exatos
  • break obrigatório em C/Java
  • default para caso não encontrado
  • Ideal para menus de opções
