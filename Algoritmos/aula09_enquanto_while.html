🤔 Por que precisamos de Repetição?

O problema de escrever código repetido

Sem laços, imprimir 1 a 100 exigiria escrever 100 linhas de código. Com laços, resolvemos em 3 linhas!

😰 Sem Estrutura de Repetição

escreva(1) escreva(2) escreva(3) ... 97 linhas depois ... escreva(100) // 100 linhas! Impraticável 😱

😎 Com Laço ENQUANTO

var i: inteiro i <- 1 enquanto (i <= 100) faca escreva(i) i <- i + 1 fimenquanto // 6 linhas! Muito melhor 😎

🏭 Analogia: Linha de Montagem

Uma linha de montagem repete o mesmo processo para cada peça que passa: parafusar, pintar, embalar. Ela só para quando acabam as peças. O laço enquanto funciona da mesma forma — repete um bloco de instruções enquanto a condição (peças disponíveis) for verdadeira.

🌍 Exemplos do Cotidiano

  • Enquanto tiver fome → comer
  • Enquanto a fila avançar → esperar
  • Enquanto não acertar → tentar de novo
  • Enquanto houver emails → ler o próximo

💻 Em Sistemas Reais

  • Processar todos os registros de um banco
  • Repetir menu até o usuário sair
  • Validar senha até acertar
  • Somar vendas do mês inteiro

🟢 Portugol

// 1. Inicialização (antes do laço) i <- 1 enquanto (i <= 10) faca // 2. Condição escreva(i) i <- i + 1 // 3. Atualização fimenquanto

🔵 C

int i = 1; // 1. Inicialização while (i <= 10) { // 2. Condição printf("%d\n", i); i++; // 3. Atualização }

🟠 Java

int i = 1; // 1. Inicialização while (i <= 10) { // 2. Condição System.out.println(i); i++; // 3. Atualização }

⚡ Os 3 Elementos Obrigatórios de Todo Laço

  • 1️⃣ Inicialização: define o valor inicial da variável de controle (antes do laço)
  • 2️⃣ Condição: testada ANTES de cada execução — se falsa, o laço termina
  • 3️⃣ Atualização: modifica a variável de controle DENTRO do laço (evita loop infinito!)

🔍 Trace Manual — Entendendo a Execução

Acompanhe passo a passo o que acontece na memória

📋 Algoritmo analisado

i <- 1    enquanto (i <= 5) faca    escreva(i)    i <- i + 1    fimenquanto

Iteração Valor de i Condição i ≤ 5 Executa bloco? Saída i após atualização
Início 1 VERDADEIRO ✔ SIM 1 2
2ª 2 VERDADEIRO ✔ SIM 2 3
3ª 3 VERDADEIRO ✔ SIM 3 4
4ª 4 VERDADEIRO ✔ SIM 4 5
5ª 5 VERDADEIRO ✔ SIM 5 6
Fim 6 FALSO ✖ NÃO — Laço encerra

✅ O que acontece

  • Condição testada antes de cada iteração
  • Se verdadeira → executa bloco → atualiza → testa de novo
  • Se falsa → laço termina imediatamente
  • Se falsa já na 1ª vez → bloco nunca executa

⚠️ Loop Infinito!

Se esquecer a atualização (i <- i + 1), o valor de i nunca muda, a condição nunca fica falsa e o programa trava para sempre!

enquanto (i <= 5) faca escreva(i) ← sem i <- i+1! fimenquanto ← loop infinito 😱

🔢 Contador e Acumulador

As duas variáveis mais importantes dentro de laços

🔢 Contador

Variável que conta quantas iterações foram executadas ou quantos itens foram processados. Incrementada de 1 em 1 (ou outro valor fixo).

// Contar de 1 até N var i, n: inteiro inicio escreva("Até qual número? ") leia(n) i <- 1 enquanto (i <= n) faca escreva(i) i <- i + 1 ← contador fimenquanto

➕ Acumulador

Variável que acumula (soma, produto, concatena) valores ao longo das iterações. Começa em 0 para somas (ou 1 para produtos).

// Somar N números lidos var i, n, num, soma: inteiro inicio soma <- 0 ← acumulador (inicia 0) i <- 1 ← contador escreva("Quantos números? ") leia(n) enquanto (i <= n) faca escreva("Número: ") leia(num) soma <- soma + num ← acumula i <- i + 1 fimenquanto escreva("Soma: ", soma)

🎯 Regras de Ouro

  • ✅ Contador: inicialize antes do laço, incremente dentro do laço
  • ✅ Acumulador de soma: sempre inicialize com 0 antes do laço
  • ✅ Acumulador de produto: sempre inicialize com 1 antes do laço
  • ✅ Leia os valores dentro do laço, não fora

💻 Exemplo 1 — Tabuada de Multiplicação

Laço simples com contador — exibindo 10 resultados

📌 Problema

Leia um número inteiro e exiba sua tabuada completa (de 1 a 10).

🟢 Portugol

algoritmo "Tabuada" var n, i: inteiro inicio escreva("Tabuada de: ") leia(n) escreva("--- Tabuada de ", n, " ---") i <- 1 enquanto (i <= 10) faca escreva(n, " x ", i, " = ", n * i) i <- i + 1 fimenquanto fimalgoritmo

🔵 C

#include <stdio.h> int main() { int n, i; printf("Tabuada de: "); scanf("%d", &n); printf("--- Tabuada de %d ---\n", n); i = 1; while (i <= 10) { printf("%d x %d = %d\n", n, i, n * i); i++; } return 0; }

🟠 Java

import java.util.Scanner; public class Tabuada { public static void main (String[] args) { Scanner sc = new Scanner(System.in); System.out.print( "Tabuada de: "); int n = sc.nextInt(); System.out.println( "--- Tabuada de " + n + " ---"); int i = 1; while (i <= 10) { System.out.printf( "%d x %d = %d%n", n, i, n * i); i++; } } }

✅ Saída para n = 5

5 x 1 = 5  |  5 x 2 = 10  |  5 x 3 = 15  |  5 x 4 = 20  |  5 x 5 = 25  |  5 x 6 = 30  |  5 x 7 = 35  |  5 x 8 = 40  |  5 x 9 = 45  |  5 x 10 = 50

💻 Exemplo 2 — Soma e Média de N Números

Contador + Acumulador trabalhando juntos

📌 Problema

Leia a quantidade de notas (N), depois leia cada nota, calcule a soma total e a média. Informe se a turma foi aprovada (média ≥ 6).

🟢 Portugol

algoritmo "MediaTurma" var n, i : inteiro nota, soma : real media : real inicio soma <- 0 i <- 1 escreva("Qtd. de alunos: ") leia(n) enquanto (i <= n) faca escreva("Nota do aluno ", i, ": ") leia(nota) soma <- soma + nota i <- i + 1 fimenquanto media <- soma / n escreva("Soma : ", soma) escreva("Média : ", media:5:2) se (media >= 6.0) entao escreva("Turma APROVADA ✔") senao escreva("Turma REPROVADA ✖") fimse fimalgoritmo

🔵 C

#include <stdio.h> int main() { int n, i; float nota, soma = 0, media; printf("Qtd. de alunos: "); scanf("%d", &n); i = 1; while (i <= n) { printf("Nota %d: ", i); scanf("%f", &nota); soma += nota; i++; } media = soma / n; printf("Média: %.2f\n", media); if (media >= 6.0f) printf("APROVADA ✔\n"); else printf("REPROVADA ✖\n"); return 0; }

🔍 Trace para n=3, notas: 7, 5, 9

i=1: soma = 0+7 = 7  |  i=2: soma = 7+5 = 12  |  i=3: soma = 12+9 = 21
i=4: 4 ≤ 3 → FALSO → sai do laço
Média = 21/3 = 7.0 → APROVADA ✔

💻 Exemplo 3 — Laço com Sentinela

Quando não sabemos quantas repetições fazer: valor especial de parada

📌 Problema

Leia salários de funcionários e calcule a soma total. O usuário não sabe quantos funcionários há — ele digita -1 para encerrar a entrada. O valor -1 é o sentinela (valor de parada).

🟢 Portugol

algoritmo "Sentinela" var salario, total: real contador : inteiro inicio total <- 0 contador <- 0 escreva("Salário (-1 p/ sair): ") leia(salario) enquanto (salario <> -1) faca total <- total + salario contador <- contador + 1 escreva("Próximo (-1 p/ sair): ") leia(salario) fimenquanto escreva("Funcionários: ", contador) escreva("Total pago: R$ ", total:8:2) fimalgoritmo

🔵 C

#include <stdio.h> int main() { float sal, total = 0; int cont = 0; printf("Salário (-1 sair): "); scanf("%f", &sal); while (sal != -1) { total += sal; cont++; printf("Próximo (-1 sair): "); scanf("%f", &sal); } printf("Funcionários: %d\n", cont); printf("Total: R$ %.2f\n", total); return 0; }

🎯 Padrão com Sentinela

  • 1️⃣ Leia o primeiro valor antes do laço
  • 2️⃣ Condição testa se é o sentinela
  • 3️⃣ Processe o valor dentro do laço
  • 4️⃣ Leia o próximo valor no final do bloco

💻 Exemplo 4 — Validação de Entrada

Repetir até o usuário digitar um valor válido

📌 Problema

Peça ao usuário que digite uma nota entre 0 e 10. Enquanto o valor estiver fora desse intervalo, exiba uma mensagem de erro e solicite novamente. Só continue quando o valor for válido.

🟢 Portugol

algoritmo "ValidaNota" var nota: real inicio escreva("Nota (0 a 10): ") leia(nota) enquanto (nota < 0) ou (nota > 10) faca escreva("❌ Inválida!") escreva("Nota (0 a 10): ") leia(nota) fimenquanto escreva("✅ Nota aceita: ", nota) fimalgoritmo

🔵 C

#include <stdio.h> int main() { float nota; printf("Nota (0 a 10): "); scanf("%f", &nota); while (nota < 0 || nota > 10) { printf("Invalida! Tente: "); scanf("%f", &nota); } printf("Nota aceita: %.1f\n", nota); return 0; }

🟠 Java

import java.util.Scanner; public class ValidaNota { public static void main (String[] args) { Scanner sc = new Scanner(System.in); System.out.print( "Nota (0 a 10): "); double nota = sc.nextDouble(); while (nota < 0 || nota > 10) { System.out.print( "Inválida! Tente: "); nota = sc.nextDouble(); } System.out.println( "Nota aceita: " + nota); } }

💡 Esse padrão é chamado de "Loop de Validação"

É extremamente comum em sistemas reais: formulários web, aplicativos mobile, sistemas bancários — todos repetem a solicitação até o usuário fornecer dados válidos. A combinação enquanto + condição inválida garante que o programa só avance com dados corretos.

⚖️ while vs do-while — Qual a diferença?

Prévia da próxima aula: quando o bloco executa ao menos uma vez

while testa a condição ANTES → pode não executar nenhuma vez
do-while testa a condição DEPOIS → executa ao menos uma vez!

🔁 while — testa ANTES

// Se condição já for falsa, // o bloco NUNCA executa i <- 10 enquanto (i <= 5) faca escreva("Nunca aparece!") i <- i + 1 fimenquanto // Saída: (nada)

✅ Use while quando...

  • Pode ser que o bloco não precise executar
  • Você processa dados que podem estar vazios
  • Número de iterações pode ser zero

🔂 do-while — testa DEPOIS

// O bloco executa pelo menos // UMA VEZ antes de testar i <- 10 repita escreva("Aparece 1 vez!") i <- i + 1 ate (i > 5) // Saída: "Aparece 1 vez!"

✅ Use do-while quando...

  • Menu que deve aparecer ao menos uma vez
  • Pedir dados ao usuário ao menos uma vez
  • Processamento obrigatório inicial

🍕 Analogia

while = verificar se tem comida ANTES de ir ao restaurante. Se não tiver, você nem sai de casa.
do-while = ir ao restaurante PRIMEIRO e perguntar se tem comida ao chegar. Você sempre vai ao menos uma vez.

🚨 Erros Comuns com Laços while

Aprenda a identificar e evitar os problemas mais frequentes

❌ Erro 1: Loop Infinito — sem atualização

// ERRADO: i nunca muda! i <- 1 enquanto (i <= 10) faca escreva(i) ← sempre exibe 1 fimenquanto ← trava forever! // CORRETO: incrementar i i <- 1 enquanto (i <= 10) faca escreva(i) i <- i + 1 ← atualiza! fimenquanto

❌ Erro 2: Acumulador não inicializado

// ERRADO: soma tem lixo de memória var soma: inteiro ← valor imprevisível! enquanto ... faca soma <- soma + valor // CORRETO: sempre inicializar soma <- 0 ← antes do laço enquanto ... faca soma <- soma + valor

❌ Erro 3: Off-by-One (erro de um)

// ERRADO: imprime só até 9! i <- 1 enquanto (i < 10) faca ← < em vez de <= escreva(i) i <- i + 1 fimenquanto // CORRETO: usar <= para incluir 10 enquanto (i <= 10) faca

❌ Erro 4: Ler fora do laço

// ERRADO: lê só o 1º valor, repete ele leia(num) ← lê 1 vez apenas enquanto (num <> -1) faca soma <- soma + num ← num nunca muda! fimenquanto // CORRETO: ler dentro do laço leia(num) enquanto (num <> -1) faca soma <- soma + num leia(num) ← lê próximo! fimenquanto

✅ Checklist antes de entregar

☑ Inicializei a variável de controle?  |  ☑ A condição está correta (< ou <=)?  |  ☑ Atualizei a variável no final do bloco?  |  ☑ Inicializei o acumulador com 0?  |  ☑ Testei com casos extremos (0 repetições, 1 repetição)?
