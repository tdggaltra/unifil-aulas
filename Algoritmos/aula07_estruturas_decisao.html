🤔 Por que tomar decisões?

Programas reais precisam reagir a diferentes situações

Na vida real, tomamos decisões o tempo todo: "Se chover, levo guarda-chuva; caso contrário, não levo." Programas precisam da mesma capacidade.

🌍 Exemplos do Cotidiano

  • Se nota ≥ 6 → aprovado
  • Se saldo > 0 → saque liberado
  • Se usuário = admin → exibe painel
  • Se produto em estoque → permite compra

💻 Em Programação

  • Verificar validade de dados
  • Controlar acesso ao sistema
  • Calcular descontos por faixa
  • Exibir mensagens personalizadas

📌 Definição Formal

Estruturas de Decisão (ou Estruturas de Seleção) permitem que o programa execute ou ignore um bloco de instruções dependendo do resultado de uma expressão lógica (condição).

🔹 Decisão Simples — SE (if)

Executa um bloco apenas quando a condição é verdadeira

Fluxograma

INÍCIO
condição?
VERDADEIRO ✔
Bloco SE

Portugol
se (condição) entao instrução_1 instrução_2 fimse
C
if (condição) { instrução_1; instrução_2; }
Java
if (condição) { instrução_1; instrução_2; }

💡 Regra de Ouro

Se a condição for verdadeira, o bloco é executado. Se for falsa, o bloco é simplesmente ignorado e o programa continua normalmente após o fimse / }.

💻 Exemplo — Decisão Simples

Verificar se um aluno foi aprovado (nota ≥ 6)

Portugol
algoritmo "Aprovacao" var nota: real inicio escreva("Digite a nota: ") leia(nota) se (nota >= 6.0) entao escreva("Aprovado!") fimse escreva("Fim.") fimalgoritmo
C
#include <stdio.h> int main() { float nota; printf("Digite a nota: "); scanf("%f", &nota); if (nota >= 6.0) { printf("Aprovado!\n"); } printf("Fim.\n"); return 0; }
Java
import java.util.Scanner; public class Aprovacao { public static void main (String[] args) { Scanner sc = new Scanner(System.in); System.out.print( "Digite a nota: "); double nota = sc.nextDouble(); if (nota >= 6.0) { System.out.println( "Aprovado!"); } System.out.println("Fim."); } }

✅ Entrada: nota = 8.0

Condição 8.0 >= 6.0 → VERDADEIRO
Saída: "Aprovado!"   "Fim."

⚠️ Entrada: nota = 4.5

Condição 4.5 >= 6.0 → FALSO
Saída: apenas "Fim."

se (condição) entao bloco_verdadeiro senao bloco_falso fimse

⚡ Diferença-chave

Na decisão composta o programa sempre executa um dos dois caminhos — não existe a possibilidade de "não executar nada".

💻 Exemplo — Decisão Composta

Aprovado ou Reprovado com mensagem de incentivo

Portugol
algoritmo "AprovaRep" var nota: real inicio escreva("Nota: ") leia(nota) se (nota >= 6.0) entao escreva("APROVADO ✔") escreva("Parabéns!") senao escreva("REPROVADO ✖") escreva("Estude mais!") fimse fimalgoritmo
C
#include <stdio.h> int main() { float nota; printf("Nota: "); scanf("%f", &nota); if (nota >= 6.0f) { printf("APROVADO\n"); printf("Parabéns!\n"); } else { printf("REPROVADO\n"); printf("Estude mais!\n"); } return 0; }
Java
import java.util.Scanner; public class AprovaRep { public static void main (String[] args) { Scanner sc = new Scanner(System.in); System.out.print("Nota: "); double nota=sc.nextDouble(); if (nota >= 6.0) { System.out.println( "APROVADO ✔"); System.out.println( "Parabéns!"); } else { System.out.println( "REPROVADO ✖"); System.out.println( "Estude mais!"); } } }
💡 Trace mental: Nota 7.5 → 7.5 ≥ 6.0 = true → executa bloco SE (APROVADO / Parabéns).  |  Nota 3.0 → 3.0 ≥ 6.0 = false → executa SENÃO (REPROVADO / Estude mais).

🔗 SE Encadeado — if / else if / else

Múltiplas condições testadas em sequência

Quando existem mais de dois caminhos possíveis, encadeamos vários senao se / else if. Apenas o primeiro bloco cuja condição for verdadeira é executado.
Portugol
se (nota >= 9.0) entao escreva("Conceito A") senao se (nota >= 7.0) entao escreva("Conceito B") senao se (nota >= 5.0) entao escreva("Conceito C") senao escreva("Conceito D") fimse fimse fimse
C / Java
if (nota >= 9.0) { printf("Conceito A\n"); } else if (nota >= 7.0) { printf("Conceito B\n"); } else if (nota >= 5.0) { printf("Conceito C\n"); } else { printf("Conceito D\n"); }
NotaCondição VerdadeiraSaída
9.5nota ≥ 9.0 ✔Conceito A
8.0nota ≥ 7.0 ✔Conceito B
6.0nota ≥ 5.0 ✔Conceito C
3.0nenhuma ✖Conceito D

⚠️ Atenção: Ordem Importa!

  • Condições são testadas de cima para baixo.
  • Assim que uma for verdadeira, as restantes não são testadas.
  • Sempre comece pelas condições mais restritivas.

🎛️ Escolha-Caso — switch / case

Ideal para múltiplas opções baseadas em um valor exato

Portugol
escolha (variável) caso 1 escreva("Segunda-feira") caso 2 escreva("Terça-feira") caso 3 escreva("Quarta-feira") outrocaso escreva("Dia inválido") fimescolha
C
switch (variavel) { case 1: printf("Segunda-feira\n"); break; case 2: printf("Terça-feira\n"); break; case 3: printf("Quarta-feira\n"); break; default: printf("Dia inválido\n"); }
Java
switch (variavel) { case 1: System.out.println( "Segunda-feira"); break; case 2: System.out.println( "Terça-feira"); break; default: System.out.println( "Dia inválido"); }

✅ Quando usar switch?

  • Comparação com valores exatos (inteiros, chars)
  • Muitos casos com o mesmo tipo
  • Menus de opções numeradas
  • Código mais legível que muitos else if

⚠️ O break é obrigatório (C/Java)!

Sem break, o switch executa os casos seguintes também — comportamento chamado de fall-through. No Portugol isso não acontece.

🚫 Switch NÃO suporta

  • Comparações com >, <, ≥, ≤
  • Expressões lógicas complexas
  • Valores de ponto flutuante (em C/Java)

🧮 Exemplo Completo — Calculadora com Menu

Combinando switch, if-else e leitura de dados

Portugol
algoritmo "Calculadora" var a, b: real op: inteiro inicio escreva("1-Soma 2-Sub 3-Mult 4-Div: ") leia(op) escreva("A: ") leia(a) escreva("B: ") leia(b) escolha (op) caso 1: escreva("= ", a+b) caso 2: escreva("= ", a-b) caso 3: escreva("= ", a*b) caso 4: se (b <> 0) entao escreva("= ", a/b) senao escreva("Div/0!") fimse outrocaso escreva("Opção inválida") fimescolha fimalgoritmo
C
#include <stdio.h> int main() { float a, b; int op; printf("1-+ 2-- 3-* 4-/: "); scanf("%d", &op); printf("A: "); scanf("%f",&a); printf("B: "); scanf("%f",&b); switch(op) { case 1: printf("%.2f\n",a+b);break; case 2: printf("%.2f\n",a-b);break; case 3: printf("%.2f\n",a*b);break; case 4: if(b != 0) printf("%.2f\n",a/b); else printf("Div/0!\n"); break; default: printf("Inválido\n"); } return 0; }

🔍 Conceitos Combinados

  • switch controla o menu de operações
  • if/else protege a divisão por zero
  • Estruturas de decisão aninhadas

⚖️ if-else vs switch — Quando usar cada um?

Comparação prática para escolher a estrutura certa

Critério if / else if / else switch / case
Tipo de condição ✅ Qualquer expressão lógica ⚠️ Apenas igualdade exata
Tipos de dados ✅ Todos os tipos ⚠️ int, char (não float/double)
Comparações (>, <, ≥, ≤) ✅ Sim ❌ Não
Múltiplas opções fixas ⚠️ Funciona, mas fica verboso ✅ Ideal — código mais limpo
Legibilidade com >5 opções ⚠️ Pode ficar confuso ✅ Muito mais claro
Faixas de valores ✅ Perfeito (nota ≥ 7.0) ❌ Não suportado

✅ Use if-else quando...

  • Condição envolve intervalos (nota >= 7.0)
  • Comparação com tipos reais (float/double)
  • Lógica complexa com && e ||
  • Poucas alternativas (2 a 3 caminhos)

✅ Use switch quando...

  • Menu de opções com inteiros ou chars
  • Muitos casos com valores fixos exatos
  • Código de dia da semana, mês, opção de menu
  • Legibilidade é prioridade

🚨 Erros Comuns — Aprenda com os Erros!

Os principais equívocos ao usar estruturas de decisão

❌ Erro 1: = em vez de ==

// ERRADO (atribuição, não comparação) if (nota = 7.0) ... // CERTO (comparação de igualdade) if (nota == 7.0) ...

❌ Erro 2: Ponto e vírgula após if

// ERRADO: if sem efeito! if (nota >= 6.0); ← ; aqui termina o if printf("Aprovado\n"); ← sempre executa // CERTO if (nota >= 6.0) { printf("Aprovado\n"); }

❌ Erro 3: break ausente no switch

switch(op) { case 1: printf("Um\n"); ← sem break! case 2: printf("Dois\n"); ← também executa! default: printf("Def\n"); ← e este também! }

❌ Erro 4: Condições na ordem errada

// ERRADO: nota 9.5 vai cair em B! if (nota >= 5.0) printf("C"); else if (nota >= 7.0) printf("B"); else if (nota >= 9.0) printf("A"); // CERTO: mais restritivo primeiro! if (nota >= 9.0) printf("A"); else if (nota >= 7.0) printf("B"); else if (nota >= 5.0) printf("C");

❌ Erro 5: Comparar float com ==

// PERIGOSO — imprecisão de ponto flutuante if (preco == 0.1 + 0.2) ... ← pode falhar! // CORRETO — usar diferença pequena (epsilon) if (fabs(preco - 0.3) < 0.0001) ...

✅ Dica de Ouro

Sempre trace manualmente seus algoritmos com valores de teste antes de programar. Isso evita a maioria dos erros lógicos!

📋 Resumo da Aula 07

Tudo que você aprendeu hoje sobre Estruturas de Decisão

🔹

SE Simples (if)

Executa um bloco somente se a condição for verdadeira. Se falsa, o bloco é ignorado.

🔸

SE / SENÃO (if-else)

Sempre executa um dos dois blocos. Nunca ignora os dois e nunca executa os dois.

🔗

SE Encadeado (else if)

Múltiplas condições em sequência. Apenas o primeiro bloco verdadeiro é executado.

🎛️

ESCOLHA-CASO (switch)

Ideal para menus e comparações de igualdade com inteiros ou chars. Requer break em C/Java.

⚖️

if vs switch

Use if para intervalos e lógica complexa. Use switch para valores exatos e menus.

🚨

Erros Comuns

= vs ==, ponto-vírgula após if, break ausente, ordem das condições.
