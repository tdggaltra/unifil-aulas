✏️ Exercício 1 — Sequência de Fibonacci

for   N termos conhecidos → laço for com dois acumuladores

📌 Enunciado Básico

Leia um número N e exiba os N primeiros termos da sequência de Fibonacci. Cada termo é a soma dos dois anteriores: 0, 1, 1, 2, 3, 5, 8, 13, 21…

🔍 Trace para N = 7

a=0, b=1: exibe 0 → próximo=1  |  exibe 1 → próximo=1  |  exibe 1 → próximo=2  |  exibe 2 → próximo=3  |  exibe 3 → próximo=5  |  exibe 5 → próximo=8  |  exibe 8 → fim
Saída: 0 1 1 2 3 5 8

🟢 Portugol

algoritmo "Fibonacci" var n, i : inteiro a, b, prox: inteiro inicio escreva("Qtd. termos: ") leia(n) a <- 0 b <- 1 para i <- 1 ate n faca escreva(a, " ") prox <- a + b a <- b b <- prox fimpara fimalgoritmo

🔵 C

#include <stdio.h> int main() { int n, a = 0, b = 1, prox; printf("Qtd. termos: "); scanf("%d", &n); for(int i = 1; i <= n; i++){ printf("%d ", a); prox = a + b; a = b; b = prox; } printf("\n"); return 0; }

🟠 Java

import java.util.Scanner; public class Fibonacci { public static void main (String[] args) { Scanner sc = new Scanner(System.in); System.out.print( "Qtd. termos: "); int n = sc.nextInt(); int a=0, b=1, prox; for(int i=1; i<=n; i++){ System.out.print(a+" "); prox = a + b; a = b; b = prox; } } }

✅ Por que FOR aqui?

Sabemos exatamente N termos — o número de iterações é determinado no início. O for traduz isso diretamente: "para i de 1 até N, faça".

✏️ Exercício 2 — Números Primos até N

for aninhado — verificar divisores de cada número

📌 Enunciado Intermediário

Leia um número N e exiba todos os números primos de 2 até N. Um número é primo quando só é divisível por 1 e por ele mesmo.

🟢 Portugol

algoritmo "Primos" var n, i, j: inteiro primo : logico inicio escreva("Até N: ") leia(n) escreva("Primos:") para i <- 2 ate n faca primo <- verdadeiro para j <- 2 ate i-1 faca se (i % j == 0) entao primo <- falso fimse fimpara se (primo) entao escreva(i, " ") fimse fimpara fimalgoritmo

🔵 C (com otimização √n)

#include <stdio.h> #include <math.h> int main() { int n; printf("Até N: "); scanf("%d", &n); printf("Primos: "); for(int i=2; i<=n; i++) { int primo = 1; // Testar até √i (otimização) for(int j=2; j<=sqrt(i); j++) if(i % j == 0){ primo=0; break; } if(primo) printf("%d ", i); } printf("\n"); return 0; }

🔑 Lógica da Flag primo

Começa como verdadeiro. Se encontrar algum divisor → vira falso. Ao final do for interno, se ainda for verdadeiro → o número é primo. Esse padrão de flag booleana é muito comum em programação.

✏️ Exercício 3 — Relatório de Vendas com Sentinela

while   Número de entradas desconhecido → sentinela -1

📌 Enunciado Intermediário

Leia valores de vendas de um vendedor (digite -1 para encerrar). Calcule: total vendido, quantidade de vendas, maior venda, menor venda e média. Classifique o desempenho: Excelente (média ≥ 5000), Bom (≥ 3000), Regular (abaixo).

🟢 Portugol

algoritmo "Vendas" var venda, total, maior, menor: real qtd : inteiro inicio total <- 0 qtd <- 0 escreva("1ª venda (R$): ") leia(venda) maior <- venda menor <- venda enquanto (venda <> -1) faca total <- total + venda qtd <- qtd + 1 se venda > maior entao maior <- venda fimse se venda < menor entao menor <- venda fimse escreva("Próxima (-1 sair): ") leia(venda) fimenquanto se (qtd > 0) entao escreva("Total : R$ ", total:8:2) escreva("Qtd. : ", qtd) escreva("Média : R$ ", total/qtd:8:2) escreva("Maior : R$ ", maior:8:2) escreva("Menor : R$ ", menor:8:2) se (total/qtd >= 5000) entao escreva("🏆 Excelente!") senao se (total/qtd >= 3000) entao escreva("👍 Bom!") senao escreva("📈 Regular") fimse fimse senao escreva("Nenhuma venda registrada.") fimse fimalgoritmo

✅ Por que WHILE aqui?

  • Não sabemos quantas vendas haverá
  • Pode ser que não haja venda nenhuma
  • Parada por sentinela (-1) = condição desconhecida
  • O while verifica antes → se não tiver vendas, não entra

⚠️ Atenção: divisão por zero

Sempre verificar se (qtd > 0) antes de calcular a média (total / qtd). Se o usuário digitar -1 na primeira entrada, qtd = 0 e a divisão causaria erro fatal!

🔍 Exemplo de execução

Entradas: 4200, 6800, 3100, -1
Total=14100 | Qtd=3 | Média=4700
Maior=6800 | Menor=3100
Desempenho: 👍 Bom!

✏️ Exercício 4 — Adivinhe com Ranking

do-while   Jogo que classifica o jogador por número de tentativas

📌 Enunciado Intermediário

O programa escolhe um número entre 1 e 50 (fixo: 37). O usuário tem até 7 tentativas para adivinhar. Ao final, exiba um ranking: ≤ 3 tentativas = Mestre, ≤ 5 = Bom, > 5 = Iniciante.

🟢 Portugol

algoritmo "Adivinhe" var secreto, chute, tent: inteiro inicio secreto <- 37 tent <- 0 escreva("=== ADIVINHE (1-50) ===") repita tent <- tent + 1 escreva("Tent. ", tent, "/7: ") leia(chute) se (chute < secreto) entao escreva("📈 Maior!") senao se (chute > secreto) entao escreva("📉 Menor!") fimse fimse ate (chute == secreto) ou (tent == 7) se (chute == secreto) entao escreva("🎉 Acertou em ", tent, " tent.!") se (tent <= 3) entao escreva("🏆 Ranking: MESTRE!") senao se (tent <= 5) entao escreva("👍 Ranking: BOM!") senao escreva("🌱 Ranking: INICIANTE") fimse fimse senao escreva("💀 Game Over! Era ", secreto) fimse fimalgoritmo

✅ Por que DO-WHILE aqui?

  • O jogador sempre faz ao menos um chute
  • Impossível testar se acertou antes de tentar
  • Bloco deve executar → depois verificar condição de parada
  • Padrão clássico de jogos interativos

🏆 Tabela de Ranking

1–3 tentativas: 🏆 MESTRE — lógica binária perfeita
4–5 tentativas: 👍 BOM — boa estratégia
6–7 tentativas: 🌱 INICIANTE — continue praticando
7 e não acertou: 💀 Game Over

✏️ Exercício 5 — Boletim Completo da Turma

for + if/else if — relatório com conceitos e estatísticas

📌 Enunciado Avançado

Leia N alunos (nome + nota). Para cada aluno, calcule o conceito (A/B/C/D/F). Ao final: exiba aprovados, reprovados, maior nota, menor nota e média da turma.

🟢 Portugol

algoritmo "Boletim" var n, i, aprov, reprov: inteiro nota, soma, maior, menor: real nome: caractere inicio escreva("Qtd. alunos: ") leia(n) soma<-0 aprov<-0 reprov<-0 escreva("Nome 1: ") leia(nome) escreva("Nota 1: ") leia(nota) maior<-nota menor<-nota para i <- 1 ate n faca se i > 1 entao escreva("Nome: ") leia(nome) escreva("Nota: ") leia(nota) fimse soma <- soma + nota se nota > maior entao maior<-nota fimse se nota < menor entao menor<-nota fimse se nota >= 9.0 entao escreva(nome, ": A - Aprovado ✔") aprov<-aprov+1 senao se nota >= 7.0 entao escreva(nome, ": B - Aprovado ✔") aprov<-aprov+1 senao se nota >= 5.0 entao escreva(nome, ": C - Aprovado ✔") aprov<-aprov+1 senao se nota >= 3.0 entao escreva(nome, ": D - Reprovado ✖") reprov<-reprov+1 senao escreva(nome, ": F - Reprovado ✖") reprov<-reprov+1 fimse fimse fimse fimse fimpara escreva("─────────────────────") escreva("Aprovados : ", aprov) escreva("Reprovados: ", reprov) escreva("Média turma: ", soma/n:5:2) escreva("Maior: ", maior, " | Menor: ", menor) fimalgoritmo
NotaConceitoSituação
9,0 – 10,0A✅ Aprovado
7,0 – 8,9B✅ Aprovado
5,0 – 6,9C✅ Aprovado
3,0 – 4,9D❌ Reprovado
0,0 – 2,9F❌ Reprovado

✅ Estruturas combinadas

  • for — N alunos (número conhecido)
  • if/else if — conceito e aprovação
  • Acumuladores — soma, aprovados, reprovados
  • Comparação iterativa — maior e menor nota

🏗️ Mini-Projeto — Caixa de Supermercado

Integrando os 3 laços em um sistema completo de vendas

📌 Especificação Desafio

Simule um caixa de supermercado que:

  • Exibe um menu (do-while): 1-Nova compra, 2-Relatório do dia, 0-Fechar caixa
  • Em "Nova compra": lê produtos com preço e quantidade até sentinela 0 (while)
  • Calcula subtotal de cada produto e total da compra
  • Aplica desconto automático: compra ≥ R$ 200 = 10%, ≥ R$ 100 = 5%
  • No "Relatório do dia": exibe total de compras e faturamento acumulado (for)

🧱 Estrutura do Programa

  • do-while → loop principal do menu
  • while → leitura de produtos por sentinela
  • for → exibição do histórico no relatório
  • if/else if → cálculo do desconto
  • Acumuladores → total_dia, num_compras

🧪 Casos de Teste

  • 1 produto, R$50: sem desconto → R$50,00
  • 3 produtos, R$120 total: 5% desc → R$114,00
  • 5 produtos, R$250 total: 10% desc → R$225,00
  • Relatório após 3 compras: exibir total do dia

🔑 Mapa de laços do projeto

do-while Menu principal (ao menos 1 interação)  →  while Produtos da compra (sentinela 0)  →  for Histórico no relatório (N compras)

💻 Mini-Projeto — Código em Portugol

Caixa de Supermercado: menu + leitura de produtos + relatório

🟢 Parte 1: Menu e Loop de Compra

algoritmo "Supermercado" var op, num_compras, i : inteiro preco, qtd, total_comp : real total_dia, faturamento : real desconto : real historico: vetor[1..50] de real inicio num_compras <- 0 faturamento <- 0 repita ← do-while: menu escreva("=== CAIXA ===") escreva("1 - Nova compra") escreva("2 - Relatório") escreva("0 - Fechar") escreva("Opção: ") leia(op) se (op == 1) entao total_comp <- 0 escreva("Preço (0 sair): ") leia(preco) enquanto (preco <> 0) faca ← while: produtos escreva("Qtd.: ") leia(qtd) total_comp <- total_comp + (preco * qtd) escreva("Subtotal: R$ ", preco*qtd:7:2) escreva("Próximo (0 sair): ") leia(preco) fimenquanto // Aplica desconto se total_comp >= 200 entao desconto <- total_comp * 0.10 escreva("Desconto 10%: -R$ ", desconto:7:2) senao se total_comp >= 100 entao desconto <- total_comp * 0.05 escreva("Desconto 5%: -R$ ", desconto:7:2) senao desconto <- 0 fimse fimse total_comp <- total_comp - desconto escreva("Total: R$ ", total_comp:8:2) num_compras <- num_compras + 1 faturamento <- faturamento + total_comp historico[num_compras] <- total_comp fimse ate (op == 0) ← fim do do-while fimalgoritmo

🟢 Parte 2: Relatório com FOR

// Dentro do bloco op == 2: se (op == 2) entao escreva("=== RELATÓRIO DO DIA ===") escreva("Compras realizadas: ", num_compras) para i <- 1 ate num_compras faca escreva(" Compra #", i, ": R$ ", historico[i]:8:2) fimpara escreva("──────────────────────") escreva("Faturamento total:") escreva("R$ ", faturamento:10:2) fimse

✅ Os 3 laços em ação

  • do-while → menu sempre aparece
  • while → produtos até digitar 0
  • for → histórico de N compras

⚖️ Quando usar cada laço — Guia Final

Consolidando a escolha do laço certo para cada problema

A escolha do laço correto é sinal de um programador que pensa antes de codificar! 🧠
Pergunta-chaveRespostaLaço ideal
Quantas vezes vou repetir?Sei exatamente: N vezesfor para
Quantas vezes vou repetir?Não sei, pode ser 0while enquanto
O bloco precisa rodar ao menos uma vez?Sim, sempredo-while repita
Como vou parar?Sentinela (valor especial)while enquanto
Como vou parar?Opção "0 – Sair" em um menudo-while repita
Percorro uma sequência numérica?1 a N, pares, múltiplos…for para
Valido entrada do usuário?Repetir até dado válidodo-while repita

🧠 Dica de Ouro: pergunte antes de codificar

"Eu sei quantas vezes?" → SIM: for
"Pode ser zero vezes?" → SIM: while
"Deve executar ao menos 1 vez?" → SIM: do-while

✅ Lembre-se: são equivalentes!

Tecnicamente, qualquer for pode ser escrito como while e vice-versa. A escolha é sobre clareza e intenção: use o laço que torna o código mais legível e expressivo.
