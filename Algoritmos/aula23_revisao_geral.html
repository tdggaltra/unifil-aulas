⚠️ Armadilhas clássicas

if(x = 5) — atribuição dentro de if (sempre true)! Use if(x == 5).

switch sem break → fall-through (executa todos os casos abaixo)!

🔁 Revisão — Estruturas de Repetição (Aulas 08–13)

while, do-while, for, break, continue — quando usar cada um

🔵 while — teste no início

// Pode não executar nenhuma vez while(condicao) { // corpo } // Sentinela clássico: int v, soma=0, n=0; scanf("%d",&v); while(v != -1) { soma += v; n++; scanf("%d",&v); }

🔵 do-while — teste no fim

// Executa ao menos UMA vez do { // corpo } while(condicao); // Menu sempre exibe ao menos 1x: int op; do { exibirMenu(); scanf("%d",&op); executar(op); } while(op != 0);

🔵 for — contador explícito

// init; condição; incremento for(int i=0; i<n; i++) { // corpo } // Decrescente: for(int i=n-1; i>=0; i--) // Passo 2: for(int i=0; i<n; i+=2)

🔵 break e continue

// break — sai do laço imediatamente for(int i=0; i<n; i++) if(v[i] == alvo) { pos=i; break; } // continue — pula para próxima iteração for(int i=0; i<n; i++) { if(v[i] < 0) continue; ← pula negativos soma += v[i]; }
LaçoQuando usar
whileQtd de repetições desconhecida, pode ser 0
do-whileMenu, validação — garante ao menos 1 execução
forQtd conhecida, contador claro e explícito
breakSair cedo (busca encontrada, condição excepcional)
continuePular item inválido sem sair do laço

📦 Revisão — Vetores e Matrizes (Aulas 14–16)

Arrays unidimensionais e bidimensionais — estruturas fundamentais de dados

🔵 Vetores — operações essenciais

int v[5] = {4,2,7,1,9}; // Percorrer for(int i=0; i<5; i++) printf("%d ",v[i]); // Maior/menor int max=v[0]; for(int i=1;i<5;i++) if(v[i]>max) max=v[i]; // Busca linear — O(n) int pos=-1; for(int i=0;i<5;i++) if(v[i]==alvo){pos=i;break;} // Bubble Sort — O(n²) for(int i=0;i<4;i++) for(int j=0;j<4-i;j++) if(v[j]>v[j+1]){ int t=v[j];v[j]=v[j+1];v[j+1]=t; }

🔵 Matrizes — percurso e operações

int m[3][3] = { {1,2,3}, {4,5,6}, {7,8,9} }; // Percurso: linha externa, coluna interna for(int i=0;i<3;i++) { for(int j=0;j<3;j++) printf("%3d",m[i][j]); printf("\n"); } // Diagonal principal: i==j // Triângulo superior: j > i // Transposta: T[j][i] = A[i][j] // Soma de linha i: int soma=0; for(int j=0;j<3;j++) soma+=m[i][j];

✅ Regra de ouro dos arrays

Índice vai de 0 a N-1 em C/Java. Em Portugol, de 1 a N. Acessar fora desse intervalo é undefined behavior — o programa pode funcionar, travar ou se comportar de forma imprevisível.

⚙️ Revisão — Funções e Recursividade (Aulas 17–18)

DRY, passagem por valor vs. referência, caso base obrigatório

🔵 Anatomia de uma função

// tipo_retorno nome(parâmetros) float media(float v[], int n) { float soma = 0; for(int i=0;i<n;i++) soma+=v[i]; return soma / n; } // Por VALOR — cópia, original seguro void dobrar(int x) { x *=2; } ← x local! // Por REFERÊNCIA — modifica original void dobrar(int *x) { *x *=2; } ← ✅ // Vetores: SEMPRE por referência void zerar(int v[], int n) { for(int i=0;i<n;i++) v[i]=0; }

🔵 Recursividade — 2 ingredientes

// 1) CASO BASE — para a recursão // 2) CHAMADA RECURSIVA — problema menor int fatorial(int n) { if(n <= 1) return 1; ← caso base return n * fatorial(n-1); ← menor } // fatorial(4) = 4×3×2×1 = 24 int buscaBinaria(int v[], int ini, int fim, int alvo) { if(ini > fim) return -1; ← base int meio = (ini+fim)/2; if(v[meio]==alvo) return meio; if(v[meio]<alvo) return buscaBinaria(v,meio+1,fim,alvo); return buscaBinaria(v,ini,meio-1,alvo); }

⚠️ Recursão sem caso base = stack overflow

Sem caso base a função chama a si mesma infinitamente até estourar a pilha. Sempre pergunte: "o que para a recursão?"

✅ Iterativo vs. Recursivo

Recursivo é elegante para problemas naturalmente recursivos (árvores, dividir e conquistar). Iterativo é mais eficiente em memória. Se há solução iterativa simples, prefira-a.

🔤🗂️ Revisão — Strings e Structs (Aulas 19–20)

Texto como vetor de char, e tipos compostos com campos heterogêneos

🔤 Strings — resumo vital

// String = char[] terminado com '\0' char nome[20] = "Tiago"; ← 5 chars + '\0' // Leitura (fgets lê com espaços) fgets(nome, 20, stdin); // Remove '\n' do fgets: nome[strcspn(nome,"\n")]=0; // Funções essenciais strlen(s) ← comprimento sem '\0' strcpy(dst, src) ← cópia (não use =!) strcat(dst, src) ← concatena strcmp(s1, s2) ← 0 se iguais (não use ==!) strstr(s, sub) ← acha substring toupper/tolower ← converte maiúscula isalpha/isdigit ← testa caractere

🗂️ Structs — resumo vital

typedef struct { char nome[50]; int matricula; float media; } Aluno; Aluno a1 = {"Maria", 2023001, 8.5f}; // Acesso: . para variável, -> para ponteiro printf("%s", a1.nome); Aluno *p = &a1; printf("%s", p->nome); // Copiar struct inteira com = Aluno a2 = a1; ← copia todos os campos! // Vetor de structs: array[i].campo Aluno turma[30]; int n=0; strcpy(turma[0].nome, "João"); n++;

❌ Os 4 erros mais comuns

String: usar == → use strcmp()  |  usar = → use strcpy()
Struct: usar . com ponteiro → use ->  |  percorrer até o tamanho máximo → percorra até n

✅ enum melhora a legibilidade

Use enum para campos com valores fixos: APROVADO/REPROVADO, ATIVO/INATIVO, AMIGO/TRABALHO. Elimina números mágicos e ativa avisos do compilador.

📍💾 Revisão — Ponteiros e Arquivos (Aulas 21–22)

Endereços de memória, alocação dinâmica e persistência em disco

📍 Ponteiros — resumo vital

int x = 42; int *p = &x; ← p guarda endereço de x *p = 99; ← modifica x via p // Array e ponteiros são equivalentes // v[i] == *(v+i) // Ponteiro NULL — sem alvo int *q = NULL; if(q) *q = 1; ← só usa se não for NULL // Alocação dinâmica int *v = (int*)malloc(n*sizeof(int)); if(!v) { puts("Sem mem!"); return 1; } // ... usa v[0]..v[n-1] ... free(v); v=NULL; ← SEMPRE liberar!

💾 Arquivos — resumo vital

// Ciclo obrigatório: fopen → operar → fclose FILE *fp = fopen("dados.txt", "w"); if(!fp) { perror("Erro"); return 1; } fprintf(fp, "Nome,Nota\nMaria,8.5\n"); fclose(fp); // Binário — structs inteiras FILE *fb = fopen("dados.dat", "wb"); fwrite(&n, sizeof(int), 1, fb); fwrite(turma, sizeof(Aluno), n, fb); fclose(fb); // Modos: "r" lê, "w" cria/apaga, // "a" adiciona, "rb"/"wb" binário

❌ Proibições em ponteiros

Usar ponteiro não inicializado (lixo) → crash  |  *p quando p == NULL → Segfault  |  malloc sem free → memory leak  |  Retornar endereço de variável local → undefined behavior

❌ Proibições em arquivos

Usar fp sem verificar NULL → crash  |  Esquecer fclose → dados ficam no buffer, não vão ao disco  |  "w" em arquivo existente → apaga tudo sem aviso  |  fwrite de struct em modo texto → corrupção

🧪 Mini-Quiz — Teste Seus Conhecimentos!

Clique em "Ver resposta" depois de pensar — sem olhar o código!

Q1. Qual a saída? int x=7, y=2; printf("%d %d", x/y, x%y);

3 1 — divisão inteira trunca: 7÷2=3 resto 1.

Q2. O que faz falta neste código?
for(int i=0; i<5; i++) printf("%d", v[i]);
O vetor tem 5 elementos, mas o código imprime lixo.

O vetor não foi inicializado. Declarar int v[5] sem inicializar deixa lixo na memória. Use = {0} ou preencha antes.

Q3. Por que if(nome == "Maria") nunca funciona em C?

Em C, == compara os endereços dos dois ponteiros, não o conteúdo. Use sempre strcmp(nome, "Maria") == 0.

Q4. Quantas vezes executa?
int i=1; while(i<=10) { i*=2; }

4 vezes: i=1→2→4→8→16 (sai quando 16>10). Ou seja: iterações com i=1,2,4,8.

Q5. Qual o erro? char s[5] = "hello";

"hello" tem 5 chars + 1 '\0' = 6 bytes, mas o array só tem 5. Correto: char s[6] ou char s[].

Q6. O que fatorial(0) retorna com a implementação da aula?

1. O caso base é if(n <= 1) return 1, então fatorial(0)=1, o que é matematicamente correto (0! = 1).

Q7. Qual a diferença entre "w" e "a" no fopen?

"w" apaga o conteúdo existente e recomeça do zero. "a" (append) adiciona ao final sem apagar o que já existe.

Q8. Qual o valor de *p após: int x=5, *p=&x; x=10;

10. p aponta para x. Quando x muda, *p reflete a mudança, pois ambos apontam para o mesmo endereço de memória.

✅ Checklist para o Projeto Final

Avalie seu projeto antes de entregar — cada item vale pontos!

Revise cada item antes da entrega — qualidade vale mais que quantidade de código! 🏆
✅
Struct bem modelada
Campos com tipos corretos, typedef, campos string com tamanho adequado (+1 para '\0')
✅
Funções modularizadas
Cada função faz UMA coisa. Nomes descritivos. Sem código duplicado (DRY)
✅
Persistência funciona
carregar() ao iniciar, salvar() ao sair. Funciona em segunda execução com dados da primeira
✅
Tratamento de erros
fopen verificado (≠ NULL). Vetor não transborda (verifica se n < MAX). malloc verificado
✅
Menu funcional
do-while garante exibição. switch para opções. Opção 0 sai e salva. Entrada inválida tratada
✅
Strings comparadas com strcmp
Nenhum == comparando strings. Nenhum = atribuindo strings (usar strcpy)
⚠️
Ponteiros inicializados
Todo ponteiro tem valor (endereço válido ou NULL). Verificado antes de desreferenciar
⚠️
Relatório exportado
Arquivo .csv gerado com cabeçalho e totais. Abre no Excel. Situação calculada corretamente
⚠️
Busca implementada
Busca por campo chave (matrícula, código, nome). Retorno -1 ou NULL quando não encontrado
⚠️
Ordenação implementada
Bubble Sort ou qsort com comparador. Resultado exibido em ordem ao listar
✅
Código compila sem warnings
Compile com -Wall -Wextra. Todo warning é um bug potencial esperando para acontecer
✅
Testado com dados reais
Cadastrar, sair, reabrir, verificar que os dados ainda estão lá. Testar edge cases (0 itens, máximo)
