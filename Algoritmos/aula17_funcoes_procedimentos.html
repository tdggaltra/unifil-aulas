🤔 Por que usar Funções?

Código sem funções × código com funções — a diferença é enorme

❌ Sem funções — código duplicado

// Calcular média 3 vezes no mesmo programa! // Turma A: somaA = 0; for(i=0;i<30;i++) somaA+=notasA[i]; mediaA = somaA/30; printf("Média A: %.2f\n",mediaA); // Turma B: (copia e cola!) somaB = 0; for(i=0;i<28;i++) somaB+=notasB[i]; mediaB = somaB/28; printf("Média B: %.2f\n",mediaB); // Turma C: (copia e cola de novo!) somaC = 0; for(i=0;i<35;i++) somaC+=notasC[i]; // Se houver bug → corrigir 3x!

✅ Com funções — código limpo

// Definir UMA VEZ: float calcMedia(float v[], int n) { float soma = 0; for(int i=0; i<n; i++) soma += v[i]; return soma / n; } // Usar quantas vezes quiser: printf("Média A: %.2f\n", calcMedia(notasA, 30)); printf("Média B: %.2f\n", calcMedia(notasB, 28)); printf("Média C: %.2f\n", calcMedia(notasC, 35)); // Bug? Corrigir em 1 lugar só!
Funções = escrever UMA VEZ, usar MUITAS VEZES. Corrigir em UM lugar, corrige em todos! ♻️

🏭 Analogia: Linha de Produção

Uma fábrica não reconstrói cada máquina do zero para cada produto. Ela tem máquinas especializadas (funções) que executam tarefas específicas. O gerente (main) coordena quais máquinas operam e em que ordem.

🏗️ Anatomia de uma Função

Tipo de retorno, nome, parâmetros e corpo — cada parte tem seu papel

🟣 tipo retorno 🔵 nome 🩵 parâmetros ⬜ corpo 🔴 return
float calcMedia(float notas[], int n) { float soma = 0; for(int i = 0; i < n; i++) soma += notas[i]; return soma / n; }

🟣 Tipo de retorno

  • int — retorna inteiro
  • float / double — retorna real
  • char — retorna caractere
  • void — não retorna nada (procedimento)

🔵 Nome da função

  • Verbo que descreve a ação
  • calcular, exibir, verificar
  • camelCase: calcularMedia
  • Sem espaços ou acentos

🩵 Parâmetros

  • Dados que a função recebe
  • Cada um: tipo nome
  • Separados por vírgula
  • Pode ter zero parâmetros: ()

✅ Função vs. Procedimento

Função: retorna um valor (return x). Usada em expressões: media = calcMedia(v, n).

Procedimento: tipo void, não retorna. Apenas executa ações: exibirMenu(), imprimirRelatorio().

No Portugol: funcao retorna; procedimento não retorna.

⚠️ Declaração x Definição x Chamada

// Declaração (protótipo) — antes do main float calcMedia(float[], int); // Chamada — dentro de outra função float m = calcMedia(notas, 30); // Definição — o corpo completo float calcMedia(float v[], int n) { ... return soma/n; }

🔄 Passagem por Valor vs. por Referência

A diferença fundamental: cópia do dado ou endereço de memória?

Por valor: a função recebe uma CÓPIA. Por referência: a função acessa o ORIGINAL! 📋 vs 🔑

📋 Passagem por VALOR — cópia

// Dobrar NÃO funciona por valor! void dobrar(int x) { x = x * 2; ← muda CÓPIA local } int num = 5; dobrar(num); printf("%d\n", num); ← ainda 5! // A variável original NÃO mudou! // ✅ Correto: retornar o novo valor int dobrarOk(int x) { return x * 2; } num = dobrarOk(num); ← num = 10 ✅

🔑 Passagem por REFERÊNCIA — ponteiro (C)

// Dobrar funciona com ponteiro! void dobrar(int *x) { *x = *x * 2; ← muda ORIGINAL } int num = 5; dobrar(&num); ← passa endereço! printf("%d\n", num); ← agora 10! ✅ // Troca de dois valores: void trocar(int *a, int *b) { int tmp = *a; *a = *b; *b = tmp; } trocar(&x, &y); ← x e y trocados!

🚗 Analogia: Chave vs. Cópia do Carro

Por valor: você recebe uma foto do carro. Pode olhar, mas não dirigir o real.
Por referência: você recebe a chave. Pode dirigir e modificar o carro original.

📐 Vetores são sempre por referência!

Em C, ao passar um vetor para uma função, você passa automaticamente o endereço do primeiro elemento. Modificações dentro da função afetam o vetor original — sem precisar de &.

// v[] é um ponteiro para o array void zerar(int v[], int n) { for(int i=0;i<n;i++) v[i]=0; } // Array original é modificado!

💻 Exemplo 1 — Calculadora Modular

Cada operação em sua própria função — código limpo e extensível

📌 Problema Básico

Criar uma calculadora com menu onde cada operação (+, -, ×, ÷) é uma função separada. O main apenas coordena o fluxo.

🔵 C — funções de operação

float somar (float a, float b){ return a+b; } float subtrair(float a, float b){ return a-b; } float multiplicar(float a, float b){ return a*b; } float dividir (float a, float b){ if(b==0){ printf("Erro: div/0\n"); return 0; } return a/b; } void exibirMenu() { printf("\n1-Somar 2-Subtrair\n"); printf("3-Mult. 4-Dividir 0-Sair\n"); printf("Opção: "); } int main() { float a, b, res; int op; do { exibirMenu(); scanf("%d", &op); if(op==0) break; printf("A B: "); scanf("%f %f",&a,&b); switch(op) { case 1: res=somar(a,b); break; case 2: res=subtrair(a,b); break; case 3: res=multiplicar(a,b); break; case 4: res=dividir(a,b); break; } printf("Resultado: %.2f\n", res); } while(1); return 0; }

✅ Vantagens da modularização

  • main limpo: só coordena o fluxo
  • Testável: cada função pode ser testada sozinha
  • Extensível: adicionar potência = criar elevar()
  • Legível: dividir(a,b) é mais claro que 5 linhas inline

📋 Responsabilidade Única

Cada função faz uma coisa só e faz bem. somar() só soma — não imprime, não lê, não valida. exibirMenu() só exibe — não processa. Esse princípio é chamado de Single Responsibility e é a base do bom código.

🔍 Diagrama de chamadas

main()
├── exibirMenu()   [void]
├── somar(a,b)     [float]
├── subtrair(a,b)  [float]
├── multiplicar()  [float]
└── dividir(a,b)   [float]

💻 Exemplo 2 — Funções com Vetores

Passar vetores para funções — sempre por referência em C

📌 Problema Intermediário

Criar um conjunto de funções para processar um vetor de notas: ler, calcular média, encontrar maior/menor e exibir formatado.

🔵 C — biblioteca de funções para vetores

// ─── Leitura ─────────────────────── void lerNotas(float v[], int n) { for(int i=0; i<n; i++) { printf("Nota[%d]: ",i+1); scanf("%f", &v[i]); } } // ─── Média ───────────────────────── float calcMedia(float v[], int n) { float s=0; for(int i=0;i<n;i++) s+=v[i]; return s/n; } // ─── Maior ───────────────────────── float maior(float v[], int n) { float m=v[0]; for(int i=1;i<n;i++) if(v[i]>m) m=v[i]; return m; } // ─── Exibir ──────────────────────── void exibirBoletim(float v[], int n) { printf("=== BOLETIM ===\n"); for(int i=0;i<n;i++) printf("Al.%d: %.1f %s\n",i+1,v[i], v[i]>=6?"✅":"❌"); printf("Média : %.2f\n",calcMedia(v,n)); printf("Maior : %.2f\n",maior(v,n)); }

🔵 C — main simplificado

#define N 6 int main() { float notas[N]; lerNotas(notas, N); exibirBoletim(notas, N); return 0; } // Main tem apenas 3 linhas úteis! // Toda a lógica está nas funções.

✅ Funções chamam funções

exibirBoletim() chama calcMedia() e maior() internamente. Funções podem usar outras funções — criando uma hierarquia de chamadas clara e reutilizável.

🔑 Regra: vetor + tamanho

Em C, ao passar um vetor para uma função, sempre passe também seu tamanho como parâmetro separado. A função não tem como saber o tamanho pelo vetor sozinho!

🔭 Escopo de Variáveis — Local vs. Global

Onde uma variável existe e pode ser acessada — uma das regras mais importantes!

Variável local vive dentro da função. Global vive em todo o programa. Prefira locais! 🏠

🔵 C — escopo na prática

int global = 100; ← GLOBAL: todo o arquivo void funcaoA() { int local = 10; ← LOCAL: só aqui printf("%d\n", global); ← ✅ ok printf("%d\n", local); ← ✅ ok } ← local morre aqui! void funcaoB() { int local = 20; ← OUTRO local printf("%d\n", global); ← ✅ ok // printf(localA) ← ERRO! // localA não existe aqui! } int main() { funcaoA(); funcaoB(); // localA e localB: não existem aqui! }

📚 Pilha de chamadas (call stack):

🏠
main()
variáveis: notas[], n
↓ chama
⚙️
calcMedia()
parâm: v[], n | local: soma, i
↩ retorna float
Cada chamada de função cria um novo frame na pilha com suas variáveis locais. Quando a função retorna, o frame é destruído — as variáveis locais somem.

⚠️ Cuidado com variáveis globais!

Variáveis globais parecem convenientes, mas criam acoplamento: qualquer função pode alterá-las sem aviso. Prefira passar dados via parâmetros — código mais previsível e testável.

💻 Exemplo 3 — Sistema de Cadastro Modular

Programa real com múltiplas funções colaborando — padrão profissional

📌 Problema Avançado

Sistema de cadastro de produtos com menu: cadastrar, listar, buscar por nome e calcular valor total do estoque. Cada operação em sua função.

🔵 C — funções do sistema

#define MAX 50 char nomes[MAX][30]; float precos[MAX]; int qtds[MAX], total=0; // ── Cadastrar produto ── void cadastrar() { if(total==MAX){ printf("Cheio!\n"); return; } printf("Nome: "); scanf("%s",nomes[total]); printf("Preço: "); scanf("%f",&precos[total]); printf("Qtd: "); scanf("%d",&qtds[total]); total++; printf("✅ Cadastrado!\n"); } // ── Listar todos ── void listar() { printf("%-20s %8s %6s\n","Nome","Preço","Qtd"); printf("──────────────────────────────────\n"); for(int i=0;i<total;i++) printf("%-20s R$%7.2f %6d\n", nomes[i],precos[i],qtds[i]); } // ── Valor total do estoque ── float valorEstoque() { float vt=0; for(int i=0;i<total;i++) vt+=precos[i]*qtds[i]; return vt; }

🔵 C — main e exibirMenu

void exibirMenu() { printf("\n╔═══════════════════╗\n"); printf("║ ESTOQUE v1.0 ║\n"); printf("╠═══════════════════╣\n"); printf("║ 1 - Cadastrar ║\n"); printf("║ 2 - Listar ║\n"); printf("║ 3 - Valor total ║\n"); printf("║ 0 - Sair ║\n"); printf("╚═══════════════════╝\n"); printf("Opção: "); } int main() { int op; do { exibirMenu(); scanf("%d",&op); switch(op){ case 1: cadastrar(); break; case 2: listar(); break; case 3: printf("Total: R$%.2f\n", valorEstoque()); break; } }while(op!=0); return 0; }

✅ main com 10 linhas úteis

Todo o main faz é: mostrar menu, ler opção, chamar a função certa. A lógica real está encapsulada em cada função. Isso é separação de responsabilidades na prática!

🚨 Erros Comuns com Funções

Armadilhas clássicas — aprenda antes de cair!

❌ Erro 1: Esquecer o return

// ERRADO: função float sem return! float calcMedia(float v[], int n) { float soma=0; for(int i=0;i<n;i++) soma+=v[i]; // ESQUECEU o return! // Retorna lixo de memória! } // CORRETO: float calcMedia(float v[], int n) { float soma=0; for(int i=0;i<n;i++) soma+=v[i]; return soma/n; ← obrigatório! }

❌ Erro 2: Parâmetros com tipo errado

// ERRADO: int recebe float void exibir(int nota) { ← int! printf("%.1f\n", nota); } exibir(8.7); ← trunca para 8! // CORRETO: tipos compatíveis void exibir(float nota) { printf("%.1f\n", nota); }

❌ Erro 3: Usar variável local fora da função

// ERRADO: resultado não existe fora! void calcular(int a, int b) { int resultado = a + b; } int main() { calcular(3, 4); printf("%d", resultado); ← ERRO! // resultado não existe aqui! } // CORRETO: retornar o valor int calcular(int a, int b) { return a + b; } int res = calcular(3,4);

❌ Erro 4: Confundir void com função que retorna

// ERRADO: usar resultado de void void exibir(float n) { printf("%.2f", n); } float x = exibir(5.0); ← ERRO! // void não retorna valor! // CORRETO: void só é chamada exibir(5.0); ← sem atribuição

✅ Checklist para funções

☑ Toda função não-void tem return?  |  ☑ Tipos dos parâmetros conferem com os argumentos?  |  ☑ Variáveis locais não são usadas fora?  |  ☑ Protótipo declarado antes do main?

✅ Conceitos aprendidos

  • Função: tipo retorno + nome + parâmetros + corpo
  • Procedimento: tipo void, só executa ações
  • Por valor: cópia | Por referência: ponteiro / vetor
  • Escopo: local (na função) × global (todo o arquivo)

✅ Exemplos resolvidos

  • Calculadora modular — cada operação numa função
  • Biblioteca de funções para vetores (ler, média, maior)
  • Sistema de estoque com menu totalmente modularizado

⚠️ Erros a evitar

  • Esquecer return em função não-void
  • Tipos incompatíveis nos parâmetros
  • Usar variável local fora de sua função
  • Funções muito longas fazendo tudo
