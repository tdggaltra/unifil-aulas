🔑 fopen — Abrindo Arquivos

O primeiro passo: escolher o arquivo e o modo de acesso correto

fopen retorna um ponteiro FILE*. Se retornar NULL, o arquivo não pôde ser aberto! 📂
ModoSignificadoArquivo existe?Arquivo não existe?
"r"Leitura (read)Abre para lerRetorna NULL ❌
"w"Escrita (write)⚠️ Apaga o conteúdo!Cria novo arquivo
"a"Apêndice (append)Adiciona ao finalCria novo arquivo
"r+"Leitura e escritaAbre sem apagarRetorna NULL ❌
"w+"Escrita e leitura⚠️ Apaga o conteúdo!Cria novo arquivo
"rb" / "wb"Binário leitura/escritaComo r/w mas binárioIdem

🔵 C — padrão correto de abertura

#include <stdio.h> // Sempre verificar retorno de fopen! FILE *fp = fopen("alunos.txt", "w"); if(fp == NULL) { printf("Erro ao abrir arquivo!\n"); return 1; ← encerrar com erro } // ... usar o arquivo ... fclose(fp); ← SEMPRE fechar ao final! // fclose garante que o buffer seja // gravado no disco (flush)

⚠️ Modo "w" apaga tudo!

Usar "w" em um arquivo existente apaga todo o conteúdo anterior instantaneamente, sem aviso. Para adicionar dados sem destruir o que existe, use "a" (append). Para ler e escrever sem apagar, use "r+".

✅ O ciclo completo

1️⃣ fopen() — abre/cria
2️⃣ fprintf() / fscanf() — opera
3️⃣ fclose() — fecha e grava

✍️ Arquivos de Texto — Escrita e Leitura

fprintf e fscanf funcionam exatamente como printf e scanf — só mudam o destino

🔵 C — escrever no arquivo

FILE *fp = fopen("notas.txt", "w"); if(!fp) { perror("Erro"); return 1; } // fprintf = printf, mas para FILE* fprintf(fp, "Maria,20230042,8.75\n"); fprintf(fp, "João,20230043,7.50\n"); fprintf(fp, "Ana,20230044,9.20\n"); // fputs — grava string simples fputs("# Fim do arquivo\n", fp); // fputc — grava um caractere fputc('!', fp); fclose(fp); printf("Arquivo gravado!\n");

🔵 C — ler do arquivo

FILE *fp = fopen("notas.txt", "r"); if(!fp) { perror("Erro"); return 1; } // Modo 1: fscanf — lê formatado char nome[50]; int mat; float media; while(fscanf(fp, "%49[^,],%d,%f\n", nome, &mat, &media) == 3) { printf("%-15s %d %.2f\n",nome,mat,media); } rewind(fp); ← volta ao início! // Modo 2: fgets — lê linha por linha char linha[100]; while(fgets(linha, 100, fp) != NULL) { // linha inclui '\n' no final printf("%s", linha); } // feof — chegou ao fim? if(feof(fp)) printf("Fim do arquivo.\n"); fclose(fp);

💿 Arquivos Binários — fwrite e fread

Gravar structs inteiras em binário — mais rápido, compacto e exato

Texto: legível por humanos. Binário: legível por máquinas — structs inteiras de uma vez! ⚡

🔵 C — gravar structs com fwrite

typedef struct { char nome[50]; int matricula; float media; } Aluno; Aluno turma[] = { {"Maria", 2023001, 8.75f}, {"João", 2023002, 7.50f}, {"Ana", 2023003, 9.20f} }; int n = 3; // Gravar em binário — modo "wb" FILE *fp = fopen("turma.dat", "wb"); if(!fp){ perror("Erro"); return 1; } // Grava n (quantidade) primeiro fwrite(&n, sizeof(int), 1, fp); // Grava o vetor inteiro de uma vez! fwrite(turma, sizeof(Aluno), n, fp); fclose(fp); printf("Gravados %d alunos.\n", n);

🔵 C — ler structs com fread

Aluno turma2[100]; int n2 = 0; // Abrir em modo "rb" FILE *fp = fopen("turma.dat", "rb"); if(!fp){ perror("Erro"); return 1; } // Ler quantidade primeiro fread(&n2, sizeof(int), 1, fp); // Ler o vetor inteiro fread(turma2, sizeof(Aluno), n2, fp); fclose(fp); // Usar os dados carregados: for(int i=0;i<n2;i++) printf("%-15s %.2f\n", turma2[i].nome, turma2[i].media);

💻 Exemplo 1 — Cadastro de Alunos Persistente

Carregar ao iniciar, salvar ao sair — dados que sobrevivem entre execuções

📌 Problema Intermediário

Sistema de cadastro que carrega alunos de alunos.dat ao iniciar (se existir), permite operações via menu e salva automaticamente ao sair. Arquivo binário com structs.

🔵 C — carregar e salvar

#define ARQUIVO "alunos.dat" #define MAX 100 Aluno turma[MAX]; int n = 0; // Carregar do disco (ao iniciar) void carregar() { FILE *fp = fopen(ARQUIVO, "rb"); if(!fp) { printf("Novo cadastro iniciado.\n"); return; ← arquivo ainda não existe } fread(&n, sizeof(int), 1, fp); fread(turma, sizeof(Aluno), n, fp); fclose(fp); printf("%d aluno(s) carregado(s).\n", n); } // Salvar no disco (ao sair) void salvar() { FILE *fp = fopen(ARQUIVO, "wb"); if(!fp){ perror("Erro"); return; } fwrite(&n, sizeof(int), 1, fp); fwrite(turma, sizeof(Aluno), n, fp); fclose(fp); printf("%d aluno(s) salvos.\n", n); }

🔵 C — main com menu

int main() { carregar(); ← carrega ao iniciar int op; do { printf("\n=== MENU ===\n"); printf("1. Cadastrar aluno\n"); printf("2. Listar alunos\n"); printf("3. Buscar por nome\n"); printf("4. Gerar relatório\n"); printf("0. Sair e salvar\n"); printf("Opção: "); scanf("%d", &op); switch(op) { case 1: cadastrar(); break; case 2: listar(); break; case 3: buscar(); break; case 4: relatorio();break; } } while(op != 0); salvar(); ← salva ao sair return 0; }

✅ Persistência transparente

O usuário não percebe a leitura/escrita — o sistema simplesmente "lembra" os dados entre execuções. Este é o padrão fundamental de todos os sistemas com banco de dados: carregar ao iniciar, trabalhar em memória, persistir ao sair.

📍 Navegação — fseek, ftell e rewind

Mover o cursor do arquivo para ler ou atualizar registros específicos

🔵 C — fseek, ftell, rewind

// fseek(fp, offset, origem) // origens: SEEK_SET=início, SEEK_CUR=atual, SEEK_END=fim FILE *fp = fopen("turma.dat", "rb"); // Ir para o início fseek(fp, 0, SEEK_SET); // equivale a: rewind(fp) // Saber o tamanho do arquivo fseek(fp, 0, SEEK_END); long tam = ftell(fp); printf("Tamanho: %ld bytes\n", tam); // Calcular número de registros int n_reg = (tam - sizeof(int)) / sizeof(Aluno); printf("Registros: %d\n", n_reg); // Saltar para o 3º registro (índice 2) long pos = sizeof(int) + 2 * sizeof(Aluno); fseek(fp, pos, SEEK_SET); Aluno a; fread(&a, sizeof(Aluno), 1, fp); printf("3º aluno: %s\n", a.nome); fclose(fp);

🔵 C — atualizar registro no lugar

// Abrir para leitura E escrita: "r+b" FILE *fp = fopen("turma.dat", "r+b"); // Buscar aluno por matrícula fseek(fp, sizeof(int), SEEK_SET); ← pula n Aluno a; int i = 0; while(fread(&a, sizeof(Aluno), 1, fp)) { if(a.matricula == 2023002) { // Voltar e sobrescrever o registro! long pos = sizeof(int) + i*sizeof(Aluno); fseek(fp, pos, SEEK_SET); a.media = 8.0f; ← nova nota fwrite(&a, sizeof(Aluno), 1, fp); printf("Atualizado!\n"); break; } i++; } fclose(fp);

📍 Cursor do arquivo

O arquivo tem um cursor (posição atual). Cada fread/fwrite avança o cursor automaticamente. fseek move para qualquer posição. ftell retorna a posição atual. rewind volta ao início.

💻 Exemplo 2 — Exportar Relatório CSV

Gerar um arquivo aberto no Excel — saída profissional para o cliente

📌 Problema Intermediário

Exportar o cadastro de alunos para um arquivo CSV (relatorio.csv) com cabeçalho, uma linha por aluno, situação calculada e linha de totais ao final. Abrir no Excel diretamente.

🔵 C — gerar CSV completo

void exportarCSV(Aluno t[], int n) { FILE *fp = fopen("relatorio.csv", "w"); if(!fp){ perror("Erro"); return; } // Cabeçalho fprintf(fp, "Nome,Matricula,Media,Situacao\n"); int aprovados=0, reprovados=0; float somaMedias=0; for(int i=0; i<n; i++) { char *sit = (t[i].media>=6.0f) ? "Aprovado" : "Reprovado"; fprintf(fp, "%s,%d,%.2f,%s\n", t[i].nome, t[i].matricula, t[i].media, sit); somaMedias += t[i].media; if(t[i].media>=6) aprovados++; else reprovados++; } // Linha em branco + totais fprintf(fp, "\n"); fprintf(fp, "TOTAL,%d,%.2f,\n", n, somaMedias/n); fprintf(fp, "Aprovados:,%d,,\n", aprovados); fprintf(fp, "Reprovados:,%d,,\n",reprovados); fclose(fp); printf("📊 relatorio.csv gerado!\n"); }

✅ CSV — o formato universal

CSV (Comma-Separated Values) é texto simples que qualquer planilha entende. Basta usar vírgula como separador. Se um campo contiver vírgula, envolva em aspas: "Silva, João". É o formato mais usado para troca de dados entre sistemas!

⚠️ Ponto e vírgula no Brasil

O Excel brasileiro usa ponto e vírgula (;) como separador, pois a vírgula é usada como decimal. Troque o separador ou salve como .csv com sep=; na primeira linha.

💻 Exemplo 3 — Log de Operações com Timestamp

Modo append — registrar cada ação sem apagar o histórico anterior

📌 Problema Intermediário

Toda operação do sistema (cadastro, busca, alteração, exclusão) é registrada com data/hora em log.txt usando modo "a" — o histórico nunca é perdido.

🔵 C — sistema de log com timestamp

#include <time.h> #include <stdio.h> // Escrever no log (mode "a" = append) void log(const char *operacao, const char *detalhe) { FILE *fp = fopen("log.txt", "a"); if(!fp) return; // Obter data/hora atual time_t now = time(NULL); struct tm *t = localtime(&now); fprintf(fp, "[%04d-%02d-%02d %02d:%02d:%02d] " "%s: %s\n", t->tm_year+1900, t->tm_mon+1, t->tm_mday, t->tm_hour, t->tm_min, t->tm_sec, operacao, detalhe); fclose(fp); } // Uso no sistema: log("CADASTRO", "Maria - mat. 2023001"); log("BUSCA", "Consulta: João"); log("ALTERACAO","Mat.2023002 nota→8.0"); log("EXCLUSAO", "Mat.2023004 removido");
📄 log.txt — histórico acumulado
[2026-03-07 14:23:01] CADASTRO: Maria - mat. 2023001
[2026-03-07 14:23:45] BUSCA: Consulta: João
[2026-03-07 14:24:12] ALTERACAO: Mat.2023002 nota→8.0
[2026-03-07 14:25:03] EXCLUSAO: Mat.2023004 removido
[2026-03-08 09:10:22] CADASTRO: Pedro - mat. 2023005

⚠️ Por que modo "a" e não "w"?

Com "w" cada vez que o programa abre o log, apaga tudo. Com "a" (append) cada execução adiciona ao final — o histórico cresce indefinidamente. Esta é a lógica de todo sistema de auditoria real.

✅ Padrões de log profissional

Sistemas reais registram quem fez, o quê, quando e resultado. O formato [DATA_HORA] NIVEL: mensagem é padrão em sistemas bancários, e-commerce e médicos — rastreabilidade obrigatória por lei.

📋 Leitura de Arquivo de Configuração

Ler e processar arquivos de texto com formato definido — chave=valor

📌 Cenário real Avançado

Todo software profissional lê um arquivo de configuração ao iniciar. Implemente leitura de config.ini no formato chave=valor, ignorando comentários (#) e linhas em branco.

📄 config.ini
# Sistema de Cadastro v1.0
# Arquivo de configuração
 
max_alunos=100
arquivo_dados=turma.dat
backup=sim
log_ativo=sim
prefixo_log=cadastro

🔵 C — struct de configuração

typedef struct { int max_alunos; char arq_dados[60]; int backup; int log_ativo; char prefixo[30]; } Config;

🔵 C — ler arquivo de configuração

Config lerConfig(const char *arquivo) { Config cfg = {50,"dados.dat",0,0,"log"}; // Valores padrão acima FILE *fp = fopen(arquivo, "r"); if(!fp) return cfg; ← padrão se não existe char linha[100], chave[40], valor[60]; while(fgets(linha, sizeof(linha), fp)) { // Ignorar comentários e linhas em branco if(linha[0]=='#' || linha[0]=='\n') continue; // Separar chave=valor if(sscanf(linha, "%39[^=]=%59[^\n]", chave, valor) == 2) { if(strcmp(chave,"max_alunos")==0) cfg.max_alunos = atoi(valor); else if(strcmp(chave,"arquivo_dados")==0) strcpy(cfg.arq_dados, valor); else if(strcmp(chave,"backup")==0) cfg.backup=(strcmp(valor,"sim")==0); else if(strcmp(chave,"log_ativo")==0) cfg.log_ativo=(strcmp(valor,"sim")==0); } } fclose(fp); return cfg; }

🚨 Erros Comuns com Arquivos

Os descuidos que custam dados — aprenda a evitá-los!

❌ Erro 1: Não verificar fopen

// ERRADO: usar fp sem verificar FILE *fp = fopen("dados.txt", "r"); fprintf(fp, "Olá"); ← CRASH se fp==NULL! // CORRETO: sempre verificar FILE *fp = fopen("dados.txt", "r"); if(fp == NULL) { perror("dados.txt"); ← mostra motivo return 1; }

❌ Erro 2: Não fechar o arquivo

// ERRADO: sem fclose — dados podem // ficar no buffer e NÃO ir ao disco! FILE *fp = fopen("saida.txt", "w"); fprintf(fp, "Dados importantes"); // fim da função sem fclose ← ❌ // CORRETO: sempre fechar fclose(fp); ← força flush para o disco ✅ // ou: fflush(fp) para gravar sem fechar

❌ Erro 3: Modo errado — "w" apaga tudo!

// ERRADO: "w" em arquivo existente FILE *fp = fopen("log.txt", "w"); // ← APAGA TODO O HISTÓRICO! ❌ // CORRETO para adicionar: FILE *fp = fopen("log.txt", "a"); // ← mantém o conteúdo existente ✅

❌ Erro 4: Misturar binário e texto

// ERRADO: gravar struct em modo texto FILE *fp = fopen("dados.dat", "w");← texto! fwrite(&aluno, sizeof(Aluno), 1, fp); // No Windows '\n' vira '\r\n' → corrompe! // CORRETO: binário para structs FILE *fp = fopen("dados.dat", "wb");← ✅ fwrite(&aluno, sizeof(Aluno), 1, fp);

✅ Checklist de arquivos

☑ Verificar retorno do fopen (!= NULL)?  |  ☑ Usar fclose ao terminar?  |  ☑ Modo correto ("r" lê, "w" apaga+cria, "a" adiciona)?  |  ☑ Binário usa "rb"/"wb"?  |  ☑ fread verifica quantidade retornada?

📝 Exercícios Propostos

Persistência é o que transforma um programa em um sistema — pratique!

Dados sem persistência se perdem. Programe hoje, e o arquivo ainda estará lá amanhã! 💾

📌 Exercício A — Agenda Persistente Básico

Crie uma agenda de contatos (nome, telefone, email) que salva em agenda.dat (binário) ao sair e carrega ao iniciar. Menu: adicionar, listar, buscar por nome, sair. Ao fechar e reabrir o programa, os contatos devem permanecer.

📌 Exercício B — Gerador de Boletim Intermediário

Leia um arquivo notas.csv com linhas no formato nome,nota1,nota2,nota3,nota4. Calcule a média e situação de cada aluno. Gere um arquivo boletim.txt formatado como relatório de texto e um resultado.csv com os dados processados.

📌 Exercício C — Backup Automático Intermediário

Estenda o sistema de cadastro da Aula para criar um backup automático: ao salvar, copiar o arquivo atual para backup_AAAAMMDD.dat com a data de hoje no nome (use time.h e sprintf para formar o nome). Implemente também uma função para listar e restaurar backups.

📌 Exercício D — Mini Banco de Dados Desafio

Crie um sistema com dois arquivos: produtos.dat (cadastro) e movimentos.txt (log de todas as operações). Implemente: cadastrar produto, registrar venda (desconta estoque), registrar compra (adiciona estoque), gerar relatório CSV e exibir histórico de movimentos filtrado por produto.
