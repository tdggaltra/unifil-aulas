🤔 Por que precisamos de break e continue?

Situações em que a condição normal do laço não é suficiente

Às vezes queremos parar o laço no meio ou pular alguns valores — sem alterar a condição principal.

⛔ Quando usar break?

  • Encontrou o que procurava — parar a busca
  • Ocorreu um erro — abortar o processamento
  • Usuário pediu para cancelar
  • Primeiro número inválido em uma sequência
// Buscar o 1º número par para i <- 1 ate 100 faca se (i % 2 == 0) entao escreva("Achei: ", i) interrompa ← sai imediatamente fimse fimpara

⏭️ Quando usar continue?

  • Ignorar valores que não atendem a critério
  • Pular linhas em branco ou dados inválidos
  • Processar só múltiplos ou ímpares
  • Filtrar itens de uma lista
// Imprimir só os ímpares para i <- 1 ate 10 faca se (i % 2 == 0) entao continue ← pula os pares fimse escreva(i) ← só chega aqui se ímpar fimpara

🎭 Analogia: Revisão de Currículos

Um RH revisa uma pilha de currículos.
break = encontrou o candidato perfeito → para de ler os outros.
continue = candidato sem experiência → descarta e passa para o próximo sem análise detalhada.

⛔ O Comando BREAK

Interrompe o laço imediatamente e salta para a instrução seguinte

📐 Como funciona

Quando o break é executado, o laço encerra imediatamente, independente da condição, e o programa continua na primeira instrução após o fimpara / }.

para i <- 1 ate 10 faca se (i == 5) entao interrompa ← C/Java: break fimse escreva(i, " ") fimpara escreva("Fim") // Saída: 1 2 3 4 Fim // (i=5 dispara break antes de escrever)

🔵 C / 🟠 Java — mesma sintaxe

for(int i=1; i<=10; i++) { if(i == 5) break; ← encerra printf("%d ", i); } printf("Fim\n"); // Saída: 1 2 3 4 Fim

⚠️ break em laços aninhados

O break encerra apenas o laço mais interno onde está inserido — não encerra todos os laços!

para i <- 1 ate 3 faca para j <- 1 ate 3 faca se j==2 entao interrompa fimse escreva(i,j) ← encerra só o interno fimpara fimpara // Saída: 11 21 31

⏭️ O Comando CONTINUE

Pula o restante da iteração atual e vai para a próxima

📐 Como funciona

Quando o continue é executado, o restante do corpo do laço é ignorado e o controle vai diretamente para o teste da condição (ou incremento no for).

para i <- 1 ate 10 faca se (i % 2 == 0) entao continue ← pula pares fimse escreva(i, " ") fimpara // Saída: 1 3 5 7 9

🔵 C / 🟠 Java

for(int i=1; i<=10; i++) { if(i % 2 == 0) continue; ← pula printf("%d ", i); } // Saída: 1 3 5 7 9

✅ Diferença crucial: continue vs break

  • break = encerra o laço inteiro
  • continue = pula só a iteração atual; o laço continua
  • break: saída total  |  continue: filtro

🎵 Analogia: Playlist

break = parar a música e fechar o app.
continue = pular a música atual e ir para a próxima.

💻 Exemplo 1 — Busca Linear com break

Encontrar o primeiro elemento que satisfaz uma condição

📌 Problema

Leia 10 números. Encontre e exiba o primeiro número divisível por 7. Ao encontrá-lo, pare a busca imediatamente.

🟢 Portugol

algoritmo "BuscaDiv7" var n, i : inteiro achado : logico inicio achado <- falso para i <- 1 ate 10 faca escreva("Número ", i, ": ") leia(n) se (n % 7 == 0) entao escreva("✅ Achei: ", n, " (na pos. ", i, ")") achado <- verdadeiro interrompa fimse fimpara se (nao achado) entao escreva("❌ Nenhum div. por 7") fimse fimalgoritmo

🔵 C

#include <stdio.h> int main() { int n, pos = -1; for(int i=1; i<=10; i++) { printf("Número %d: ", i); scanf("%d", &n); if(n % 7 == 0) { printf("Achei: %d" " (pos %d)\n", n, i); pos = i; break; } } if(pos == -1) printf("Nenhum div. 7\n"); return 0; }

🟠 Java

import java.util.Scanner; public class BuscaDiv7 { public static void main(String[] args) { Scanner sc = new Scanner(System.in); int pos = -1; for(int i=1; i<=10; i++){ System.out.print( "Número " + i + ": "); int n = sc.nextInt(); if(n % 7 == 0){ System.out.println( "Achei: " + n); pos = i; break; } } if(pos==-1) System.out.println( "Nenhum."); } }

✅ Por que break aqui economiza processamento?

Se o primeiro número já for divisível por 7, o laço encerra na primeira iteração — sem verificar os outros 9. Em listas enormes, isso faz enorme diferença de desempenho.

💻 Exemplo 2 — Filtro com continue

Processar somente os valores que atendem a um critério

📌 Problema

Leia 10 números. Some e conte apenas os positivos, ignorando zeros e negativos com continue.

🟢 Portugol

algoritmo "SomaPositivos" var n, i, qtd: inteiro soma : real inicio soma <- 0 qtd <- 0 para i <- 1 ate 10 faca escreva("Número ", i, ": ") leia(n) se (n <= 0) entao escreva("⏭️ Ignorado") continue ← pula não-positivos fimse soma <- soma + n qtd <- qtd + 1 escreva(" ✅ Somado: ", n) fimpara escreva("Positivos: ", qtd) escreva("Soma : ", soma) se (qtd > 0) entao escreva("Média : ", soma/qtd:6:2) fimse fimalgoritmo

🔵 C

#include <stdio.h> int main() { int n, qtd = 0; float soma = 0; for(int i=1; i<=10; i++) { printf("Número %d: ", i); scanf("%d", &n); if(n <= 0) { printf("Ignorado\n"); continue; } soma += n; qtd++; printf(" Somado: %d\n", n); } printf("Positivos: %d\n", qtd); printf("Soma: %.1f\n", soma); if(qtd>0) printf("Media: %.2f\n", soma/qtd); return 0; }

💡 continue vs if sem continue

O mesmo resultado pode ser obtido com se (n > 0) entao... fimse, mas o continue é mais expressivo quando há muito código depois: deixa claro que o caso negativo é uma exceção a ser descartada logo no início.

💻 Exemplo 3 — break em Validação de Senha

Combinando break com contador de tentativas em while

📌 Problema

Sistema de login: 3 tentativas. Se acertar a senha em qualquer momento, use break para sair imediatamente. Diferencie "saiu por acerto" de "saiu por limite".

🟢 Portugol

algoritmo "Login" var senha : caractere tent : inteiro acertou: logico inicio acertou <- falso para tent <- 1 ate 3 faca escreva("Tent. ", tent, "/3: ") leia(senha) se (senha == "1234") entao acertou <- verdadeiro interrompa ← sai ao acertar senao escreva("❌ Errou!") fimse fimpara // Verifica como saiu do laço se (acertou) entao escreva("✅ Acesso liberado!") senao escreva("🔒 Conta bloqueada!") fimse fimalgoritmo

✅ O padrão "flag + break"

Use uma variável booleana (acertou) para registrar por que o laço terminou — condição normal ou break. Após o laço, verifique a flag para tomar a decisão correta.

EntradaResultado "0000", "1234"✅ Acesso liberado na 2ª tent. "1234"✅ Acesso liberado na 1ª tent. "abc", "xyz", "999"🔒 Conta bloqueada (3 erros)

🎯 Quando usar flag + break

  • Precisa saber como o laço terminou
  • Condição de saída é mista (acerto OU limite)
  • Código após o laço depende do motivo da saída

💻 Exemplo 4 — continue com Laços Aninhados

Filtrar apenas os múltiplos de 3 na tabuada

📌 Problema

Exibir a tabuada de 1 a 5, mas mostrando apenas os produtos divisíveis por 3. Use continue para pular os demais.

🟢 Portugol

algoritmo "TabuadaMult3" var i, j, prod: inteiro inicio escreva("Produtos divisíveis por 3:") escreval("─────────────────────────") para i <- 1 ate 5 faca para j <- 1 ate 10 faca prod <- i * j // Pular se não é mult. de 3 se (prod % 3 <> 0) entao continue fimse escreva(i, " x ", j, " = ", prod) fimpara fimpara fimalgoritmo
Produtos divisíveis por 3: ───────────────────────── 1 x 3 = 3 1 x 6 = 6 1 x 9 = 9 2 x 3 = 6 2 x 6 = 12 2 x 9 = 18 3 x 1 = 3 3 x 2 = 6 3 x 3 = 9 3 x 4 = 12 3 x 5 = 15 ... // 5×7 = 35 → não mult.3 → pula // 5×9 = 45 → mult.3 → exibe

🔍 continue em laço aninhado

Dentro do laço interno (j), o continue pula apenas a iteração atual do interno — voltando ao teste de j. O laço externo (i) não é afetado.

Quando usar cada um e os cuidados necessários

Critério⛔ break⏭️ continue

⚠️ Não abuse do break e continue

  • Usar em excesso dificulta a leitura do código
  • Muitos breaks em um laço = sinal de lógica confusa
  • Prefira condições claras na assinatura do laço quando possível
  • Nunca substitua toda a lógica de condição por break

✅ Uso adequado

  • break: 1 ponto de saída antecipada bem identificado
  • continue: filtragem inicial de casos excepcionais
  • Sempre documente com comentário o porquê do break/continue
  • Verifique se a flag booleana é necessária após o laço

🛒 Analogia: Fila do Caixa

break = fechar o caixa no meio da fila — todos os clientes restantes ficam sem atendimento.
continue = pular o cliente que chegou sem produto — atende o próximo mas a fila continua.

🚨 Erros Comuns com break e continue

Armadilhas frequentes — especialmente em laços aninhados

❌ Erro 1: break fora de laço ou switch

// ERRADO: break fora de laço se (x > 0) entao interrompa ← erro de compilação! fimse // break/continue só funcionam // DENTRO de um laço (for/while/do)

❌ Erro 2: Achar que break encerra todos os laços

// Tentativa de sair do externo para i <- 1 ate 5 faca para j <- 1 ate 5 faca se (i*j==6) entao interrompa ← sai só do interno! fimse fimpara ← externo ainda continua! fimpara // SOLUÇÃO: flag booleana var parar: logico parar <- falso para i <- 1 ate 5 faca se (parar) entao interrompa fimse para j <- 1 ate 5 faca se (i*j==6) entao parar <- verdadeiro interrompa fimse fimpara fimpara

❌ Erro 3: continue no for — incremento ainda executa

// No FOR, continue vai para o incremento for(int i=1; i<=5; i++) { if(i==3) continue; printf("%d ", i); } // Saída: 1 2 4 5 (o i++ SEMPRE executa) // Sem loop infinito — OK! // No WHILE, cuidado: o incremento // pode não executar! int i=1; while(i<=5){ if(i==3) continue; ← loop infinito! printf("%d ", i); i++; } // CORRETO para while: while(i<=5){ if(i==3){ i++; continue; } ← incrementa antes! printf("%d ", i); i++; }

✅ Regra de ouro para continue em while

Sempre incremente a variável de controle antes do continue em um while, ou você cria um loop infinito! No for isso não é problema — o incremento da assinatura sempre executa.

✅ break — encerrar o laço

  • Sai imediatamente do laço mais interno

  • Pula o restante da iteração atual
  • O laço continua normalmente
  • No while: incrementar antes do continue!
  • Uso: filtro, dados inválidos, exceções
