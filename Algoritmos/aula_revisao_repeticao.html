🔹 Problema 1: Soma até digitar 0 (while)

Sentinela: loop controlado por valor especial

📝 Enunciado

Escreva um programa que leia números inteiros do usuário e some todos eles. O programa deve parar quando o usuário digitar 0 (zero). Ao final, mostre a soma total.

✅ Solução em Portugol
programa { funcao inicio() { inteiro numero, soma = 0 // Ler primeiro número escreva("Digite um número (0 para sair): ") leia(numero) // Loop enquanto não digitar 0 enquanto (numero != 0) { soma = soma + numero escreva("Digite outro número (0 para sair): ") leia(numero) } escreva("Soma total: ", soma, "\n") } }

🔍 Por que while?

  • NÃO sabemos quantos números serão digitados
  • Depende do usuário digitar 0
  • Pode executar 0 vezes (se digitar 0 logo)
  • Padrão: sentinela

📊 Trace (exemplo)

Entrada: 10, 20, 30, 0

  • Iteração 1: numero=10, soma=0+10=10
  • Iteração 2: numero=20, soma=10+20=30
  • Iteração 3: numero=30, soma=30+30=60
  • numero=0 → loop termina
  • Saída: 60

⚠️ Erro Comum

Esquecer de ler o número DENTRO do loop! Se não ler novamente, o loop fica infinito porque numero nunca muda.

🔹 Problema 2: Validação de Senha (do-while)

Execução garantida pelo menos uma vez

📝 Enunciado

Crie um programa que peça ao usuário para digitar a senha correta. A senha é "1234". O programa deve continuar pedindo até o usuário acertar. Ao acertar, mostre "Acesso liberado!".

✅ Solução em Portugol
programa { funcao inicio() { inteiro senha, senhaCorreta = 1234 faca { escreva("Digite a senha: ") leia(senha) se (senha != senhaCorreta) { escreva("Senha incorreta! Tente novamente.\n") } } enquanto (senha != senhaCorreta) escreva("Acesso liberado!\n") } }

🔍 Por que do-while?

  • DEVE pedir senha PELO MENOS uma vez
  • Teste da condição é no FINAL
  • Ideal para validações
  • Garante execução mínima

🆚 while vs do-while

  • while: testa ANTES → pode não executar
  • do-while: testa DEPOIS → executa 1x no mínimo
  • Use do-while para menus e validações
  • Use while para sentinelas

✅ Quando usar do-while

  • Menus de opções (mostrar pelo menos 1x)
  • Validação de entrada (pedir pelo menos 1x)
  • Jogos (jogar pelo menos 1 rodada)
  • Confirmações (perguntar pelo menos 1x)

🔹 Problema 3: Tabuada Completa (for)

Quantidade de iterações conhecida

📝 Enunciado

Leia um número do usuário e mostre a tabuada completa desse número, de 1 até 10. Formato: "5 x 1 = 5", "5 x 2 = 10", etc.

✅ Solução em Portugol
programa { funcao inicio() { inteiro numero escreva("Digite um número: ") leia(numero) escreva("\nTabuada do ", numero, ":\n") para (inteiro i = 1; i <= 10; i++) { escreva(numero, " x ", i, " = ", numero * i, "\n") } } }

🔍 Por que for?

  • Sabemos EXATAMENTE: 10 iterações
  • Contador cresce de 1 em 1
  • Início, fim e incremento claros
  • Padrão perfeito para for

📊 Exemplo de Saída

Entrada: 5

5 x 1 = 5
5 x 2 = 10
5 x 3 = 15
...
5 x 10 = 50

🎯 Estrutura do for

for (inicialização; condição; incremento)
• Inicialização: executada 1 vez no início
• Condição: testada antes de cada iteração
• Incremento: executado após cada iteração

🔹 Problema 4: Fatorial (for)

Multiplicação acumulada

📝 Enunciado

Calcule o fatorial de um número N. Fatorial: N! = N × (N-1) × (N-2) × ... × 1. Exemplo: 5! = 5 × 4 × 3 × 2 × 1 = 120

✅ Solução em Portugol
programa { funcao inicio() { inteiro numero, fatorial = 1 escreva("Digite um número: ") leia(numero) // Multiplicar de N até 1 para (inteiro i = numero; i >= 1; i--) { fatorial = fatorial * i } escreva(numero, "! = ", fatorial, "\n") } }

🔍 Conceitos-chave

  • Acumulador multiplicativo: começa em 1
  • Decremento: i-- (de N até 1)
  • IMPORTANTE: inicializar fatorial = 1
  • Se iniciar em 0, resultado sempre 0!

📊 Trace: 5!

  • Início: fatorial = 1
  • i=5: fatorial = 1 × 5 = 5
  • i=4: fatorial = 5 × 4 = 20
  • i=3: fatorial = 20 × 3 = 60
  • i=2: fatorial = 60 × 2 = 120
  • i=1: fatorial = 120 × 1 = 120
  • Resultado: 120 ✓

⚠️ Erro Comum

Inicializar o acumulador com 0 em vez de 1. Para multiplicação, sempre inicialize com 1, não 0!

🔹 Problema 5: Sequência de Fibonacci (for)

Dois acumuladores trabalhando juntos

📝 Enunciado

Mostre os primeiros N termos da sequência de Fibonacci. Fibonacci: cada termo é a soma dos dois anteriores. Começa com 0 e 1. Exemplo (5 termos): 0, 1, 1, 2, 3

✅ Solução em Portugol
programa { funcao inicio() { inteiro n, a = 0, b = 1, proximo escreva("Quantos termos? ") leia(n) escreva("Fibonacci: ") para (inteiro i = 1; i <= n; i++) { escreva(a, " ") // Calcular próximo termo proximo = a + b a = b b = proximo } escreva("\n") } }

🔍 Lógica do Fibonacci

  • Dois acumuladores: a e b
  • Mostrar a, depois "avançar"
  • proximo = a + b
  • a recebe b, b recebe proximo

📊 Trace: 5 termos

  • i=1: mostra a=0, próx=0+1=1, a=1, b=1
  • i=2: mostra a=1, próx=1+1=2, a=1, b=2
  • i=3: mostra a=1, próx=1+2=3, a=2, b=3
  • i=4: mostra a=2, próx=2+3=5, a=3, b=5
  • i=5: mostra a=3
  • Saída: 0 1 1 2 3

💡 Padrão Comum

Sequências que dependem de termos anteriores sempre usam acumuladores. No Fibonacci: precisamos de 2 acumuladores (termo atual e anterior).

🔹 Problema 6: Verificar Número Primo (for + flag)

Decisão dentro de repetição

📝 Enunciado

Verifique se um número N é primo. Número primo: divisível apenas por 1 e ele mesmo. Exemplo: 7 é primo (divisores: 1 e 7). 6 não é primo (divisores: 1, 2, 3, 6).

✅ Solução em Portugol
programa { funcao inicio() { inteiro numero logico primo = verdadeiro escreva("Digite um número: ") leia(numero) // Números menores que 2 não são primos se (numero < 2) { primo = falso } senao { // Testar divisibilidade de 2 até numero-1 para (inteiro i = 2; i < numero; i++) { se (numero % i == 0) { primo = falso pare // Encontrou divisor, pode parar } } } se (primo) { escreva(numero, " é primo\n") } senao { escreva(numero, " não é primo\n") } } }

🔍 Conceitos-chave

  • Flag: variável booleana (primo)
  • Começa verdadeiro (otimista)
  • Se encontrar divisor → muda para falso
  • pare: otimização (sai do loop)

✅ Exemplos

  • N=7: testa 2,3,4,5,6 → nenhum divide → PRIMO
  • N=6: testa 2 → 6%2=0 → divide → NÃO PRIMO
  • N=2: não entra no loop → PRIMO
  • N=1: menor que 2 → NÃO PRIMO

💡 Padrão: Flag (Bandeira)

Use uma variável lógica para "lembrar" se algo aconteceu dentro do loop. Muito útil quando você precisa testar várias condições!

🔹 Problema 7: Menu Interativo (do-while + switch)

Programa completo no Portugol Web Studio

📝 Enunciado

Crie uma calculadora com menu: 1-Somar, 2-Subtrair, 3-Multiplicar, 4-Dividir, 0-Sair. O menu deve aparecer repetidamente até o usuário escolher sair (0).

✅ Solução COMPLETA em Portugol (copie e teste!)
programa { funcao inicio() { inteiro opcao real a, b, resultado faca { // Mostrar menu escreva("\n╔════════════════════════╗\n") escreva("║ CALCULADORA v1.0 ║\n") escreva("╚════════════════════════╝\n") escreva("1 - Somar\n") escreva("2 - Subtrair\n") escreva("3 - Multiplicar\n") escreva("4 - Dividir\n") escreva("0 - Sair\n") escreva("Escolha: ") leia(opcao) // Processar escolha se (opcao >= 1 e opcao <= 4) { escreva("Digite dois números: ") leia(a) leia(b) escolha (opcao) { caso 1: resultado = a + b escreva("\nResultado: ", a, " + ", b, " = ", resultado, "\n") pare caso 2: resultado = a - b escreva("\nResultado: ", a, " - ", b, " = ", resultado, "\n") pare caso 3: resultado = a * b escreva("\nResultado: ", a, " × ", b, " = ", resultado, "\n") pare caso 4: se (b != 0) { resultado = a / b escreva("\nResultado: ", a, " ÷ ", b, " = ", resultado, "\n") } senao { escreva("\n⚠️ ERRO: Divisão por zero!\n") } pare } } senao se (opcao != 0) { escreva("\n⚠️ Opção inválida!\n") } } enquanto (opcao != 0) escreva("\n👋 Até logo!\n") } }

2. Cole o código acima
3. Clique em "Executar"
4. Teste todas as operações!

📊 Comparação: Quando Usar Cada Estrutura

Guia de decisão rápida

Situação Estrutura Exemplo
Sei EXATAMENTE quantas vezes for Tabuada (1 a 10), Listar vetor[10]
NÃO sei quantas vezes (sentinela) while Ler até digitar 0, Jogo até acertar
Executar PELO MENOS 1 vez do-while Menu, Validação de senha
Contador crescente/decrescente for 1 a 100, 10 a 1, Pares de 0 a N
Baseado em condição complexa while Enquanto (x>10 e y<5), Algoritmos

✅ Use FOR quando:

  • Sabe o número exato de iterações
  • Tem contador com início e fim claros
  • Percorrer arrays/vetores
  • Gerar sequências numéricas

✅ Use WHILE quando:

  • Condição depende de entrada do usuário
  • Usar valor sentinela (0, -1, "sair")
  • Algoritmos (Euclides, busca)
  • Pode NÃO executar nenhuma vez

✅ Use DO-WHILE quando:

  • Menu de opções
  • Validação de entrada
  • Jogos (jogar pelo menos 1 rodada)
  • Confirmações (perguntar sempre 1x)

⚠️ Evite Loop Infinito!

Sempre garanta que a condição do loop eventualmente se torne falsa. Em while/do-while: modifique a variável da condição DENTRO do loop!

📝 Problema 8: Contar Pares de 1 a N

Leia N. Conte quantos números pares existem de 1 até N.

✅ Solução
inteiro n, contador = 0 leia(n) para (inteiro i = 1; i <= n; i++) { se (i % 2 == 0) { contador++ } } escreva("Pares: ", contador)

📝 Problema 9: Soma de Pares e Ímpares Separados

Leia 10 números. Calcule a soma dos pares e a soma dos ímpares separadamente.

✅ Solução
inteiro numero, somaPares = 0, somaImpares = 0 para (inteiro i = 1; i <= 10; i++) { escreva("Número ", i, ": ") leia(numero) se (numero % 2 == 0) { somaPares += numero } senao { somaImpares += numero } } escreva("Soma pares: ", somaPares, "\n") escreva("Soma ímpares: ", somaImpares, "\n")

📝 Problema 10: Jogo de Adivinhação

Número secreto = 42. Usuário tenta adivinhar. Dê dicas "maior" ou "menor".

✅ Solução
inteiro secreto = 42, palpite, tentativas = 0 escreva("Adivinhe o número (1-100): ") faca { leia(palpite) tentativas++ se (palpite < secreto) { escreva("Muito baixo! Tente: ") } senao se (palpite > secreto) { escreva("Muito alto! Tente: ") } } enquanto (palpite != secreto) escreva("🎉 Acertou em ", tentativas, " tentativas!\n")
