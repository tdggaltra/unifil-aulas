🔁 Revisão: o while e seu limite

Por que precisamos de um laço diferente?

O while pode executar zero vezes. E quando precisamos que o bloco execute ao menos uma vez? 🤔

🔁 while — problema real

// Pedir senha ao usuário enquanto (senha <> "1234") faca escreva("Digite a senha: ") leia(senha) fimenquanto // ❌ Problema: senha nem foi // lida ainda na 1ª verificação! // Precisa inicializar antes...

⚠️ Solução gambiarra

Inicializar senha <- "" antes do laço — funciona, mas é artificioso e confuso.

🔂 do-while — solução elegante

// Pedir senha — versão limpa repita escreva("Digite a senha: ") leia(senha) ate (senha == "1234") // ✅ Lê PRIMEIRO, testa DEPOIS // Natural e direto!

✅ Por que é melhor aqui

Nenhuma inicialização artificial necessária. O bloco executa, lê a senha, e só então verifica se está correta.

🍽️ Analogia: Restaurante

while = ligar antes de ir → se o restaurante estiver fechado, você nem sai de casa.
do-while = ir ao restaurante e, ao chegar, verificar se está aberto. Você sempre vai ao menos uma vez — mesmo que esteja fechado.

⚡ Diferença visual

🟢 Portugol

// Atenção: condição é de PARADA // (para quando for verdadeira) repita instrução_1 instrução_2 atualização ate (condição_de_parada)

🔵 C

// Condição é de CONTINUAÇÃO // (continua enquanto for verdadeira) do { instrução_1; instrução_2; atualização; } while (condição_de_continuação);

🟠 Java

// Igual ao C — condição de CONTINUAÇÃO do { instrução_1; instrução_2; atualização; } while (condição_de_continuação);

⚠️ Atenção: A condição é INVERTIDA no Portugol!

Em C/Java: a condição while(...) é de continuação (continua enquanto verdadeira) — igual ao while comum.
Em Portugol: ate(...) é condição de parada (para quando se torna verdadeira) — o oposto!

✅ Quando condição já é falsa na largada

int i = 10; do { printf("Executa!"); ← executa 1x i++; } while (i <= 5); ← 10≤5 → FALSO // Saída: "Executa!" (1 vez)

Mesmo com condição já falsa, executa uma vez!

⚠️ while com mesma condição

int i = 10; while (i <= 5) { ← 10≤5 → FALSO printf("Nunca!"); i++; } // Saída: (nada) — 0 execuções

O while não executa nenhuma vez!

⚖️ while vs do-while — Comparação Completa

Escolhendo a estrutura certa para cada situação

Critério while (enquanto) do-while (repita-até)
Quando a condição é testada ANTES de executar o bloco DEPOIS de executar o bloco
Mínimo de execuções 0 (pode nunca executar) 1 (sempre executa ao menos uma vez)
Condição no Portugol enquanto (continuar) — de continuação ate (parar) — de parada
Condição em C/Java while (continuar) — de continuação while (continuar) — de continuação (igual!)
Caso de uso típico Processar listas (que podem estar vazias) Menus, pedido de dados, validação de entrada
Exemplo clássico Ler registros de arquivo (pode ser vazio) Exibir menu e ler opção do usuário

✅ Use while quando...

  • O bloco pode precisar executar 0 vezes
  • Você verifica a condição com dados já disponíveis
  • Processar arquivos ou listas que podem estar vazios
  • Iteração sobre coleções

✅ Use do-while quando...

  • O bloco deve executar ao menos uma vez
  • Você precisa ler dados antes de validá-los
  • Menus de sistemas interativos
  • Loop de validação de entrada do usuário

💻 Exemplo 1 — Menu Interativo

O caso de uso mais clássico do do-while

📌 Problema

Crie um menu com 3 opções e a opção 0 para sair. O menu deve aparecer sempre ao menos uma vez e se repetir até o usuário escolher sair.

🟢 Portugol

algoritmo "Menu" var opcao: inteiro inicio repita escreva("=== MENU ===") escreva("1 - Cadastrar") escreva("2 - Consultar") escreva("3 - Relatório") escreva("0 - Sair") escreva("Opção: ") leia(opcao) escolha(opcao) caso 1: escreva("Cadastrando...") caso 2: escreva("Consultando...") caso 3: escreva("Gerando...") caso 0: escreva("Até logo!") outrocaso: escreva("Inválida!") fimescolha ate (opcao == 0) fimalgoritmo

🔵 C

#include <stdio.h> int main() { int opcao; do { printf("\n=== MENU ===\n"); printf("1 - Cadastrar\n"); printf("2 - Consultar\n"); printf("3 - Relatorio\n"); printf("0 - Sair\n"); printf("Opcao: "); scanf("%d", &opcao); switch(opcao) { case 1: printf("Cadastrando\n"); break; case 2: printf("Consultando\n"); break; case 3: printf("Gerando\n"); break; case 0: printf("Ate logo!\n"); break; default:printf("Invalida!\n"); } } while (opcao != 0); return 0; }

🟠 Java

import java.util.Scanner; public class Menu { public static void main (String[] args) { Scanner sc = new Scanner(System.in); int op; do { System.out.println( "\n=== MENU ==="); System.out.println( "1-Cadastrar"); System.out.println( "2-Consultar"); System.out.println( "0-Sair"); System.out.print("Opcao: "); op = sc.nextInt(); switch(op) { case 1: System.out.println( "Cadastrando"); break; case 2: System.out.println( "Consultando"); break; case 0: System.out.println( "Ate logo!"); break; default:System.out.println( "Invalida!"); } } while (op != 0); } }

💻 Exemplo 2 — Validação de Senha com Tentativas

do-while com contador de tentativas e limite máximo

📌 Problema

Peça a senha ao usuário. Ele tem no máximo 3 tentativas. Se acertar, exiba "Acesso liberado". Se esgotar as tentativas, bloqueie o acesso.

🟢 Portugol

algoritmo "Senha" var senha : caractere tentativa: inteiro inicio tentativa <- 0 repita tentativa <- tentativa + 1 escreva("Tentativa ", tentativa, "/3 — Senha: ") leia(senha) se (senha <> "1234") entao escreva("❌ Senha incorreta!") fimse ate (senha == "1234") ou (tentativa == 3) se (senha == "1234") entao escreva("✅ Acesso liberado!") senao escreva("🔒 Conta bloqueada!") fimse fimalgoritmo

🔵 C

#include <stdio.h> #include <string.h> int main() { char senha[20]; int tentativa = 0; do { tentativa++; printf("Tentativa %d/3: ", tentativa); scanf("%s", senha); if(strcmp(senha,"1234")!=0) printf("Incorreta!\n"); } while(strcmp(senha,"1234")!=0 && tentativa < 3); if(strcmp(senha,"1234")==0) printf("Acesso liberado!\n"); else printf("Conta bloqueada!\n"); return 0; }

🔍 Trace com senha "0000", "abc", "1234"

Tent.1: "0000" ≠ "1234" → msg. erro, tentativa=1 → condição: não acertou E tentativa < 3 → repete
Tent.2: "abc" ≠ "1234" → msg. erro, tentativa=2 → repete
Tent.3: "1234" = "1234" → tentativa=3 → condição falsa → sai → Acesso liberado!

💻 Exemplo 3 — Calculadora com Repetição

do-while para permitir múltiplos cálculos na mesma sessão

📌 Problema

Crie uma calculadora que realiza uma operação e pergunta ao final se o usuário deseja fazer outro cálculo (S/N). A calculadora deve rodar ao menos uma vez.

🟢 Portugol

algoritmo "Calculadora" var a, b : real op : inteiro continua: caractere inicio repita escreva("1-+ 2-- 3-* 4-/: ") leia(op) escreva("A: ") leia(a) escreva("B: ") leia(b) escolha(op) caso 1: escreva("= ", a+b) caso 2: escreva("= ", a-b) caso 3: escreva("= ", a*b) caso 4: se (b <> 0) entao escreva("= ", a/b) senao escreva("Erro: div/0") fimse fimescolha escreva("Outro cálculo? (S/N): ") leia(continua) ate (continua == "N") ou (continua == "n") escreva("Obrigado!") fimalgoritmo

🔵 C

#include <stdio.h> int main() { float a, b; int op; char cont; do { printf("1-+ 2-- 3-* 4-/: "); scanf("%d", &op); printf("A: "); scanf("%f", &a); printf("B: "); scanf("%f", &b); switch(op) { case 1: printf("=%.2f\n",a+b);break; case 2: printf("=%.2f\n",a-b);break; case 3: printf("=%.2f\n",a*b);break; case 4: if(b!=0) printf("=%.2f\n",a/b); else printf("Div/0!\n"); break; } printf("Outro? (S/N): "); scanf(" %c", &cont); ← espaço! } while(cont == 'S' || cont == 's'); printf("Obrigado!\n"); return 0; }

✅ Por que do-while aqui?

Uma calculadora sempre precisa executar ao menos uma operação. Só faz sentido perguntar "continuar?" depois do cálculo — nunca antes.

💻 Exemplo 4 — Jogo: Adivinhe o Número

do-while com dicas ao usuário e contagem de tentativas

📌 Problema

O programa "pensa" em um número fixo (ex: 42). O usuário chuta valores. A cada tentativa, o programa diz se o número secreto é maior ou menor. O jogo termina quando acertar. Conte as tentativas.

🟢 Portugol

algoritmo "Adivinha" var secreto, chute, tent: inteiro inicio secreto <- 42 tent <- 0 escreva("=== ADIVINHE O NÚMERO ===") escreva("(entre 1 e 100)") repita tent <- tent + 1 escreva("Tentativa ", tent, ": ") leia(chute) se (chute < secreto) entao escreva("📈 Maior!") senao se (chute > secreto) entao escreva("📉 Menor!") fimse fimse ate (chute == secreto) escreva("🎉 Acertou em ", tent, " tentativas!") fimalgoritmo

🔵 C

#include <stdio.h> int main() { int secreto = 42; int chute, tent = 0; printf("Adivinhe (1-100):\n"); do { tent++; printf("Tent.%d: ", tent); scanf("%d", &chute); if (chute < secreto) printf("Maior!\n"); else if (chute > secreto) printf("Menor!\n"); } while (chute != secreto); printf("Acertou em %d tent.!\n", tent); return 0; }

🔍 Por que do-while e não while?

O jogador sempre precisa fazer ao menos um chute — não faz sentido verificar se acertou antes de chutar! Com do-while, o fluxo é natural: chuta → recebe dica → tenta de novo.

🚨 Erros Comuns com do-while

Armadilhas específicas desse laço — especialmente para quem vem do while

❌ Erro 1: Condição invertida no Portugol

// ERRADO: "ate (continuar)" sai logo repita escreva("Menu...") leia(op) ate (op <> 0) ← sai quando op ≠ 0! // Qualquer opção que não seja 0 // encerra o laço imediatamente! // CORRETO: condição de PARADA ate (op == 0) ← sai quando op = 0

❌ Erro 2: Esquecer ponto e vírgula em C/Java

// ERRADO: falta ; no final! do { printf("Ola\n"); i++; } while (i <= 5) ← sem ; → erro de compilação! // CORRETO } while (i <= 5); ← com ;

❌ Erro 3: Usar do-while onde while é correto

// PROBLEMA: processar arquivo vazio // Arquivo tem 0 registros repita // processa registro (não existe!) leia(reg) ate (semMaisRegistros) // ❌ Processa lixo na 1ª iteração! // CORRETO: use while aqui enquanto (temRegistros) faca leia(reg) fimenquanto

❌ Erro 4: Loop infinito por condição nunca atingida

// ERRADO: usuario pode nunca digitar 0 repita leia(n) soma <- soma + n ate (n == 0) // Se usuario só digitar valores // positivos, trava! // MELHOR: limitar tentativas ou // aceitar qualquer valor ≤ 0 ate (n <= 0)

✅ Regra prática para escolher while ou do-while

Pergunte: "O bloco precisa executar mesmo que a condição já seja falsa de início?"
SIM → use do-while   |   NÃO → use while

O que aprendemos sobre o laço FAÇA-ENQUANTO

Agora você tem dois laços no arsenal: while e do-while. Escolha o certo! 🏆

✅ Estrutura do do-while

  • Bloco executa antes da condição ser testada
  • Garante ao menos uma execução
  • Portugol: repita...ate(parada)
  • C/Java: do{}while(continuação);

✅ Quando usar cada laço

  • while: quando pode executar 0 vezes
  • do-while: quando executa ao menos 1 vez
  • Menus → do-while
  • Validação de entrada → do-while
  • Processar listas/arquivos → while
