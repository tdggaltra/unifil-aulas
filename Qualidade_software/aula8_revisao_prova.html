🎯 Aulas 1 e 2: Qualidade e Custos

Fundamentos e economia da qualidade

📖 Aula 1: Introdução à Qualidade
Definição de Qualidade
ISO 25010: 6 dimensões (Funcionalidade, Desempenho, Compatibilidade, Usabilidade, Confiabilidade, Segurança, Manutenibilidade, Portabilidade)
Produto vs Processo
Qualidade do PRODUTO: o que entregamos ao cliente. Qualidade do PROCESSO: como fazemos (bom processo → bom produto)
Crise do Software
Standish Group: 31% projetos cancelados, 53% atrasados/orçamento, apenas 16% bem-sucedidos
Regra 1-10-100
Prevenir custa R$1, corrigir em desenvolvimento R$10, corrigir em produção R$100
💰 Aula 2: Custos da Qualidade e ROI
Categoria Descrição Exemplos
Prevenção 🛡️ Evitar defeitos Treinamento, planejamento, code review
Avaliação 🔍 Detectar defeitos Testes, inspeções, auditorias
Falhas Internas 🔧 Defeitos pré-entrega Retrabalho, debugging, refatoração
Falhas Externas 💥 Defeitos pós-entrega Suporte, recalls, danos reputação

📊 Fórmula do ROI

ROI = [(Ganho - Investimento) / Investimento] × 100%

Exemplo: Investimento R$200k, economia R$750k
ROI = [(750-200)/200] × 100% = 275%

  • ✓ Entenda a regra 1-10-100
  • ✓ Saiba calcular ROI
  • ✓ Conheça estatísticas da crise do software
  • ✓ Diferencie qualidade de produto vs processo

📋 Aulas 4 e 5: Níveis G e F

Gerenciamento básico e qualidade

📖 Nível G: GPR e GRE

GPR - Gerência de Projetos

Propósito: Estabelecer planos, prover informações de andamento

  • 17 resultados esperados
  • Escopo, WBS, cronograma, riscos
  • Orçamento, recursos, marcos
  • Monitoramento e ações corretivas

GRE - Gerência de Requisitos

Propósito: Gerenciar requisitos, assegurar alinhamento

  • 5 resultados esperados
  • Entendimento com fornecedores
  • Rastreabilidade bidirecional
  • Gerenciamento de mudanças
📖 Nível F: +5 Processos
MED - Medição
Coletar, armazenar, analisar dados de produtos e processos. Métricas: produtividade, qualidade, prazo, custo.
GQA - Garantia da Qualidade
Assegurar aderência de produtos e processos. Auditorias, code review, não conformidades.
GCO - Gerência de Configuração
Controlar versões, mudanças, baselines. Git, SVN, CCB (Change Control Board).
AQU - Aquisição
Gerenciar fornecedores terceiros. RFP, seleção, contrato, monitoramento.
GPP - Gerência de Portfólio
Priorizar projetos estrategicamente. ROI, recursos, alinhamento estratégico.

🎯 Pontos-Chave para a Prova

  • ✓ GPR: 17 resultados, GRE: 5 resultados
  • ✓ Artefatos: Plano Projeto, Baseline Requisitos, Matriz Rastreabilidade
  • ✓ Nível G: capacidade 2 (Gerenciado) para ambos
  • ✓ Nível F: 5 processos (MED, GQA, GCO, AQU, GPP)
  • ✓ Diferença GRE (gerenciar) vs DRE (desenvolver)

🏢 Aula 5: Nível E

Processos organizacionais e RH

Nível E = Padronizar e Reutilizar! 📚
📖 Nível E: 3 Processos Organizacionais
DFP - Definição do Processo Organizacional
Biblioteca de processos reutilizáveis!
• Processos-padrão organização
• Descrições ciclo vida
• Guias adaptação
• Biblioteca ativos (templates)
• Repositório medições
• Padrões ambiente
AMP - Avaliação e Melhoria
Melhoria contínua sistemática!
• Identificar melhorias processos
• Planejar/implementar melhorias
• Confirmar eficácia
• Ciclo PDCA
GRH - Gerência de Recursos Humanos
Gestão estratégica pessoas!
• Matriz competências
• Plano desenvolvimento (PDI)
• Trilhas carreira
• Programa treinamento
• Base conhecimento
Aspecto Antes de E (até F) Com Nível E
Processos Cada projeto cria seus próprios Processos-padrão organizacionais
Reutilização Baixa, cada projeto "reinventa" Alta, biblioteca ativos
Melhoria Por projeto, não sistemática Programa formal (AMP)
Pessoas Informal, ad-hoc Gestão estratégica (GRH)

💡 Benefícios do Nível E

  • ✓ -25% tempo criando processos
  • ✓ +30% consistência entre projetos
  • ✓ 50% mais fácil treinar novos
  • ✓ Melhoria contínua sistemática
  • ✓ Retenção talentos

⚠️ Quando buscar E?

  • ✓ Nível F consolidado (6+ meses)
  • ✓ Múltiplos projetos simultâneos
  • ✓ Muito retrabalho entre projetos
  • ✓ Dificuldade treinar novos
  • ✓ Quer escalar operação

🎯 Pontos-Chave para a Prova

  • ✓ Nível E: 3 processos (DFP, AMP, GRH)
  • ✓ DFP: 6 componentes (processos-padrão, ciclos vida, guias, biblioteca, repositório, padrões)
  • ✓ Grande mudança: processos individuais → organizacionais
  • ✓ AMP: ciclo PDCA de melhoria
  • ✓ GRH: matriz competências, PDI, trilhas

🏗️ Aula 6: Nível D

Engenharia completa do produto

Nível D = DRE + DES + VAL! 🎯
📖 Nível D: 3 Processos de Engenharia
DRE - Desenvolvimento de Requisitos
Engenharia de requisitos!
• Elicitar, analisar, especificar
• Decompor em componentes
• Validar com stakeholders
• Rastreabilidade completa
≠ GRE (que apenas gerencia)
DES - Design e Construção
Do design à integração!
• Design arquitetural + detalhado
• Make-or-buy analysis
• Implementação + verificação
• Integração componentes
Processo técnico completo
VAL - Validação
Produto certo?
• Testes aceitação, UAT
• Beta testing, pilotos
• Validação usuários reais
• Foco: valor cliente
≠ Verificação (DES)
Conceito Definição Como?
Verificação (DES) "Construí CERTO?" Unit tests, code review, inspeções
Validação (VAL) "Construí o CERTO?" UAT, beta, usuários reais
GRE (Nível G) Gerenciar requisitos Rastrear, versionar, mudanças
DRE (Nível D) Desenvolver requisitos Elicitar, analisar, especificar

🎯 DRE: 5 Fases

  • 1. Elicitação: Entrevistas, workshops
  • 2. Análise: Conflitos, priorização
  • 3. Especificação: Documentar claramente
  • 4. Decomposição: Requisitos componentes
  • 5. Validação: Confirmar com cliente

🎯 DES: Ciclo Técnico

  • 1. Design Arquitetural: Estrutura alto nível
  • 2. Design Detalhado: Classes, APIs, BD
  • 3. Implementação: Codificação
  • 4. Verificação: Tests, review
  • 5. Integração: Juntar componentes

🎯 Pontos-Chave para a Prova

  • ✓ Nível D: 3 processos (DRE, DES, VAL)
  • ✓ Diferença Verificação vs Validação
  • ✓ Diferença GRE vs DRE
  • ✓ Nível D = primeiro nível com engenharia técnica
  • ✓ DES cobre design, implementação, verificação, integração

🏆 Aula 7: Níveis C, B e A

A elite da maturidade

C + B + A = Excelência! 🌟
Nível Nome Processos Foco
C Definido +4 = 17 total VER, ITP, GRI, GDE
B Gerenciado Quantitativamente +2 = 19 total GQP (controle estatístico)
A Em Otimização +2 = 21 total IIO, ACP (melhoria contínua)
📖 Nível C: 4 Processos
VER - Verificação
Confirmar que produtos refletem requisitos. Peer review, testes, inspeções formais.
ITP - Integração do Produto
Compor produto a partir de componentes. Estratégia integração, testes integração.
GRI - Gerência de Riscos
Identificar, analisar, tratar riscos proativamente. Matriz riscos, planos mitigação.
GDE - Gerência de Decisões
Decisões formais com critérios objetivos. Análise alternativas, seleção baseada em critérios.

📊 Nível B: Quantitativo!

Grande Mudança: Qualitativo → Quantitativo

  • Antes (até C): "Qualidade boa?" → Opinião
  • B: "Defeitos/KLOC: 2.5 ± 0.3" → Dados!
  • Controle estatístico processos (SPC)
  • Modelos preditivos
  • Decisões baseadas em DADOS

🚀 Nível A: Otimização!

Melhoria Contínua Sistemática

  • IIO: Inovação e Implantação
  • ACP: Análise Causas Problemas
  • Ciclo: Medir → Analisar → Inovar → Implantar
  • Cultura de excelência
  • Apenas ~0.5% empresas BR!

🎯 Pontos-Chave para a Prova

  • ✓ C: VER, ITP, GRI, GDE (completar engenharia)
  • ✓ B: GQP (controle estatístico, quantitativo)
  • ✓ A: IIO, ACP (otimização, análise causas)
  • ✓ Diferença qualitativo (até C) vs quantitativo (B+)
  • ✓ Pouquíssimas empresas alcançam C, B, A (~5% total)
