☀️ O Que É Daily Scrum?

O pulso diário do Sprint

📖 Definição do Scrum Guide

"O propósito do Daily Scrum é inspecionar o progresso em direção ao Sprint Goal e adaptar o Sprint Backlog conforme necessário."

Daily NÃO é reunião de status para gerente! É sincronização do TIME! 👥

🎯 Propósito

  • Inspecionar progresso
  • Adaptar plano
  • Sincronizar time
  • Identificar impedimentos
  • Planejar próximas 24h

👥 Participantes

  • Developers: OBRIGATÓRIO
  • PO: Opcional
  • SM: Opcional
  • Stakeholders: NÃO
  • Gerentes: NÃO

15 min

TIMEBOX SAGRADO! Não negocie!

Daily é PARA o time, PELO time. Não é prestação de contas!

❓ As 3 Perguntas Clássicas

Estrutura tradicional do Daily

3 perguntas simples que mantêm time alinhado! 🎯

1️⃣ O que EU fiz ONTEM?

Foco: Progresso real
BOM: "Completei a API de login, passou em todos os testes"
RUIM: "Estive trabalhando no login" (vago!)

2️⃣ O que EU vou fazer HOJE?

Foco: Plano para próximas 24h
BOM: "Vou integrar login com frontend"
RUIM: "Vou continuar trabalhando" (inespecífico!)

3️⃣ Algum IMPEDIMENTO?

Foco: Bloqueios reais
BOM: "Preciso credenciais do ambiente de testes"
RUIM: "Teclado poderia ser melhor" (não é bloqueador!)

🎯 Dica de Ouro: Foco no Sprint Goal

TODAS as 3 perguntas mencionam "Sprint Goal"!
Se algo não contribui para Sprint Goal, por que estamos fazendo? 🤔

✅ Boas Práticas do Daily

Como fazer Daily eficaz

1️⃣ Timebox de 15 Minutos

Timer visível. Aos 14min, SM avisa "1 minuto!"
Se não terminou, Daily acaba. Conversas extras → parking lot

2️⃣ Mesma Hora, Mesmo Lugar

Bloquear calendário. Criar ritual. NÃO cancelar!
Consistência cria hábito.

3️⃣ Em Pé (Stand-up!)

Sem cadeiras. Mantém energia alta. Incentiva brevidade.

4️⃣ Fale com o Time

Olhe para colegas, não para SM/PO. Daily é para TIME.

5️⃣ Problemas → "Depois do Daily"

Discussões >2min: "vamos falar depois". Interessados ficam, resto trabalha.

Daily bem feito é tão natural que time sente falta quando não acontece!

⚠️ Anti-Padrões do Daily

Armadilhas que destroem o Daily

Reconheça os cheiros ruins! 👃

❌ 1. Daily de 30-60 Minutos

Causa: Resolvendo problemas no Daily
Solução: Timer visível. Parking lot. SM corta discussões.

❌ 2. Status Report para Gerente

Causa: SM/gerente tratando como prestação de contas
Solução: SM explica que Daily é PARA time. Gerentes não participam!

❌ 3. Resolvendo Problemas

Causa: Não usar parking lot
Solução: "Boa pergunta! Vamos conversar após Daily."

❌ 4. Sempre Mesma Ordem

Causa: Vira robótico
Solução: Variar ordem. Usar token. Walk the board.

🚨 Red Flag Máximo

Se Daily ultrapassa 20 minutos, algo está MUITO errado!
Pare tudo. Retro de emergência. Corrija AGORA.

🌐 Daily Scrum Remoto

Daily eficaz para times distribuídos

Daily remoto pode ser TÃO eficaz quanto presencial! 💻

✅ Setup Ideal

  • Zoom/Meet: Link fixo
  • Câmeras ON: Obrigatório!
  • Em pé: Mesmo remoto!
  • Headset: Áudio claro
  • Board compartilhado
  • Timer projetado

⚙️ Ferramentas

  • Geekbot: Daily async Slack
  • Standuply: Bot Teams
  • Miro: Board visual
  • Timer online compartilhado
  • Zoom Reactions: 👍

✅ FAÇA

  • ✓ Câmeras ON sempre
  • ✓ Mute quando não falando
  • ✓ Compartilhar board
  • ✓ Começar NO HORÁRIO

❌ NÃO FAÇA

  • ❌ Câmera desligada
  • ❌ Multitasking (email)
  • ❌ Barulho de fundo
  • ❌ Esperar atrasados
Daily remoto exige mais disciplina. Câmeras ON fazem TODA diferença! 📹
