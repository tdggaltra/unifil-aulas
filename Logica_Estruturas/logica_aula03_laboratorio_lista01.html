Percurso, leitura, soma e média · ★☆☆ Iniciante

01

Par ou Ímpar — Percurso e Classificação (Portugol)

Declare · leia · percorra · classifique cada elemento

★☆☆

Declare um vetor de 8 inteiros em Portugol. Leia os valores com um laço para. Em seguida, percorra o vetor novamente e exiba, para cada elemento, se é par ou ímpar. Use o operador módulo % (resto da divisão).

📥 Entrada: 3 8 5 12 7 4 9 2
📤 Saída esperada:
v[0]=3 → ímpar | v[1]=8 → par | v[2]=5 → ímpar | v[3]=12 → par
v[4]=7 → ímpar | v[5]=4 → par | v[6]=9 → ímpar | v[7]=2 → par
💡 Dica: se (v[i] % 2 == 0) → par; caso contrário → ímpar. O operador % retorna o resto da divisão.
02

Notas da Turma — Soma e Média (C)

Leitura do usuário · acumulador · divisão real

★☆☆

Em C, leia as notas de 5 alunos (floats entre 0.0 e 10.0). Calcule e exiba: (a) a soma total das notas, (b) a média da turma, (c) se a média foi satisfatória (≥ 6.0) ou insatisfatória.

📥 Entrada: 7.5 4.0 8.5 6.0 9.0
📤 Saída esperada:
Soma = 35.00
Média = 7.00
Resultado: satisfatória
💡 Dica: Use float notas[5] e scanf("%f", ¬as[i]). Para a média, faça o cast: media = (float)soma / 5.

03

Temperatura Máxima e Mínima da Semana (C)

Máximo · mínimo · posição · amplitude térmica

★★☆

Leia as temperaturas (floats) registradas em 7 dias da semana. Encontre e exiba: (a) a temperatura máxima e em qual dia ocorreu (1 a 7), (b) a temperatura mínima e em qual dia ocorreu, (c) a amplitude térmica (máx − mín).

📥 Entrada (7 dias): 28.5 31.0 25.5 33.0 29.0 27.5 30.0
📤 Saída esperada:
Máxima: 33.0°C no dia 4
Mínima: 25.5°C no dia 3
Amplitude térmica: 7.5°C

#include <stdio.h>
#define N 7

int main() {
    float temp[N];
    float maxTemp, minTemp, amplitude;
    int   diaMax, diaMin;

    // Leitura
    for (int i = 0; i < N; i++) {
        printf("Temperatura dia %d: ", i+1);
        scanf("%f", &temp[i]);
    }

    // Inicialização — complete com v[0]
    maxTemp = ???;    diaMax = 1;   // ← preencha
    minTemp = ???;    diaMin = 1;   // ← preencha

    // Busca do máximo e mínimo
    for (int i = 1; i < N; i++) {
        if (temp[i] ??? maxTemp) { maxTemp = temp[i]; diaMax = i+1; }
        if (temp[i] ??? minTemp) { minTemp = temp[i]; diaMin = i+1; }
    }

    amplitude = maxTemp - minTemp;

    printf("Máxima: %.1f°C no dia %d\n", maxTemp, diaMax);
    printf("Mínima: %.1f°C no dia %d\n", minTemp, diaMin);
    printf("Amplitude térmica: %.1f°C\n", amplitude);
    return 0;
}
💡 Resposta dos ???: maxTemp = temp[0] · minTemp = temp[0] · comparação máx usa > · comparação mín usa <.

📝 Exercício 04 — Contagem com Condição

Contar elementos que satisfazem critérios · ★★☆ Intermediário

04

Sistema de Notas — Aprovados, Reprovados e Recuperação (C)

Três contadores · condições múltiplas · percentuais

★★☆

Leia as notas de 10 alunos (0.0 a 10.0). Classifique cada aluno como: Aprovado (nota ≥ 7.0), Recuperação (5.0 ≤ nota < 7.0) ou Reprovado (nota < 5.0). Exiba a quantidade de alunos em cada categoria e o percentual de aprovação.

📥 Entrada: 8.0 4.5 7.0 6.5 9.5 3.0 7.5 5.5 8.5 2.0
📤 Saída esperada:
Aprovados: 4 (40.0%)
Recuperação: 3 (30.0%)
Reprovados: 3 (30.0%)
ex04_notas.c — estrutura base 
#include <stdio.h>
#define N 10

int main() {
    float notas[N];
    int aprovados = 0, recuperacao = 0, reprovados = 0;

    for (int i = 0; i < N; i++) {
        printf("Nota do aluno %d: ", i+1);
        scanf("%f", ¬as[i]);

        if      (notas[i] >= 7.0f) aprovados++;
        else if (notas[i] >= 5.0f) recuperacao++;
        else                         reprovados++;
    }

    printf("Aprovados   : %d (%.1f%%)\n", aprovados,   (float)aprovados/N*100);
    printf("Recuperação : %d (%.1f%%)\n", recuperacao, (float)recuperacao/N*100);
    printf("Reprovados  : %d (%.1f%%)\n", reprovados,  (float)reprovados/N*100);
    return 0;
}
💡 Detalhe do %%: Para imprimir o símbolo % com printf, use %% (dois sinais). Um único % inicia um especificador de formato.
💡 else if encadeado: A ordem importa! Teste primeiro ≥ 7, depois ≥ 5. Se testar ≥ 5 primeiro, aprovados (≥7) também entrariam nessa condição.

📝 Exercício 05 — Alunos Acima da Média

Duas passagens pelo vetor · índices dos elementos · ★★☆ Intermediário

05

Quem ficou acima da média? (C)

Duas passagens · exibir nomes por índice · percentual

★★☆

Leia as notas de 8 alunos. Calcule a média. Depois, em uma segunda passagem, exiba quais alunos (pelo número, de 1 a 8) ficaram acima da média e quantos são no total. Exiba também a maior nota acima da média.

📥 Entrada: 6.0 8.5 4.0 7.5 9.0 5.5 8.0 6.5
📤 Saída esperada:
Média da turma: 6.88
Alunos acima da média: 2, 4, 5, 7 → total: 4 (50.0%)
Maior nota entre eles: 9.0
ex05_acima_media.c — esqueleto 
#include <stdio.h>
#define N 8

int main() {
    float notas[N], soma = 0, media;
    int   acima = 0;
    float maiorAcima;

    /* 1ª passagem: leitura e soma */
    for (int i = 0; i < N; i++) {
        scanf("%f", ¬as[i]);
        soma += notas[i];
    }
    media = (float)soma / N;
    printf("Média da turma: %.2f\n", media);

    /* 2ª passagem: quem está acima? */
    maiorAcima = notas[0];   // será ajustado dentro do loop
    printf("Alunos acima da média: ");
    for (int i = 0; i < N; i++) {
        if (notas[i] > media) {
            printf("%d ", i+1);
            acima++;
            if (notas[i] > maiorAcima) maiorAcima = notas[i];
        }
    }
    printf("\nTotal: %d (%.1f%%)\nMaior nota: %.1f\n",
           acima, (float)acima/N*100, maiorAcima);
    return 0;
}

⚠️ Atenção: maiorAcima deve ser inicializada com uma nota que esteja acima da média

No código acima, maiorAcima = notas[0] pode ser problemático se notas[0] estiver abaixo da média. Uma solução mais robusta: inicialize com -1 e atualize somente quando encontrar o primeiro elemento acima da média.

🔥 Exercício 06 — Desafio Combinado

Todas as operações em um único programa · ★★★ Avançado

🔥

Estação Meteorológica Completa (C)

Soma · média · máx · mín · contagem · relatório formatado

★★★

Uma estação meteorológica registra a precipitação diária (mm) durante #define N 30 dias. Escreva um programa em C que:

  1. Leia as 30 precipitações (floats ≥ 0)
  2. Calcule: precipitação total e média diária
  3. Encontre: dia de maior precipitação (dia 1–30) e o valor
  4. Conte: quantos dias tiveram precipitação zero (sem chuva)
  5. Conte: quantos dias tiveram precipitação acima da média
  6. Exiba um relatório formatado com todos os resultados
📥 Teste com: 0 5.5 0 12.3 0 8.7 0 3.2 0 15.0 0 6.8 0 9.1 0 4.4 0 11.2 0 7.6 0 0 2.3 0 14.5 0 5.9 0 8.3 0
📤 Saída esperada (valores aproximados):
Precipitação total: 124.8 mm | Média diária: 4.16 mm
Maior precipitação: 15.0 mm (dia 10)
Dias sem chuva: 15 (50.0%) | Dias acima da média: 10 (33.3%)
💡 Estratégia: Use 2 passagens. 1ª: leia + soma + conta zeros + acha máx. 2ª: conta acima da média (precisa da média calculada).
💡 Zeros: if (prec[i] == 0.0f) para floats pode ter imprecisão. Prefira if (prec[i] < 0.001f) para detectar "praticamente zero".
💡 Relatório: Use printf com alinhamento. Ex: "%-25s %6.2f mm\n" para colunas alinhadas à esquerda e à direita.

🐛 Erros Comuns — Diagnóstico e Correção

Os bugs mais frequentes de vetores e como corrigir

❌ Erro 1 — Esqueceu o &
scanf("%d", v[i]); // ERRADO
scanf("%d", &v[i]); // CERTO

Sem &, scanf recebe o valor do elemento como endereço — crash ou corrupção de memória.

❌ Erro 2 — Divisão inteira
media = soma / N; // se ambos int → trunca!
media = (float)soma / N; // correto

40/6 em inteiros = 6 (não 6.67). Sempre faça cast quando a média deva ser real.

❌ Erro 3 — Índice fora dos limites
int v[5]; for(i=0; i<=5; i++) // ← i=5 é inválido! v[i] = 0;
for(i=0; i<5; i++) // correto: i < N

Use < (estritamente menor), nunca <= no laço de vetor — o último índice é N-1.

❌ Erro 4 — Acumulador sem inicializar
int soma; // lixo de memória! soma += v[0];
int soma = 0; // sempre inicialize

Em C, variáveis locais não são zeradas automaticamente — sempre inicialize acumuladores e contadores.

❌ Erro 5 — Máx inicializado com 0
maior = 0; // errado se vetor tem negativos // ou se todos os valores são negativos
maior = v[0]; // correto: usa primeiro elemento

Inicializar máx com 0 falha quando todos os valores são negativos. Use sempre v[0].

✅ Dica geral de debug: Se o resultado está errado, adicione printf dentro do laço para ver o valor de cada variável a cada iteração. É o trace manual no código!
printf("i=%d v[i]=%d soma=%d\n", i, v[i], soma);

✅ Gabarito — Exercício 01 (Portugol)

Par ou Ímpar — solução completa comentada

ex01_par_impar.por — gabarito completo 
programa {
    funcao inicio() {
        // Declaração do vetor e variáveis auxiliares
        inteiro N = 8
        inteiro v[8]
        inteiro pares = 0, impares = 0

        // Leitura dos 8 valores
        escreva("Digite 8 números inteiros:\n")
        para (inteiro i = 0; i < N; i++) {
            escreva("v[" + i + "]: ")
            leia(v[i])
        }

        // Classificação: par ou ímpar
        escreva("\nClassificação:\n")
        para (inteiro i = 0; i < N; i++) {
            se (v[i] % 2 == 0) {
                escreva("v[" + i + "]=" + v[i] + " → PAR")
                pares = pares + 1
            } senao {
                escreva("v[" + i + "]=" + v[i] + " → ÍMPAR")
                impares = impares + 1
            }
        }

        // Resumo
        escreva("\nTotal pares: " + pares)
        escreva("Total ímpares: " + impares)
    }
}

✅ Pontos de Destaque

  • Dois laços separados: 1º lê, 2º classifica — mais claro
  • Operador % (módulo) retorna o resto da divisão
  • Contadores de par/ímpar como bônus de aprendizado
  • Poderia ser um laço só — tente refatorar!

🔄 Versão compacta (um laço)

  • É possível ler e classificar no mesmo laço:
  • leia(v[i]) e em seguida o se/senao
  • Elimina a necessidade de um segundo percurso
  • Funciona porque a classificação não depende de outros elementos

✏️ Exercícios de Fixação

Questões objetivas sobre o conteúdo do laboratório

1. Em C, um programa calcula a média de 10 notas com float media = soma / 10; onde soma é int. O resultado de soma é 87. Qual é o valor de media?

2. No Exercício 05, por que são necessárias duas passagens pelo vetor para contar quem ficou acima da média?

3. Um aluno escreveu maior = 0; para inicializar o máximo antes do laço. Em qual situação isso causaria resultado errado?

4. 💬 Reflexão — Analise e responda:

No Exercício 04 (classificação de notas), a ordem dos testes no else if importa? O que aconteceria se você testasse nota ≥ 5.0 antes de nota ≥ 7.0? Justifique com um exemplo.

💡 Gabarito: Sim, a ordem importa! Se testar ≥ 5.0 primeiro, um aluno com nota 8.0 (que deveria ir para aprovado) entraria na condição de recuperação, pois 8.0 ≥ 5.0 é verdadeiro. O else if encadeado para quando encontra a primeira condição verdadeira — por isso sempre teste do mais restritivo (≥ 7.0) para o menos restritivo (≥ 5.0).
