🎯 Por que strings agora?

Você já domina vetores de int e float. Uma string em C é simplesmente um vetor de char com uma regra especial: o último byte válido é seguido pelo terminador nulo \0. Todo o conhecimento de percurso e índices se aplica diretamente!

📌 O tipo char em C

  • Ocupa 1 byte (8 bits) de memória
  • Armazena um inteiro de 0 a 255
  • Interpretado como código ASCII
  • 'A'=65 · 'a'=97 · '0'=48 · ' '=32
  • '\0'=0 — o terminador obrigatório de toda string
  • Aritmética funciona: 'A'+1 == 'B'

🎯 Objetivo desta aula

Entender o modelo de memória de strings em C (char[] + \0), ler e escrever strings e usar as quatro funções fundamentais de <string.h>: strlen · strcpy · strcmp · strcat.

🧱 O que é uma String em C?

Vetor de char com terminador nulo obrigatório

Em C não existe tipo string. Uma string é um vetor de char onde o último caractere válido é seguido pelo byte '\0' (nulo, valor 0).

⚠️ Regras Cruciais

  • Tamanho mínimo = comprimento + 1 (para o \0)
  • "Londrina" tem 8 chars → vetor precisa de mínimo 9 posições
  • Funções de string dependem do \0 para saber onde termina
  • Sem \0 → comportamento indefinido — lê além do vetor!
  • Literal = "texto" insere o \0 automaticamente

📥📤 Leitura e Escrita de Strings

scanf · fgets · printf · puts — diferenças que importam

📥 Lendo strings em C

  • scanf("%s", s) — lê até 1º espaço · sem & (s já é endereço)
  • "Maria Silva" com scanf → lê só "Maria"
  • fgets(s, 50, stdin) — lê linha inteira com espaços ✅
  • fgets inclui \n no final — remova se necessário
  • Nunca use gets() — depreciado, buffer overflow!

📤 Escrevendo strings em C

  • printf("%s", s) — imprime até o \0
  • puts(s) — imprime a string + \n automático
  • printf("%c", s[i]) — imprime char por índice
  • putchar(c) — imprime um único char

⚠️ Por que sem & no scanf de string?

O nome do vetor s já é o endereço do primeiro elemento. Para um elemento individual use &v[i], mas para a string inteira use apenas s.

strlen() — como funciona

  • Percorre do índice 0 até encontrar \0
  • Retorna o número de chars antes do \0
  • Tipo de retorno: size_t (inteiro sem sinal)
  • Requer #include <string.h>
  • Nunca inclui o \0 na contagem
strlen + percurso manual em C
#include <stdio.h>
#include <string.h>
int main() {
  char s[50] = "UniFil";
  int  tam = strlen(s);  // tam = 6
  printf("Comprimento: %d\n", tam);

  /* Padrão 1: percurso com strlen */
  printf("Chars: ");
  for(int i=0; i<tam; i++)
    printf("%c ", s[i]);     // %c imprime char
  printf("\n");

  /* Padrão 2: percurso até '\0' */
  for(int i=0; s[i]!='\0'; i++)
    printf("%c", s[i]);

  /* Exemplo: contar vogais */
  int vogais = 0;
  for(int i=0; i<tam; i++) {
    char c = s[i];
    if(c=='a'||c=='e'||c=='i'||c=='o'||c=='u'||
       c=='A'||c=='E'||c=='I'||c=='O'||c=='U')
      vogais++;
  }
  printf("Vogais: %d\n", vogais);  // 3 em "UniFil"
  return 0;
}

💡 Dois padrões de percurso

Com strlen: for(i=0; i<strlen(s); i++) — mais legível.
Até \0: for(i=0; s[i]!='\0'; i++) — mais eficiente (não chama strlen repetidamente).

🐛 5 Erros Clássicos com Strings em C

Os bugs mais perigosos — lado a lado errado × correto

❌ Erro 1 — Comparar com ==

if (s1 == s2) // ERRADO!
// Compara ENDEREÇOS,
// não o conteúdo!
if (strcmp(s1,s2)==0) // ✅
// Compara conteúdo
// char a char

Java: use .equals(), nunca ==.

❌ Erro 2 — Atribuir com =

char s[20] = "Alice";
s = "Bob"; // ERRO de
// compilação!
strcpy(s, "Bob"); // ✅
// Copia o conteúdo

❌ Erro 3 — Buffer pequeno

char dst[5];
strcpy(dst,"Londrina");
// 8+1=9 bytes > 5
// OVERFLOW! 💥
char dst[50]; // ✅
strncpy(dst,"Londrina",49);
dst[49] = '\0';

Aloque com folga — mínimo strlen+1.

❌ Erro 4 — & no scanf

scanf("%s", &s); // ambíguo
// &s é endereço
// do ponteiro, não do vetor
scanf("%s", s); // ✅
// s já é o endereço
// do 1º elemento

❌ Erro 5 — \n do fgets

fgets(s, 50, stdin);
// s = "Alice\n\0"
strcmp(s,"Alice"); // ≠0!
fgets(s, 50, stdin); // ✅
int n = strlen(s);
if(s[n-1]=='\n')
s[n-1] = '\0';

💡 Regra de ouro

1) Aloque com folga · 2) Use strncpy/strncat · 3) Compare com strcmp · 4) Remova \n do fgets.
