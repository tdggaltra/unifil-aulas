💡 O problema que esta disciplina resolve

Você já sabe guardar uma nota em float nota;. Mas e se precisar de 50 notas? Declarar nota1, nota2, ... nota50 é inviável. Precisamos de uma estrutura que guarde múltiplos valores do mesmo tipo de forma organizada e acessível por índice — o arranjo!

🤔 O Problema Motivador

Por que variáveis simples não são suficientes?

Preciso calcular a média de 6 alunos e mostrar quem ficou acima dela. Com variáveis simples, sou obrigado a percorrer os dados duas vezes — e não consigo generalizar para N alunos.

❌ Solução ingênua — variáveis separadas

// Calcular média de 6 notas float n1, n2, n3, n4, n5, n6; scanf("%f%f%f%f%f%f", &n1,&n2,&n3,&n4,&n5,&n6); float media = (n1+n2+n3+n4+n5+n6) / 6; // Para mostrar quem está acima: if(n1 > media) printf("n1 acima\n"); if(n2 > media) printf("n2 acima\n"); // ... repete até n6 — impossível para N grande!
Problemas: código não generaliza para N diferente · duplicação excessiva · impossível usar laços para processar · aumentar o tamanho exige reescrever tudo

✅ Solução com arranjo — escalável

// Funciona para qualquer N! #define N 6 float notas[N]; float soma = 0; for(int i=0; i<N; i++) { scanf("%f", ¬as[i]); soma += notas[i]; } float media = soma / N; for(int i=0; i<N; i++) if(notas[i] > media) printf("Aluno %d acima\n", i+1);
Vantagens: para mudar para 100 alunos: troca apenas o #define N 100 · usa laços · código genérico e reutilizável · fácil de ler e manter

📦 Conceito de Arranjo

Definição, propriedades e modelo mental correto

Um arranjo é uma coleção de variáveis do mesmo tipo, armazenadas em posições de memória contíguas e acessíveis por um índice inteiro.

Modelo de memória — vetor notas[6] com valores 7.5, 8.0, 5.5, 9.0, 6.5, 7.0:

[0]
[1]
[2]
[3]
[4]
[5]
notas[ ]
7.51000h
8.01004h
5.51008h
9.0100Ch
6.51010h
7.01014h
← posições de memória contíguas · cada float ocupa 4 bytes · endereço = base + índice×4 →

📌 Propriedades Fundamentais

  • Tipo único: todos os elementos têm o mesmo tipo
  • Tamanho fixo: definido na declaração (estático)
  • Índice base 0: primeiro elemento = índice 0
  • Memória contígua: elementos seguidos na RAM
  • Acesso O(1): acesso direto por índice em tempo constante

🔢 Dimensões de Arranjos

  • 1D — Vetor: uma fileira de elementos v[i]
  • 2D — Matriz: tabela de linhas × colunas m[i][j]
  • 3D+: possíveis, mas raros na prática
  • Unidade I: vetores (1D)
  • Unidade II: matrizes (2D)

✅ Vantagens dos Arranjos

  • Acesso aleatório instantâneo (índice → posição)
  • Iteração com laços — código genérico
  • Uso eficiente de memória (sem overhead)
  • Base para busca e ordenação clássicas
  • Foundation de listas, pilhas e filas

🎨 Arranjo — Analogias para Fixar

Modelos mentais que tornam o conceito concreto e intuitivo

🚂 Trem com Vagões

Um trem tem vagões numerados 0, 1, 2, 3… Cada vagão carrega uma carga do mesmo tipo. Para acessar o vagão 3, você vai diretamente ao nº 3 — não precisa passar pelos anteriores.

notas[3] → vagão 3 → valor 9.0

🏢 Prédio com Apartamentos

Um prédio tem apartamentos numerados sequencialmente. Todos do mesmo tipo (residencial). Você acessa direto o apto 307 sem visitar os outros. O número É o endereço.

v[307] → apartamento 307

📦 Estante de Caixas

Uma estante com N caixas enumeradas de 0 a N-1. Todas do mesmo tamanho e tipo. Você abre a caixa pelo número, coloca ou retira o item. Simples e direto.

notas[i] = 8.5; // caixa i ← 8.5

⚠️ Armadilha Clássica — Índice Fora dos Limites

Um vetor int v[5]; tem índices válidos de 0 a 4. Acessar v[5] ou v[-1] é comportamento indefinido em C — pode corromper memória ou causar crash. Sempre verifique se 0 ≤ índice < tamanho!

🖥️ Declarando Vetores nas 3 Linguagens

Portugol · C · Java — sintaxes lado a lado

// Portugol — declaração de vetor programa { funcao inicio() { // vetor de 5 inteiros inteiro notas[5] // atribuição por índice (começa em 0!) notas[0] = 7 notas[1] = 9 notas[2] = 5 notas[3] = 8 notas[4] = 6 // leitura com laço para (inteiro i = 0; i < 5; i++) { escreva("nota[" + i + "] = " + notas[i]) } } }
/* Linguagem C — declaração de vetor */ #include <stdio.h> int main() { // Declaração: tipo nome[tamanho] int notas[5]; // Inicialização direta (literal) int pares[] = {2, 4, 6, 8, 10}; // tamanho inferido = 5 // Inicialização parcial (resto = 0) float medias[3] = {7.5, 8.0}; // medias[2] = 0.0 // Percurso com for for (int i = 0; i < 5; i++) { printf("pares[%d] = %d\n", i, pares[i]); } return 0; }
// Java — declaração de array public class ExemploArray { public static void main(String[] args) { // Forma 1: declarar e alocar separado int[] notas; notas = new int[5]; // alocação dinâmica! // Forma 2: declarar e alocar junto int[] pares = new int[5]; // Forma 3: inicialização literal int[] primos = {2, 3, 5, 7, 11}; // Propriedade length — tamanho do array System.out.println(primos.length); // 5 // Percurso com for clássico for (int i = 0; i < primos.length; i++) System.out.println(primos[i]); } }

🟣 Portugol

tipo nome[tamanho]
Índice base 0. Sem new. Sem inicialização automática.

🔵 C

tipo nome[tamanho]
Iniciação com = {}. Tamanho constante em compile-time. Sem checagem de limites!

🟡 Java

tipo[] nome = new tipo[N]
Alocação com new. Propriedade .length. Lança ArrayIndexOutOfBoundsException se índice inválido.

📥📤 Leitura e Escrita de Vetores com Laços

O padrão fundamental: for (i=0; i<N; i++)

🔑 O Padrão de Percurso

Todo vetor de tamanho N tem índices válidos de 0 a N-1. O laço padrão começa em 0, termina antes de N e incrementa de 1 em 1. Esse padrão se repete em toda operação com vetores.

// Lê 5 notas e calcula a soma programa { funcao inicio() { inteiro N = 5 real notas[5], soma = 0 // LEITURA do vetor para (inteiro i = 0; i < N; i++) { escreva("Digite nota " + (i+1) + ": ") leia(notas[i]) soma = soma + notas[i] } // ESCRITA do vetor escreva("Notas digitadas:") para (inteiro i = 0; i < N; i++) { escreva(" notas[" + i + escreva("Média = " + (soma / N)) } }
#include <stdio.h> #define N 5 int main() { float notas[N]; float soma = 0; /* LEITURA do vetor */ for (int i = 0; i < N; i++) { printf("Digite nota %d: ", i+1); scanf("%f", ¬as[i]); // endereço do elemento! soma += notas[i]; } /* ESCRITA do vetor */ printf("\nNotas digitadas:\n"); for (int i = 0; i < N; i++) printf(" notas[%d] = %.1f\n", i, notas[i]); printf("Media = %.2f\n", soma / N); return 0; }
import java.util.Scanner; public class LeituraVetor { public static void main(String[] args) { Scanner sc = new Scanner(System.in); final int N = 5; float[] notas = new float[N]; float soma = 0; // LEITURA for (int i = 0; i < N; i++) { System.out.print("Digite nota " + (i+1) + ": "); notas[i] = sc.nextFloat(); soma += notas[i]; } // ESCRITA System.out.println("\nNotas digitadas:"); for (int i = 0; i < N; i++) System.out.printf(" notas[%d] = %.1f%n", i, notas[i]); System.out.printf("Media = %.2f%n", soma / N); } }

⚡ Destaque em C: scanf("%f", ¬as[i])

O & (endereço-de) antes de notas[i] é obrigatório no scanf. Em C, notas[i] é o valor; ¬as[i] é o endereço de memória onde o valor deve ser gravado. Esquecer o & causa comportamento indefinido!

💡 Conclusão do Trace

O laço executou exatamente 5 vezes (i = 0,1,2,3,4). A condição i < 5 foi verdadeira 5 vezes e falsa na 6ª verificação (i=5), encerrando o laço. A soma acumulou 35.0 e a média resultou em 7.0.
