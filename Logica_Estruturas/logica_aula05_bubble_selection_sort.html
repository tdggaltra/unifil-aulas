🎯 Por que aprender a ordenar?

Ordenar é pré-requisito para busca binária — que é 50.000× mais rápida que a linear em vetores grandes. Além disso: exibição de listas ordenadas, geração de relatórios, eliminação de duplicatas e algoritmos mais avançados (merge sort, heap sort) todos dependem dessa base.

🫧 Bubble Sort

Compara pares adjacentes e troca

O maior elemento "borbulha" para o final a cada passagem. Fácil de entender visualmente. O(n²) comparações.

🎯 Selection Sort

Seleciona o mínimo e coloca no lugar

A cada passagem, encontra o menor elemento restante e o posiciona. Menos trocas que o Bubble. O(n²) comparações.

🫧 Bubble Sort — A Bolha que Sobe

Cada passagem leva o maior elemento restante para sua posição final

💧 Analogia: Bolha na Água

Imagine bolhas de ar em um aquário. As maiores sobem mais rápido e chegam ao topo primeiro. No Bubble Sort: a cada passagem pelo vetor, o maior elemento ainda não ordenado "borbulha" para sua posição final (o fim do trecho não ordenado), através de comparações e trocas de pares adjacentes.

🫧 Bubble Sort — Pseudocódigo e Implementação

Dois laços aninhados + otimização com flag de troca

📐 Pseudocódigo — Bubble Sort
// Ordena v[0..N-1] em ordem crescente
PARA pass DE 0 ATÉ N-2 FAÇA
    // após 'pass' passagens, os últimos
    // 'pass' elementos já estão ordenados
    PARA j DE 0 ATÉ N-2-pass FAÇA
        SE v[j] > v[j+1] ENTÃO
            temp    ← v[j]
            v[j]    ← v[j+1]
            v[j+1] ← temp
        FIM-SE
    FIM-PARA
FIM-PARA

💡 Otimização com flag

  • Se uma passagem completa não fez nenhuma troca, o vetor já está ordenado
  • Use variável trocou (bool): inicia falso, vira verdadeiro a cada troca
  • Se após a passagem trocou == falso → encerra
  • No melhor caso (já ordenado): O(n) comparações!
para (inteiro pass = 0; pass < N-1; pass++) { logico trocou = falso para (inteiro j = 0; j < N-1-pass; j++) { se (v[j] > v[j+1]) { inteiro temp = v[j] v[j] = v[j+1] v[j+1] = temp trocou = verdadeiro } } se (!trocou) pare // já ordenado! }
#include <stdio.h> #define N 6 void bubbleSort(int v[], int n) { int temp, trocou; for (int pass = 0; pass < n-1; pass++) { trocou = 0; for (int j = 0; j < n-1-pass; j++) { if (v[j] > v[j+1]) { temp = v[j]; v[j] = v[j+1]; v[j+1] = temp; trocou = 1; } } if (!trocou) break; // otimização } } int main() { int v[N] = {5,3,8,1,9,2}; bubbleSort(v, N); for(int i=0;i<N;i++) printf("%d ",v[i]); return 0; }
static void bubbleSort(int[] v) { boolean trocou; for (int pass = 0; pass < v.length-1; pass++) { trocou = false; for (int j = 0; j < v.length-1-pass; j++) { if (v[j] > v[j+1]) { int temp = v[j]; v[j] = v[j+1]; v[j+1] = temp; trocou = true; } } if (!trocou) break; } }

🎯 Selection Sort — Selecionar e Posicionar

A cada rodada, seleciona o menor e o coloca no lugar certo

🃏 Analogia: Organizar Cartas na Mão

Imagine organizar cartas do baralho: você vasculha todas e coloca a menor na posição 0, depois vasculha as restantes e coloca a menor na posição 1, e assim por diante. A diferença do Bubble: o Selection faz no máximo N-1 trocas no total (uma por passagem), enquanto o Bubble pode fazer muitas mais.

🎯 Selection Sort — Pseudocódigo e Implementação

Encontrar o mínimo em cada subvetor e trocar com a posição correta

📐 Pseudocódigo — Selection Sort
// Ordena v[0..N-1] em ordem crescente
PARA i DE 0 ATÉ N-2 FAÇA
    // v[0..i-1] já está ordenado
    posMin ← i   // assume i como mínimo

    PARA j DE i+1 ATÉ N-1 FAÇA
        SE v[j] < v[posMin] ENTÃO
            posMin ← j   // novo mínimo
        FIM-SE
    FIM-PARA

    // troca v[i] com o mínimo encontrado
    SE posMin ≠ i ENTÃO
        temp       ← v[i]
        v[i]       ← v[posMin]
        v[posMin] ← temp
    FIM-SE
FIM-PARA

💡 Vantagem do Selection Sort

  • Realiza no máximo N-1 trocas totais
  • Útil quando a operação de troca é custosa (ex: objetos grandes)
  • A verificação posMin ≠ i evita troca desnecessária consigo mesmo
para (inteiro i = 0; i < N-1; i++) { inteiro posMin = i // assume i como mínimo para (inteiro j = i+1; j < N; j++) { se (v[j] < v[posMin]) { posMin = j // novo mínimo } } se (posMin != i) { // troca se necessário inteiro temp = v[i] v[i] = v[posMin] v[posMin] = temp } }
#include <stdio.h> #define N 6 void selectionSort(int v[], int n) { int posMin, temp; for (int i = 0; i < n-1; i++) { posMin = i; for (int j = i+1; j < n; j++) if (v[j] < v[posMin]) posMin = j; if (posMin != i) { temp = v[i]; v[i] = v[posMin]; v[posMin] = temp; } } } int main() { int v[N] = {5,3,8,1,9,2}; selectionSort(v, N); for(int i=0;i<N;i++) printf("%d ",v[i]); return 0; }
static void selectionSort(int[] v) { for (int i = 0; i < v.length-1; i++) { int posMin = i; for (int j = i+1; j < v.length; j++) if (v[j] < v[posMin]) posMin = j; if (posMin != i) { int temp = v[i]; v[i] = v[posMin]; v[posMin] = temp; } } }

🔄 Padrão de Troca (swap)

A troca de dois elementos sempre usa uma variável auxiliar temp: salva v[a] em temp, copia v[b] em v[a], restaura temp em v[b]. Sem temp o valor é perdido!

⚖️ Bubble Sort × Selection Sort

Semelhanças, diferenças e quando usar cada um

Critério 🫧 Bubble Sort 🎯 Selection Sort
Estratégia Compara pares adjacentes e troca se fora de ordem Encontra o mínimo do subvetor e o coloca no início
Comparações (pior caso) O(n²) — n(n-1)/2 comparações O(n²) — n(n-1)/2 comparações (sempre)
Trocas (pior caso) O(n²) — pode trocar muito O(n) — no máximo N-1 trocas
Melhor caso O(n) — com flag de troca (já ordenado) O(n²) — sempre percorre tudo
Estável? Sim — mantém ordem relativa de iguais Não — pode inverter iguais ao trocar
Memória extra O(1) — apenas variável temp O(1) — apenas temp e posMin
Melhor uso Vetores quase ordenados (otimizado com flag) Quando trocas são custosas (minimiza movimentações)

📐 Contar comparações: N=6

  • Passagem 0: 5 comparações (j = 0..4)
  • Passagem 1: 4 comparações (j = 0..3)
  • Passagem 2: 3 · Passagem 3: 2 · Passagem 4: 1
  • Total: 5+4+3+2+1 = 15 = n(n-1)/2
  • Para N=100 → 4.950 comparações

🎓 Na prática

Para vetores pequenos (N < 50), ambos funcionam bem. Para vetores grandes, use algoritmos O(n log n) como Merge Sort ou Quick Sort (conteúdo de disciplinas avançadas). A biblioteca padrão do C (qsort()) e Java (Arrays.sort()) usam Dual-Pivot Quicksort.

🔃 Variações: Decrescente e Floats

Adaptar o algoritmo para outros tipos e ordens

Para ordenar em ordem decrescente: troque o operador > por < no Bubble Sort, e < por > no Selection Sort. Só isso!
/* Bubble Sort DECRESCENTE — só muda o sinal! */ void bubbleSortDesc(int v[], int n) { int temp, trocou; for(int p=0; p<n-1; p++) { trocou = 0; for(int j=0; j<n-1-p; j++) { if(v[j] < v[j+1]) { // < em vez de > temp=v[j]; v[j]=v[j+1]; v[j+1]=temp; trocou=1; } } if(!trocou) break; } } /* Selection Sort para FLOATS — mesmo algoritmo! */ void selectionSortF(float v[], int n) { float temp; int posMin; for(int i=0; i<n-1; i++) { posMin = i; for(int j=i+1; j<n; j++) if(v[j] < v[posMin]) posMin = j; if(posMin != i) { temp=v[i]; v[i]=v[posMin]; v[posMin]=temp; } } }
import java.util.Arrays; // Forma MAIS RÁPIDA em Java — use sempre em produção! int[] v = {5,3,8,1,9,2}; Arrays.sort(v); // O(n log n) — muito mais rápido! // Para decrescente com Integer[] (não int[]) Integer[] v2 = {5,3,8,1,9,2}; Arrays.sort(v2, Collections.reverseOrder()); // Nosso Bubble Sort manual (para aprender o algoritmo) static void bubbleSortDesc(int[] v) { boolean trocou; for(int p=0; p<v.length-1; p++) { trocou = false; for(int j=0; j<v.length-1-p; j++) if(v[j] < v[j+1]) { // < em vez de > int t=v[j]; v[j]=v[j+1]; v[j+1]=t; trocou = true; } if(!trocou) break; } }

🔃 Decrescente

  • Bubble: troca > por <
  • Selection: troca < por > e renomeia posMin para posMax
  • Só o operador de comparação muda!

🔢 Outros tipos

  • Floats: troque int por float e %d por %.2f
  • Chars: compare diretamente — char é int em C
  • Strings: use strcmp() na comparação

⚡ Ordem de uso

  • Aprendizado: implemente manual (como aqui)
  • Produção: use qsort() (C) ou Arrays.sort() (Java)
  • Na prova: implemente manual!

🎯 Resumo e Próxima Aula

Dois algoritmos de ordenação O(n²) — cada um com seu ponto forte

Agora você sabe como colocar dados em ordem — e pode combinar ordenação + busca binária para criar sistemas eficientes! 🫧🎯

✅ Bubble Sort — A Bolha que Sobe

Dois laços aninhados, compara pares adjacentes, troca com temp. Otimização com flag: se não houve troca, encerra. Melhor caso O(n).

✅ Selection Sort — Selecionar e Posicionar

Encontra o posMin do subvetor restante e o troca com a posição i. Máximo N-1 trocas totais. Sem otimização de melhor caso.

✅ Padrão de Troca (swap)

temp = v[a]; v[a] = v[b]; v[b] = temp; — sempre use variável auxiliar. Válido para qualquer tipo.

✅ Ordem Decrescente

Troque o operador de comparação: >↔<. Para vetores de outro tipo, ajuste o tipo da variável temp.

✅ Conexão com Busca Binária

Ordene com Selection/Bubble Sort → aplique busca binária. O custo de ordenar O(n²) vale a pena se forem feitas muitas buscas O(log n).
