🎯 Por que estudar casos especiais?

Reconhecer que uma matriz é simétrica, triangular ou identidade permite economizar memória (armazenar só metade), acelerar algoritmos (pular zeros), e explorar propriedades matemáticas como A×I = A. São padrões que aparecem constantemente em sistemas reais.

📋 O que vemos hoje

  • Nula: todos os elementos são 0
  • Identidade: diagonal = 1, resto = 0
  • Simétrica: A[i][j] == A[j][i]
  • Antissimétrica: A[i][j] == −A[j][i]
  • Triangular Superior: zeros abaixo da diagonal
  • Triangular Inferior: zeros acima da diagonal

🎯 Objetivo desta aula

Identificar visualmente cada tipo especial de matriz, entender sua propriedade definidora e implementar funções de verificação em C que retornam 1 (verdadeiro) ou 0 (falso) — reutilizáveis em qualquer programa.

ehNula() em C
/* Verifica se m[L][C] é nula */
int ehNula(int m[3][3], int l, int c) {
    for(int i=0;i<l;i++)
        for(int j=0;j<c;j++)
            if(m[i][j] != 0) return 0;
    return 1; // todos zero
}

ehIdentidade() + gerarIdentidade() em C
/* Verifica se m[N][N] é identidade */
int ehIdentidade(int m[][3], int n) {
    for(int i=0;i<n;i++)
        for(int j=0;j<n;j++) {
            int esperado = (i==j) ? 1 : 0;
            if(m[i][j] != esperado) return 0;
        }
    return 1;
}

/* Gera a identidade N×N */
void gerarIdentidade(int m[][3], int n) {
    for(int i=0;i<n;i++)
        for(int j=0;j<n;j++)
            m[i][j] = (i==j) ? 1 : 0;
}

💡 O operador ternário (i==j) ? 1 : 0

Equivale a: se i==j então 1, senão 0. Compacto e elegante para preencher a identidade em uma linha. O mesmo padrão serve para qualquer matriz diagonal!

Propriedades definidas pela relação entre A[i][j] e A[j][i]

ehSimetrica() em C
int ehSimetrica(int m[][3], int n) {
    for(int i=0;i<n;i++)
        for(int j=i+1;j<n;j++) // só acima da diag
            if(m[i][j] != m[j][i]) return 0;

}
// j começa em i+1: não verifica diagonal
// nem repete comparações já feitas

ehAntissimetrica() em C
int ehAntissimetrica(int m[][3], int n) {
    for(int i=0;i<n;i++)
        for(int j=0;j<n;j++)
            if(m[i][j] != -m[j][i]) return 0;
    return 1;
}
// Verifica TODOS os pares, inclusive diag
// Diagonal: m[i][i] != -m[i][i] se ≠ 0

💡 Otimização da função simétrica: j começa em i+1

Para verificar simetria só precisamos checar os elementos acima da diagonal: se A[i][j]==A[j][i] para todo j>i, a matriz é simétrica. A diagonal (i==j) é automaticamente simétrica (A[i][i]==A[i][i]). Isso reduz as comparações de n² para n(n-1)/2.

△ Matrizes Triangulares

Superior · Inferior · Diagonal — zeros em posições previsíveis

Uma matriz triangular tem zeros em um triângulo inteiro — acima ou abaixo da diagonal. São a base do método de eliminação de Gauss e de algoritmos O(n²) para sistemas lineares.

🔗 Relação entre os tipos

  • Toda matriz diagonal é triangular superior E inferior
  • A identidade é diagonal com 1 na diagonal
  • A matriz nula é um caso especial de todas as anteriores
  • Simétrica ≠ triangular (em geral)

verificador_completo.c — todas as funções em um programa
#include <stdio.h>
#define N 3
typedef int Mat[N][N];

int ehNula(Mat m)    { for(int i=0;i<N;i++) for(int j=0;j<N;j++) if(m[i][j]!=0) return 0; return 1; }
int ehIdentidade(Mat m) { for(int i=0;i<N;i++) for(int j=0;j<N;j++) if(m[i][j]!=((i==j)?1:0)) return 0; return 1; }
int ehSimetrica(Mat m)  { for(int i=0;i<N;i++) for(int j=i+1;j<N;j++) if(m[i][j]!=m[j][i]) return 0; return 1; }
int ehTriSup(Mat m)    { for(int i=1;i<N;i++) for(int j=0;j<i;j++) if(m[i][j]!=0) return 0; return 1; }
int ehTriInf(Mat m)    { for(int i=0;i<N-1;i++) for(int j=i+1;j<N;j++) if(m[i][j]!=0) return 0; return 1; }

int main() {
    Mat A = {{1,5,9},{5,2,7},{9,7,3}}; // simétrica
    printf("Nula?       %s\n", ehNula(A)       ? "Sim" : "Não");
    printf("Identidade? %s\n", ehIdentidade(A) ? "Sim" : "Não");
    printf("Simétrica?  %s\n", ehSimetrica(A)  ? "Sim" : "Não"); // Sim!
    printf("Tri.Sup?    %s\n", ehTriSup(A)     ? "Sim" : "Não");
    printf("Tri.Inf?    %s\n", ehTriInf(A)     ? "Sim" : "Não");
    return 0;
}

🔍 Busca em Matrizes

Encontrar um valor em uma tabela bidimensional

Busca em matriz = busca linear percorrendo todos os L×C elementos com dois laços aninhados. Retorna a posição [i][j] ou indica "não encontrado".
busca_matriz.c — retorna linha e coluna
#include <stdio.h>
#define L 3
#define C 4

/* Busca linear em matriz
   Preenche *li e *co com a posição encontrada
   Retorna 1 se achou, 0 se não achou          */
int buscaMatriz(int m[][C], int chave,
                 int *li, int *co) {
    for(int i=0;i<L;i++)
        for(int j=0;j<C;j++)
            if(m[i][j]==chave) {
                *li=i; *co=j;
                return 1; // retorno antecipado!
            }
    return 0;
}

int main() {
    int m[L][C] = {{2,4,6,8},
                   {1,3,5,7},
                   {9,11,13,15}};
    int chave, lin, col;

    printf("Buscar: "); scanf("%d",&chave);

    if(buscaMatriz(m,chave,&lin,&col))
        printf("Encontrado em [%d][%d]\n",lin,col);
    else
        puts("Não encontrado.");
    return 0;
}

💡 Retorno de posição com ponteiros

  • A função precisa retornar dois valores: linha e coluna
  • Em C, uma função só retorna 1 valor diretamente
  • Solução: usar ponteiros (int *li, *co) — passagem por referência
  • Chamada: buscaMatriz(m, chave, &lin, &col)
  • Dentro da função: *li = i; *co = j; modifica as variáveis originais
  • Prévia da Unidade III — Sub-rotinas com ponteiros!

🔍 Busca de todas as ocorrências

  • Se a chave pode aparecer múltiplas vezes, não use retorno antecipado
  • Continue o laço até o final, contando ou armazenando cada posição
  • Use um contador e imprima cada [i][j] encontrado
  • Complexidade sempre O(L×C) — percorre toda a matriz

⚠️ Diferença: busca linear em vetor vs matriz

Vetor: 1 laço, retorna índice i. Matriz: 2 laços, retorna par [i][j]. O conceito é o mesmo — a segunda dimensão apenas adiciona o laço interno e um índice a mais.

🔃 Ordenação em Matrizes

Ordenar linhas individualmente · linha mais pesada · todas as linhas

🎯 Três abordagens de ordenação em matrizes

Não existe "ordenar a matriz inteira" de forma trivial — depende do critério. As abordagens mais comuns são: (1) ordenar cada linha individualmente; (2) ordenar as linhas pelo valor de uma coluna específica (ex: coluna de notas); (3) achatar a matriz em vetor, ordenar e reconstituir.

ordenar_linhas.c — Bubble Sort em cada linha
#define L 3
#define C 4

/* Ordena cada linha da matriz individualmente */
void ordenarLinhas(int m[][C]) {
    for(int i=0;i<L;i++) {          // para cada linha
        /* Bubble Sort na linha i */
        for(int p=0;p<C-1;p++) {
            for(int j=0;j<C-1-p;j++) {
                if(m[i][j]>m[i][j+1]) {
                    int t=m[i][j];
                    m[i][j]=m[i][j+1];
                    m[i][j+1]=t;
                }
            }
        }
    }
}
/* Antes: {{5,3,8,1},{4,9,2,7},{6,1,5,3}}  */
/* Depois: {{1,3,5,8},{2,4,7,9},{1,3,5,6}} */
ordenar por coluna específica (Selection Sort)
/* Ordena LINHAS por valor da coluna col
   Mantém cada linha intacta (como um registro) */
void ordenarPorColuna(int m[][C], int col) {
    for(int i=0;i<L-1;i++) {
        int pm=i;
        for(int k=i+1;k<L;k++)
            if(m[k][col]<m[pm][col]) pm=k;
        if(pm!=i) {  /* troca linhas inteiras */
            for(int j=0;j<C;j++) {
                int t=m[i][j];
                m[i][j]=m[pm][j];
                m[pm][j]=t;
            }
        }
    }
}
/* Uso: ordenarPorColuna(notas, 3) */
/* Ordena turmas pelo aluno da col 3 */

🔑 Troca de linhas inteiras

  • Para trocar duas linhas completas: laço em j e troca m[i][j] ↔ m[pm][j]
  • Mantém a coerência de cada "registro" (linha) ao reordenar
  • Mesmo princípio dos vetores paralelos da Aula 05!

esqueleto Ex 03 — ranking de turmas com ordenação
#include <stdio.h>
#include <string.h>
#define NT 4   // turmas (linhas)
#define NA 5   // alunos (colunas)

/* Calcula média de cada linha */
void calcMedias(float m[NT][NA], float med[]) {
    for(int i=0;i<NT;i++) {
        float s=0;
        for(int j=0;j<NA;j++) s+=m[i][j];
        med[i]=s/NA;
    }
}

/* Selection Sort decrescente por media[] — troca linhas e nomes */
void ordenarTurmas(float m[NT][NA], float med[], char nomes[NT][20]) {
    for(int i=0;i<NT-1;i++) {
        int pm=i;
        for(int k=i+1;k<NT;k++) if(med[k]>med[pm]) pm=k;
        if(pm!=i) {
            /* troca linha inteira */
            for(int j=0;j<NA;j++) { float t=m[i][j]; m[i][j]=m[pm][j]; m[pm][j]=t; }
            float tm=med[i]; med[i]=med[pm]; med[pm]=tm;
            char tn[20]; strcpy(tn,nomes[i]); strcpy(nomes[i],nomes[pm]); strcpy(nomes[pm],tn);
        }
    }
}

🎯 Resumo e Próxima Aula

Casos especiais dominados — Unidade II quase completa!

Reconhecer matrizes especiais é a diferença entre um algoritmo genérico lento e uma solução elegante e eficiente! 🔷

✅ Matriz Nula e Identidade

Nula: todos zeros. Identidade: diagonal 1, resto 0 ((i==j)?1:0). Verificadas com 2 laços em O(n²).

✅ Simétrica e Antissimétrica

Simétrica: m[i][j]==m[j][i] — otimize com j=i+1. Antissimétrica: m[i][j]==-m[j][i], diagonal sempre zero.

✅ Triangulares e Diagonal

Superior: zeros abaixo (i>j). Inferior: zeros acima (i<j). Diagonal: zeros fora da diagonal (i≠j).

✅ Busca em Matriz

2 laços, retorno antecipado ao encontrar. Posição retornada via ponteiros *li, *co. O(L×C) sempre.

✅ Ordenação de Linhas

Bubble/Selection Sort adaptado: Trocar linhas inteiras para preservar a coerência de cada "registro".
