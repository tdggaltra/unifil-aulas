🎯 O Problema de Busca

Dado um vetor com N elementos e um valor chave, encontrar a posição onde a chave se encontra — ou informar que ela não existe. É um dos problemas mais fundamentais da Computação: pesquisa em banco de dados, sistemas de GPS, mecanismos de busca — tudo usa alguma variação desse conceito.

🔴 Busca Linear

Percorre o vetor do início ao fim até encontrar a chave ou chegar ao final. Funciona em qualquer vetor — ordenado ou não. Simples mas pode ser lento para N grande.

🔵 Busca Binária

Divide o vetor ao meio repetidamente, descartando metade dos candidatos a cada passo. Exige que o vetor esteja previamente ordenado. Muito mais rápida — O(log n).

🔴 Busca Linear — Conceito e Pseudocódigo

O algoritmo mais simples e universal de busca

Percorra o vetor do índice 0 ao N-1. Se v[i] == chave, encontrou! Retorne a posição. Se percorreu tudo sem encontrar, retorne -1.

📐 Pseudocódigo — Busca Linear
// Entrada: vetor v[N] e valor chave a buscar
// Saída: posição da chave (-1 se não encontrada)
PARA i DE 0 ATÉ N-1 FAÇA
    SE v[i] == chave ENTÃO
        RETORNE i          // encontrou!
    FIM-SE
FIM-PARA
RETORNE -1              // não encontrada

🔴 Busca Linear — Implementação

Portugol · C · Java — com sentinela e retorno antecipado

programa { funcao inicio() { inteiro v[7] = {4, 9, 3, 7, 11, 2, 8} inteiro chave, pos = -1 escreva("Digite o valor a buscar: ") leia(chave) para (inteiro i = 0; i < 7; i++) { se (v[i] == chave) { pos = i pare // interrompe o laço ao encontrar } } se (pos != -1) { escreva("Encontrado na posição " + pos) } senao { escreva("Valor não encontrado") } } }
#include <stdio.h> #define N 7 /* Função de busca linear — retorna posição ou -1 */ int buscaLinear(int v[], int n, int chave) { for (int i = 0; i < n; i++) { if (v[i] == chave) return i; // retorno antecipado! } return -1; // não encontrado } int main() { int v[N] = {4, 9, 3, 7, 11, 2, 8}; int chave, pos; printf("Valor a buscar: "); scanf("%d", &chave); pos = buscaLinear(v, N, chave); if (pos != -1) printf("Encontrado na posição %d\n", pos); else printf("Não encontrado.\n"); return 0; }
import java.util.Scanner; public class BuscaLinear { // Método estático reutilizável static int buscaLinear(int[] v, int chave) { for (int i = 0; i < v.length; i++) { if (v[i] == chave) return i; } return -1; } public static void main(String[] args) { Scanner sc = new Scanner(System.in); int[] v = {4, 9, 3, 7, 11, 2, 8}; System.out.print("Valor a buscar: "); int chave = sc.nextInt(); int pos = buscaLinear(v, chave); if (pos != -1) System.out.println("Encontrado na posição " + pos); else System.out.println("Não encontrado."); } }

🔑 Pontos de Destaque

  • Retorno -1: convenção para "não encontrado" — posições válidas são ≥ 0
  • Retorno antecipado: ao encontrar, sai do laço imediatamente (economiza comparações)
  • Em C: a função recebe v[] — array decai para ponteiro (veremos na Unidade III)
  • Versão com pare (Portugol) ou variável achou são equivalentes

📊 Complexidade

  • Melhor caso O(1): chave está na posição 0
  • Caso médio O(n/2): chave está no meio
  • Pior caso O(n): chave no fim ou não existe
  • Para N=1.000.000 → até 1.000.000 comparações

🔵 Busca Binária — O Poder de Dividir

De O(n) para O(log n) — a condição é o vetor estar ordenado

📖 Analogia: Dicionário Impresso

Para encontrar a palavra "Python" num dicionário, você não lê da página 1. Você abre no meio — se está em "M", vai para a segunda metade. Abre no meio dessa metade — "S", vai para o quarto anterior. E assim até encontrar "P". Esse é exatamente o raciocínio da busca binária!

⚡ 10 elementos → apenas 3 comparações!

A busca linear precisaria de até 10 comparações. A binária precisou de 3. Para 1.000.000 elementos, a linear precisa de até 1.000.000 comparações — a binária precisa de apenas 20 (log₂ 1.000.000 ≈ 20).

🔵 Busca Binária — Pseudocódigo e Trace

As três variáveis essenciais: esq · dir · meio

📐 Pseudocódigo — Busca Binária
// Pré-condição: v[0..N-1] está ORDENADO!
esq  ← 0
dir  ← N - 1

ENQUANTO esq <= dir FAÇA
    meio ← (esq + dir) / 2   // divisão inteira!

    SE v[meio] == chave ENTÃO
        RETORNE meio       // encontrou!

    SENÃO SE v[meio] < chave ENTÃO
        esq ← meio + 1    // busca à direita
    SENÃO
        dir ← meio - 1    // busca à esquerda
    FIM-SE
FIM-ENQUANTO
RETORNE -1              // não encontrada

💡 Por que esq <= dir?

  • Enquanto o intervalo de busca for válido (esq ≤ dir) ainda há elementos para verificar
  • Quando esq > dir, o intervalo ficou vazio — o elemento não existe
  • O meio é calculado com divisão inteira: (0+9)/2 = 4

🔵 Busca Binária — Implementação

Portugol · C · Java — laço while com três variáveis de controle

programa { funcao inicio() { // VETOR DEVE ESTAR ORDENADO! inteiro v[10] = {2,5,8,12,16,23,38,45,56,72} inteiro chave, esq = 0, dir = 9, meio, pos = -1 escreva("Valor a buscar: ") leia(chave) enquanto (esq <= dir) { meio = (esq + dir) / 2 // divisão inteira se (v[meio] == chave) { pos = meio pare // encontrou! } senao se (v[meio] < chave) { esq = meio + 1 // busca à direita } senao { dir = meio - 1 // busca à esquerda } } se (pos != -1) { escreva("Posição: " + pos) } senao { escreva("Não encontrado") } } }
#include <stdio.h> #define N 10 /* Função de busca binária — v[] deve estar ordenado */ int buscaBinaria(int v[], int n, int chave) { int esq = 0, dir = n - 1, meio; while (esq <= dir) { meio = (esq + dir) / 2; if (v[meio] == chave) return meio; else if (v[meio] < chave) esq = meio + 1; else dir = meio - 1; } return -1; } int main() { int v[N] = {2,5,8,12,16,23,38,45,56,72}; int chave, pos; printf("Valor a buscar: "); scanf("%d", &chave); pos = buscaBinaria(v, N, chave); (pos != -1) ? printf("Encontrado na posição %d\n", pos) : printf("Não encontrado.\n"); return 0; }
import java.util.Scanner; public class BuscaBinaria { static int buscaBinaria(int[] v, int chave) { int esq = 0, dir = v.length - 1; while (esq <= dir) { int meio = esq + (dir - esq) / 2; // evita overflow! if (v[meio] == chave) return meio; else if (v[meio] < chave) esq = meio + 1; else dir = meio - 1; } return -1; } public static void main(String[] args) { int[] v = {2,5,8,12,16,23,38,45,56,72}; Scanner sc = new Scanner(System.in); System.out.print("Valor a buscar: "); int pos = buscaBinaria(v, sc.nextInt()); System.out.println(pos != -1 ? "Posição: " + pos : "Não encontrado."); } }

⚠️ Overflow em Java/C — Use esq + (dir-esq)/2

A fórmula (esq+dir)/2 pode causar overflow de inteiro quando esq e dir são valores muito grandes (ex: N próximo de INT_MAX). A forma segura é esq + (dir-esq)/2, que produz o mesmo resultado sem risco de overflow.

🎓 Regra Prática

Se o vetor já está ordenado e raramente muda → use binária. Se o vetor muda frequentemente ou é pequeno → use linear. Na prática, bibliotecas padrão (C: bsearch(), Java: Arrays.binarySearch()) já implementam busca binária otimizada.

/* Verifica se v[] está em ordem crescente */ int estaOrdenado(int v[], int n) { for (int i = 0; i < n - 1; i++) if (v[i] > v[i+1]) return 0; // achou desordem return 1; // tudo em ordem } /* Uso no main: */ if (!estaOrdenado(v, N)) { printf("Erro: vetor não está ordenado!\n"); return 1; }
// Busca binária com contagem de comparações static int[] buscaBinariaComContagem(int[] v, int chave) { int esq = 0, dir = v.length-1, comp = 0; while (esq <= dir) { int meio = esq + (dir-esq)/2; comp++; // conta comparação if (v[meio] == chave) return new int[]{meio, comp}; else if (v[meio] < chave) esq = meio + 1; else dir = meio - 1; } return new int[]{-1, comp}; }

4. 💬 Análise Crítica:

Um sistema de cadastro de 500.000 usuários realiza milhares de buscas por segundo por nome de usuário. Os registros são mantidos em um array ordenado alfabeticamente, raramente inserindo novos usuários. Qual algoritmo de busca você escolheria e por quê? Justifique com números.

🎯 Resumo e Próxima Aula

Dois algoritmos de busca, uma decisão fundamentada

Busca linear é universal mas lenta. Busca binária é incrivelmente rápida — mas exige vetor ordenado. Saiba quando usar cada uma! 🔍

✅ Busca Linear O(n)

Percorre do índice 0 ao N-1. Retorna a posição ou -1. Funciona em qualquer vetor. Pior caso: N comparações.

✅ Busca Binária O(log n)

Três variáveis: esq, dir, meio. Divide o intervalo ao meio a cada iteração. Pré-requisito: vetor ordenado.

✅ Comparativo de Eficiência

N=1.000.000 → linear: 1.000.000 comparações; binária: apenas 20. A diferença cresce com N.

✅ Quando usar cada uma

Linear: vetor pequeno, não ordenado, com mudanças frequentes. Binária: vetor grande, ordenado, estático.

✅ Implementação limpa em C

Funções separadas buscaLinear() e buscaBinaria() — retornam posição ou -1. Reutilizáveis em qualquer contexto.
