📌 Recapitulando o padrão de percurso

  • Vetor de N elementos → índices válidos: 0 até N-1
  • Laço padrão: for(i=0; i < N; i++)
  • Acesso ao elemento: v[i]
  • Em C: leitura com scanf("%tipo", &v[i])
  • Elementos em memória contígua → acesso O(1)

Σ Algoritmo de Soma

Acumulando valores com um único percurso

📐 Pseudocódigo — Soma de vetor
// Entrada: vetor v com N elementos inteiros
// Saída: soma de todos os elementos
soma ← 0                       // acumulador inicia em zero
PARA i DE 0 ATÉ N-1 FAÇA
    soma ← soma + v[i]        // acumula cada elemento
FIM-PARA
ESCREVA "Soma = ", soma
programa { funcao inicio() { inteiro N = 6 inteiro v[6] = {4, 9, 12, 7, 2, 6} inteiro soma = 0 para (inteiro i = 0; i < N; i++) { soma = soma + v[i] // acumula } escreva("Soma = " + soma) // Soma = 40 } }
#include <stdio.h> #define N 6 int main() { int v[N] = {4, 9, 12, 7, 2, 6}; int soma = 0; for (int i = 0; i < N; i++) { soma += v[i]; // soma += v[i] ≡ soma = soma + v[i] } printf("Soma = %d\n", soma); // Soma = 40 return 0; }
public class Soma { public static void main(String[] args) { int[] v = {4, 9, 12, 7, 2, 6}; int soma = 0; for (int i = 0; i < v.length; i++) { soma += v[i]; } // alternativa moderna: for-each int soma2 = 0; for (int x : v) soma2 += x; System.out.println("Soma = " + soma); // 40 } }

💡 Ponto-chave: o acumulador

O acumulador soma deve ser inicializado com zero antes do laço. Se não for inicializado, conterá lixo de memória em C — e a soma será incorreta. Em Java e Portugol o compilador avisa, em C não.

x̄ Algoritmo de Média Aritmética

Soma dividida pela quantidade — e o cuidado com a divisão inteira

Média = Soma de todos os elementos / Quantidade de elementos  →  x̄ = Σv[i] / N
📐 Pseudocódigo — Média
soma ← 0
PARA i DE 0 ATÉ N-1 FAÇA
    soma ← soma + v[i]
FIM-PARA
media ← soma / N      // atenção ao tipo!
ESCREVA "Média = ", media

⚠️ Armadilha — Divisão Inteira em C

  • Se soma e N são int, a divisão soma/N é inteira — trunca o resultado!
  • 40 / 6 em inteiros = 6 (não 6.67!)
  • Solução: usar float ou cast → (float)soma / N
  • Em Java o mesmo problema ocorre com int/int
inteiro v[6] = {4,9,12,7,2,6} real soma = 0, media para (inteiro i = 0; i < 6; i++) { soma = soma + v[i] } media = soma / 6 escreva("Média = " + media) // 6.666...
int v[N] = {4,9,12,7,2,6}; int soma = 0; float media; for(int i=0; i<N; i++) soma += v[i]; /* cast garante divisão real */ media = (float)soma / N; printf("Média = %.2f\n", media); // Média = 6.67
int[] v = {4,9,12,7,2,6}; int soma = 0; double media; for(int x : v) soma += x; // cast para double antes da divisão media = (double)soma / v.length; System.out.printf("Média = %.2f%n", media); // Média = 6.67

↑ Algoritmo do Maior Valor (Máximo)

Percorrer e comparar — a estratégia do "campeão atual"

🏆 Estratégia: "Campeão Atual"

Assuma que o primeiro elemento é o maior. Percorra o restante comparando cada elemento com o "campeão atual". Se encontrar um maior, ele vira o novo campeão. No final, o campeão é o máximo.

📐 Pseudocódigo — Máximo
maior ← v[0]              // assume 1º como maior
posMax ← 0                // posição do maior

PARA i DE 1 ATÉ N-1 FAÇA  // começa no índice 1!
    SE v[i] > maior ENTÃO
        maior  ← v[i]
        posMax ← i
    FIM-SE
FIM-PARA
ESCREVA "Máx = ", maior, " na posição ", posMax

int v[N] = {4,9,12,7,2,6}; int maior = v[0], posMax = 0; for (int i = 1; i < N; i++) { // começa em 1! if (v[i] > maior) { maior = v[i]; posMax = i; } } printf("Maior: %d na pos %d\n", maior, posMax); // Maior: 12 na pos 2
int[] v = {4,9,12,7,2,6}; int maior = v[0], posMax = 0; for (int i = 1; i < v.length; i++) { if (v[i] > maior) { maior = v[i]; posMax = i; } } System.out.printf("Maior: %d na pos %d%n", maior, posMax); // Maior: 12 na pos 2

💡 Detalhe importante: começa em 1

  • O laço começa em i = 1 porque já assumimos que v[0] é o maior
  • Comparar v[0] com ele mesmo seria inútil
  • O mesmo vale para o mínimo — começar em 1 é mais eficiente

↓ Algoritmo do Menor Valor (Mínimo)

Mesma estratégia do máximo — apenas a comparação muda

Máximo usa > (maior que).  Mínimo usa < (menor que).  Todo o resto do algoritmo é idêntico!
📐 Pseudocódigo — Mínimo
menor ← v[0]             // assume 1º como menor
posMin ← 0

PARA i DE 1 ATÉ N-1 FAÇA
    SE v[i] < menor ENTÃO  // < em vez de >
        menor  ← v[i]
        posMin ← i
    FIM-SE
FIM-PARA
ESCREVA "Mín = ", menor, " na posição ", posMin

🔄 Máx e Mín no mesmo laço

É possível calcular máximo e mínimo em um único percurso, com duas variáveis e dois if dentro do mesmo laço — mais eficiente para vetores grandes.

💻 Programa Completo em C

Todos os quatro algoritmos juntos com leitura do usuário

estatisticas_vetor.c
#include <stdio.h> #define N 6 int main() { int v[N]; int soma = 0, maior, menor, posMax = 0, posMin = 0; float media; /* ── Leitura ── */ printf("Digite %d valores inteiros:\n", N); for (int i = 0; i < N; i++) { printf("v[%d]: ", i); scanf("%d", &v[i]); } /* ── Processamento ── */ maior = v[0]; menor = v[0]; // inicializa com 1º elemento for (int i = 0; i < N; i++) { soma += v[i]; // soma if (v[i] > maior) { maior = v[i]; posMax = i; } // max if (v[i] < menor) { menor = v[i]; posMin = i; } // min } media = (float)soma / N; /* ── Saída formatada ── */ printf("\n── Estatísticas ──\n"); printf("Soma : %d\n", soma); printf("Média : %.2f\n", media); printf("Máximo : %d (posição %d)\n", maior, posMax); printf("Mínimo : %d (posição %d)\n", menor, posMin); return 0; }

✅ Saída esperada (entrada: 4 9 12 7 2 6)

  • Soma : 40
  • Média : 6.67
  • Máximo : 12 (posição 2)
  • Mínimo : 2 (posição 4)

🔑 Boas Práticas Aplicadas

  • Único laço para soma + máx + mín → 1 percurso
  • Cast (float) antes da divisão
  • Variáveis inicializadas com v[0]
  • Saída alinhada com \t ou espaços
  • Constante #define N para fácil manutenção

🔢 Contagem e Verificação em Vetores

Contar elementos que satisfazem uma condição — padrão essencial

🎯 Problema típico

Dado um vetor de notas, quantos alunos foram aprovados (nota ≥ 6.0)? Quantos ficaram acima da média? Esses problemas exigem um contador — variável inicializada em 0 que incrementa quando a condição é verdadeira.

// Conta aprovados (nota >= 6.0) e acima da média real notas[5] = {7.5, 4.0, 8.5, 5.5, 9.0} real soma = 0, media inteiro aprovados = 0, acimaMedia = 0 para (inteiro i = 0; i < 5; i++) { soma = soma + notas[i] se (notas[i] >= 6.0) { aprovados = aprovados + 1 } } media = soma / 5 para (inteiro i = 0; i < 5; i++) { se (notas[i] > media) { acimaMedia = acimaMedia + 1 } } escreva("Aprovados: " + aprovados) // 3 escreva("Acima da média: " + acimaMedia) // 3
#include <stdio.h> #define N 5 int main() { float notas[N] = {7.5f, 4.0f, 8.5f, 5.5f, 9.0f}; float soma = 0, media; int aprovados = 0, acimaMedia = 0; /* 1ª passagem: soma e conta aprovados */ for (int i = 0; i < N; i++) { soma += notas[i]; if (notas[i] >= 6.0f) aprovados++; } media = soma / N; /* 2ª passagem: conta acima da média */ for (int i = 0; i < N; i++) if (notas[i] > media) acimaMedia++; printf("Aprovados: %d\nAcima da média: %d\n", aprovados, acimaMedia); return 0; }
public class Contagem { public static void main(String[] args) { float[] notas = {7.5f, 4.0f, 8.5f, 5.5f, 9.0f}; float soma = 0, media; int aprovados = 0, acimaMedia = 0; for (float n : notas) { soma += n; if (n >= 6.0f) aprovados++; } media = soma / notas.length; for (float n : notas) if (n > media) acimaMedia++; System.out.println("Aprovados: " + aprovados); System.out.println("Acima da média: " + acimaMedia); } }

💡 Por que 2 passagens?

Para saber quem ficou acima da média, precisamos primeiro calcular a média — o que requer a soma de todos os elementos. Por isso são necessárias duas passagens pelo vetor: a 1ª calcula a média, a 2ª conta quem ficou acima dela.
